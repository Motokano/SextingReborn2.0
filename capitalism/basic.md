## 九、贸易与旅行商人系统

> 说明：本章中涉及的“货车”“役兽”“车队头”等与运输相关的称呼，以及各类“店铺”“行会”“职业”“行脚商人”等商业与从业者称呼，均为便于描述规则而使用的技术性用语。实际游戏世界内，不同行省、不同文化圈会对同类事物与从业者使用各自的在地叫法。玩家在 `语言` 技能与长期交往提升后，才会逐渐将这些地方称呼统一理解为同一类概念。

### 9.1 设计目标与定位

- **玩法定位**：在「基地–野外–城镇–地牢」结构上，新增一条**旅行商人**长期发展路线，可与战斗/探索并行，不强制。
- **核心体验**：在不同主城与聚落之间运货跑商、利用价格差与情报获利；在**市场 / 店铺 / 交易中心**以不同复杂度完成从小额零售到批发、再到高风险投机的升级。
- **与现有系统勾连**：依赖 `贸易`、`谈判`、`物流管理`、`语言`、`鉴定` 等技能，以及城镇与野外设定。

### 9.2 交易场景分级

- **市场（Market）——小额零售区**
  - 每个主城/聚落有「市场」区域，进入后可与摊贩/NPC 交互。
  - 单次交易金额与数量有上限；仅简单买卖，价格波动较小；提供基础情报（如某类货紧缺/过剩）。
  - 适合试水与小规模套利。
- **店铺（Shop）——专业交易点**
  - 城镇内专门店铺（武器店、药铺、布庄等），有门类限制。
  - 支持批量采购与出售，受店家「最大承接额」限制；同一物品有进货价/回收价差异（与好感度、谈判相关）；支持长期订单（未来 N 天内按约定价格收购）。
  - 店铺库存受时间与玩家行为影响，可能被买空或压价。
  - 适合中期稳定跑商与稳定货源。
- **交易中心（Exchange）——高级复杂交易**
  - 仅在少数主城开放（类似拍卖行/商会）。
  - 委托挂单（求购/求售，设定数量、价格、有效时间）；NPC 商队也挂单构成市场深度。
  - 批量/远期合约（如 N 天后交付某货，预收定金，违约扣定金并降声望）。
  - 围绕事件（战乱、节庆等）的临时价格异常与投机；付费情报服务（概率性预期价格区间）。
  - 适合后期将跑商作为主业或重要副业。

#### 9.2.1 交易上下文与四档规则对照

- **交易界面的打开**：玩家与商人 NPC 对话，在可选项中选择「交易」后打开交易界面；本次交易的场景类型（Market / Shop / Exchange / Travel）与可交易对象由当前所处地点及对话对象共同决定。
- **货币交易与以物易物**：同一交易界面内同时支持**货币买卖**与**以物易物**；玩家可选用货币支付或选用己方物品抵换，一次确认可完成一笔（含混合交易）。以物易物时**不向玩家展示明确的价值数字**；仅在**超出持有货币的部分**给予**模糊暗示**（如对方是否显得犹豫、是否爽快等），具体形式留实现或文案定。底层价值评估与 NPC 主观估值见 9.7。
- **界面内不切换商人**：当前版本不支持在交易界面内切换商人；欲与同场景其他商人交易，需先关闭界面，再与另一 NPC 对话并选择「交易」。
- **交易上下文的含义**：玩家每次打开交易界面，都处于一个「交易上下文」中。实现时可用一组字段描述：`scene_type`（Market / Shop / Exchange / Travel）、`location_id`（可选，Travel 时为空）、`merchants[]`（当前可交互的商人列表）。所有限额、是否允许挂单/合约等，由 `scene_type` 决定，商人只提供「谁在卖、卖什么、态度与资金」。
- **场景与地点的关系**：城镇/据点内的交易场所由「地点」挂载场景。例如主城可有一个 Market、若干 Shop、至多一个 Exchange；小村可能只有一个 Market 或一个小型 Shop。Travel 场景不绑定地点，只在野外/道路自动行进时临时创建，`location_id` 为空或仅记录父场景/事件 ID。
- **场景与商人类型的关系**：
  - `merchant_type = 摊贩` → 仅出现在 **Market**。
  - `merchant_type = 店主` → 出现在 **Shop**（个别活动时可临时参与 Exchange，由事件控制）。
  - `merchant_type = 旅行商队` → 可出现在 **Market**（进城时）、**Exchange**（少数主城），以及 **Travel**（野外/道路自动行进时）。
- **四档场景规则对照**（实现时按 scene_type 查表）：


| 项目              | Market    | Shop        | Exchange  | Travel          |
| --------------- | --------- | ----------- | --------- | --------------- |
| 单笔金额/数量上限       | 有         | 受「最大承接额」等限制 | 按挂单/合约规则  | 有（同 Market 或略低） |
| 批量买卖            | 否/极有限     | 是           | 是         | 否/极有限           |
| 长期订单（N 天内约定价收购） | 否         | 是           | —         | 否               |
| 委托挂单（求购/求售）     | 否         | 否           | 是         | 否               |
| 批量/远期合约         | 否         | 否           | 是         | 否               |
| 情报等级            | 基础（紧缺/过剩） | 可扩展         | 付费情报、事件预期 | 无或极简            |
| 出现条件            | 主城/聚落市场区域 | 城镇内专门店铺     | 少数主城      | 仅野外/道路自动行进      |
| 典型商人类型          | 摊贩        | 店主          | 旅行商队、部分店主 | 旅行商队            |


- **Travel 相对 Market 的数值差异**（具体系数留待场景参数表）：Travel 场景中旅行商人可在区域基准价上加一定溢价；常规补给品数量有限，小概率携带少量稀有货；触发频率由路径事件池权重与冷却控制，见 9.4.1。
- **选「交易」后校验不通过时的反馈**：区分两种情况并采用不同提示文案，例如「此地不宜谈生意」（当前地图不可交易）与「对方不愿在此地交易」（该商人不允许在当前 scene_type 交易）。**实现备注**：商人类型（merchant_type）与 scene_type 的匹配规则需做成可配置（如配置表或可改常量），便于后续修改，实现时请备注。

#### 9.2.2 单次会话与周期内购买限制（防钻空子）

- **默认行为**：理想状态下，商人可售物品**仅受库存限制**，库存售完即止；不做「一次会话/一周期内总购买件数」的全局上限。本小节限制机制仅针对**需要单独限量的具体物品**（如稀有品、任务相关品等），避免玩家通过短时间内多次少量购买突破设计意图。
- **单次会话**：此处「单笔」指**打开一次交易/对话窗口**这一行为。**结算时机**：以任何形式关闭交易界面（含 Esc、点关闭、切场景等）**均不自动结算**；只有玩家**点击结算按钮**时才会执行结算操作，真正修改双方库存与资金。会话内未点结算即关闭的，本会话内已选数量不生效。对**未配置限额**的物品，可买量仅受商人当前可售库存约束。
- **按物品的限额**：对**指定物品**可配置「该商人（或该场景）下」的**会话上限**与**周期内上限**：本会话内该物品最多卖给该玩家 N 个（session_cap），本周期内最多卖 M 个（period_cap）；计量按**堆叠单位**（每买 1 个堆叠单位计 1 件）。未出现在限额配置中的物品无此限制，仅受库存约束。**以物易物换入**的该物品件数**同样计入**本会话/本周期已购量，与用货币购买共用同一限额。实际可买量 = min(该物品会话剩余额度, 该物品周期剩余额度, 商人当前该物品库存)。
- **限额配置**：具体物品的限额由单独配置表（如 item_trade_limits.csv）给出，字段可含：物品 ID、商人 ID 或 scene_type、session_cap、period_cap、period_duration。周期按**玩家–商人**维度、按物品分别累计；同一周期内从商人 A 某物品买满后，仍可从商人 B、C 购买，或从同一商人处购买其他未限物品，为玩家保留操作空间与成就感。
- **周期长度（period_duration）**：如 1 游戏日、6 小时等。可由配置表按 scene_type 或按（物品、商人）给出；同一场景内可共用默认周期长度。
- **特殊 NPC 覆盖**：若该商人被标记为特殊 NPC 并对某物品配置了自有的 session_cap / period_cap / period_duration，则采用该商人配置，覆盖场景或通用配置。
- **达上限时的反馈**：当玩家对**某物品**的购买已达该物品在本会话或本周期的上限时，**硬封顶**，该物品本次不可再购买；玩家对该物品再次按下购买时，弹出一行提示文案。
- **达上限文案规则**：
  - **会话到顶**与**周期到顶**使用不同文案（实现时根据触发的是 session_cap 还是 period_cap 选择）。
  - 文案中需代入该物品的实际上限值 **[N]**（会话上限或周期上限）。
  - 输出句子**随商人类型（merchant_type）与商人态度（merchant_attitude）变化**：同一「会话到顶」或「周期到顶」可因摊贩/店主/旅行商队、以及冷淡/谨慎/友好/热情等，呈现不同措辞与语气；具体语句由实现时配置或由文案老师细化，此处仅约定机制。
- **商人资金池与场景系数**：商人本有的资金池上限（见商人模板）可在本场景下乘以**场景系数**（merchant_pool_modifier），得到本场景下该商人实际可接纳的收购总量；空或 1.0 表示不修正。

#### 9.2.3 出售时资金池不足与超出部分换商誉

- **触发与确认**：当玩家向商人**出售**物品时，若该笔成交金额超过商人当前剩余资金池，先**弹出提示**，由玩家**确认是否仍要贩卖**。具体提示与确认按钮文案待定。
- **按单笔结算**：「超出部分」仅指**这一笔**中超出商人当前剩余资金池的金额；不按本会话累计。
- **确认贩卖后**：在资金池范围内的部分正常结算（玩家获得对应资金，商人扣减资金池并收下货物）；**超出部分不产生资金收益**，但**在该地区产生商誉**，换算方式为按超出部分的价值（金额或内部价值）依公式折算为地区商誉增量，公式与数值留待实现阶段确定。
- **资金池归零后**：某次结算后若商人资金池变为 0，则本会话内玩家**不可再向该商人出售**；**购买**仍可进行（若设计上商人卖货会即时增加资金池，则可持续购买）。
- **相关文案**：上述提示、确认与反馈语句的具体措辞由实现时或文案老师细化，此处仅约定规则。

### 9.3 商品与价格（概要）

- **商品分类**：生活物资、生产材料、成品、稀有/奢侈品；属性含稀有度、体积/重量、腐败性、易波动度。
- **区域基准价**：每区域每类商品有基准价；实际价格 = 基准价 × (1 + 供需差 + 事件修正) × 贸易/谈判系数 × 关系系数。
- **技能数值（示例）**：
  - `贸易`：熟练度影响买入折扣与卖出溢价。
  - `谈判`：在与行商/远道商队/交易中心交易时，影响稀有货出现几率与议价空间。
  - `物流管理`：降低腐败速度；**并影响代理跑商的任务耗时与收益**（见 9.5）。

### 9.3.1 货币与钱庄（概要）

- **货币**：以物品形式存在于背包与仓库中，占用格位与负重；本作无各地通用货币，流通性由商人个体决定（每个商人配置接受的货币短码，见 `trader_template.md` 商人模板与 `items/currency_base.csv`）。**约定**：文档中凡涉及金额、支付、奖励、示例数值处，均以**游戏内已有货币名称与单位**为准，不写死「金币」等称呼。
- **钱庄**：提供货币兑换。当前约定——（1）**低级货币跨区兑换**：钱庄可将其他地区的低级货币按 **1:1** 兑成任意一种低级货币，不分买卖价；（2）**高级货币**：仅本地区钱庄可办理该地区高级货币的兑换，具体牌价由后续钱庄模板与表实现。详细规则与字段见 `items_template_and_style.md` 中「三、货币类物品 · 钱庄与货币的交换方式」。

### 9.4 旅行商人核心循环与风险

- **循环**：筹备 → 采购 → 选路线 → 野外行进与事件 → 抵达城镇 → 市场试探 → 店铺批量/订单 → 交易中心高阶操作 → 结算后回基地或继续下一站。
- **运输风险**：劫匪/怪物袭击（高价值货易被盯上）；战败可能丢货或获得赃物线索；自然风险（暴风雨等）加速腐败、延误订单。
- **政治/法律风险**：关卡盘查与税费（`语言`、`谈判` 可减轻）；违禁品被查获则充公并降声望，可延伸走私支线。
- **经济风险**：短期大量涌入同类货导致行情崩盘；低信誉处进货有假货/品质纠纷，`鉴定` 可降低概率。

#### 9.4.1 无大地图版本的行进与旅途事件

- **当前版本不再使用城镇间抽象大地图**：原先设想的「城镇与据点之间采用抽象化大地图连接，出发前选择目的地或大致方向」的方案在当前版本中暂不实现。现在的游戏中，玩家只会在**单个场景内**沿预设路径进行自动行进，未来若重新启用城镇间大地图，将在新版文档中单独补充。
- **场景内的挂机方向与预设路径**：在标记为「野外 / 道路」的场景中，玩家可以选择一个挂机方向，系统沿该方向事先配置好的路径自动寻路前进（直线、环形或其他形状），省略逐格操作，仅在内部处理体力消耗、事件判定等。
- **旅途事件池（其他事件）**：每一条预设路径可挂有「旅途中事件池」，包含天气变化、路况波动、小规模战斗等事件。自动行进过程中，以「行进距离 / 行进时间」为单位定期 roll 事件，命中后暂停行进并进入对应事件流程。旅行商人的相遇方式见下条及 9.4.2。
- **挂机行进时与旅行商人的相遇**：当玩家**正在挂机行进**且**当前场景内存在旅行商人**时，触发事件：**打断**玩家挂机，**弹出选项**询问是否与商人交互。若本场景有多支旅行商人，则弹窗**列出所有可交互的旅行商人**供玩家选择一支；若仅一支则直接询问是否与该商人交互。玩家选**「是」**（并选定一支时）：与商人**隔空交互**（不要求角色走到商人身边、不改变玩家位置、不触发重新寻路）；交互结束后可继续挂机行进。玩家选**「否」**或关闭弹窗：无视商人，**继续**挂机行进。隔空交互时打开交易界面，`scene_type = Travel`，规则见本小节下文及 9.2.1。
- **Travel 场景的交易规则（沿用 Market 的简化版）**：Travel 场景在数值与交互上整体沿用「市场（Market）」的规则：以小额零售为主，存在单笔金额与数量上限，只允许即时买卖，不提供挂单、远期合约或代理跑商委托。与同一区域内的市场相比，旅行商人的价格允许在基准价基础上略高一些（体现「路上难得」），常规补给品数量有限，小概率携带少量稀有货物，具体系数留待后续在「场景参数表」中细化。
- **与商人类型的关系**：Travel 场景默认只由 `merchant_type = 旅行商队` 的商人出现；摊贩与固定店主仍只在 Market / Shop / Exchange 这三类城镇场景中活动。玩家在 Travel 场景中的交易主要影响该旅行商人个体的印象与后续遇见时的态度，不直接作用于某一城镇或地区的商会商誉。

#### 9.4.2 旅行商人在场景内的状态与移动

- **两种状态**：旅行商人在同一场景（地图）内具有两种行为状态，并在**固定间隔**（或按配置的规则）下切换：
  - **旅行**：商人自地图上的 **A 点**进入，朝 **B 点**缓慢移动，最终自 B 离开当前地图、进入下一张地图。移动过程中玩家可与之交互；**一旦玩家与之交互，商人停下**。玩家**结束交互**后，经过若干时间，商人**重新开始移动**并走向 B 离开。离开时为真实移动到 B 并切换至下一张图，并非从玩家身边消失；若玩家**沿同一方向进入下一张地图**，可在该图中再次遇到同一商人并继续交互，从而通过逐图跟随抵达商队**最终目的地**。
  - **扎营**：商人自 **A 点**进入地图，移动至地图上的**固定扎营点 T**，在 T **停留足够长时间**（扎营休息）。扎营结束后**切换为旅行状态**，自 T 向 B 移动，自 B 离开当前地图。
- **与挂机触发的衔接**：9.4.1 中「挂机行进时当前场景存在旅行商人即触发弹窗、选是则隔空交互」中的「存在」即指该商人正在本场景内处于上述旅行或扎营状态之一。

#### 9.4.3 远处发现旅行商人（事件提示与隐藏倒计时）

玩家在场景内（不限于挂机）可能触发**事件**，得知远处有旅行商人，从而有机会追过去交互：

- **触发与提示**：事件触发后，**打印一段文字**，提示玩家「某某方向存在一个商人」；同时开始**隐藏倒计时**（玩家不看到具体倒计时数字）。
- **倒计时内从该方向离图**：若玩家在倒计时结束前，从**通往该方向的出口**离开当前地图，则进入下一张图后**可见到该旅行商人并与之交互**；此种方式下遇到的商人**必然处于扎营状态**。
- **倒计时结束未离图**：若倒计时结束时玩家尚未从该出口离开，则再显示一行文案（如「旅行商人走远了」），本次机会结束。
- **与 9.4.1 的两种相遇方式**：与旅行商人相遇共有两种途径——（1）挂机行进时本场景内已存在旅行商人 → 打断挂机、弹窗列出可交互商人、选一支则隔空交互；（2）本条所述事件提示 → 隐藏倒计时 → 倒计时内从指定方向出口离图 → 在新地图见到该商人（扎营）并交互，超时则提示走远。

### 9.5 代理跑商（Agent）

玩家**不必亲自到场**即可通过**代理（随机生成的跑商小人）**执行交易：在任意安全地点下达指令，由代理在指定路线与目标城镇完成运输与买卖，结果在任务结束后异步结算。代理执行交易时，适用的 scene_type、按物品限额、资金池与结算规则与玩家亲自交易一致，见 9.2；若有不一致处再单独约定。

- **基本流程**
  - 雇佣代理：在城镇/基地雇佣一名随机生成 NPC 作为跑商代理（可见少量属性：可靠度、跑腿速度等）。
  - 下达委托：选择代理 → 设定起点仓库存货、目标城镇、交易策略（只卖/到价就卖/优先出清易腐品、仅市场/优先店铺等）。
  - 任务执行：任务进入队列，按游戏内时间或现实时间计时；玩家可继续探索、下地牢等。
  - 结算：结束时反馈实际耗时、卖出/买入明细、税费与代理分成、最终盈亏；有概率附带途中小事件（盘查、节市高价、暴雨损耗等）。
- **物流管理 对代理的数值影响**
  - **任务耗时**：基础耗时由路程、地形危险度、目标城镇繁忙度决定；`物流管理` 每 +1% 熟练度，任务总耗时 × (1 − 0.3%)（系数可调），即物流越高代理跑得越顺、用时越短。
  - **成交价格（收益）**：代理存在「代理损耗」（买入略贵、卖出略贱）；`物流管理` 可部分抵消：例如基础代理损耗 ±5%，每 10% 熟练度减少 1%，直至完全抵消，体现「流程顺畅、少被坑」。
  - **腐败与损耗**：路上时间越久，易腐品损耗越大；`物流管理` 延长腐败阈值、降低途中自然损耗概率，使代理更适合跑带腐败属性的货。
  - **与其他技能的扩展**
  - `贸易`：玩家自身的交易上限与折扣/溢价仍生效；代理视为「代为执行你的策略」，其上叠加的代理损耗由 `物流管理` 抵消。
  - `谈判`：代理仅跑市场/普通店铺时可不吃谈判；若跑交易中心/远道商队，可要求玩家曾亲自做过同类交易才开放代理，且代理任务只享受玩家 `谈判` 部分加成（如 50% 效率）。
  - 代理「可靠度」：低可靠度有小概率少交钱/货；高可靠度更稳但雇佣费高；`物流管理` 可提高「最低可靠度门槛」，便于筛人。
- **限制与防 AFK**
  - 同时进行的代理任务数量受 `物流管理` 等级或物流设施等级限制。
  - 同一路线重复派代理会导致该路线价格逐步调整、收益衰减，鼓励开拓新路线。
  - 高阶玩法（事件投机、黑市走私等）仍要求**玩家本人到场**，代理仅做常规货运与基础套利。

#### 9.5.1 解锁条件与成长路线

- **起点：只能亲自跑商**：玩家初始穿越到异世界时，`语言` 技能极低，对本地货名、地名、计量单位与价格说法理解非常有限，也缺乏对契约、欠条与违约等观念的共识。在此阶段，玩家既难以向他人清晰说明完整的委托内容，也缺乏任何可供信任的社会背景，因此只能依靠亲自扛货、赶车与面对面谈价完成最基础的跑商活动。
- **前提一：语言与行商逻辑的掌握**：随着 `语言` 技能提升与多次亲自跑商，玩家逐渐能够在本地语境下准确表达地点、时间、数量、价格与风险说明，听懂对方的疑问与附加条件，并理解本地对货款、预付、欠条与违约的基本处理方式。只有当玩家在认知层面真正“理解了这套行商逻辑”之后，世界中与之对应的代理委托结构才会在他眼前逐步显形。
- **前提二：本地隐性商誉与个人关系积累**：在至少一个行省内，玩家通过多次以相对合理价格进行的交易、小规模的救济性补给以及对既有承诺的履行，积累出一定的本地隐性商誉，并与若干关键 NPC（如店主、车把式、驿站负责人等）建立起可被视为“愿为其作证”的关系。只有在这些社会条件具备时，当地的跑腿者与小型商人才有动机接受玩家的代理委托。
- **阶段一：短距离的临时差事**：最初解锁的代理形式仅限于同一城镇或相邻聚落之间的小规模“有偿帮忙”，例如让熟悉的搬运工或伙计帮忙运送少量货物。这一阶段的代理更接近临时雇佣苦力，而非成熟的跨地区代理跑商系统，主要用于验证世界对“玩家差遣他人替自己完成简单事务”的接受程度。
- **阶段二：熟悉路线上的小型代理跑商**：当玩家在某条具体路线（如两镇之间）反复亲自往返，形成稳定的收益模式，并与沿线至少一两个节点 NPC 建立了较高信任后，才会逐步解锁在该特定路线上的小规模代理跑商能力。此时，玩家可以委托熟识的 NPC 代为在这条熟悉路线间往返，但运量、频率与可选路线仍受到明显限制。
- **阶段三：逐步形成跨地区的代理网络**：只有当玩家在多个行省内都具备一定隐性商誉、理解当地语言与习俗并拥有若干长期合作对象时，跨行省的代理跑商才会自然出现为可能选项。即便如此，代理网络也并非“任意地点即时派单”，而是在每个地区内，基于玩家过往行为与当地条件，逐渐生长出数量有限、形态各异的可委托模式与愿意为玩家跑商的人。

### 9.6 进阶与长期发展

- **隐性商人地位与商誉**：系统内部维护基于累计贸易额、完成订单数、覆盖城镇数等的隐性商人地位/商誉，用于驱动 NPC 对玩家态度与交易条件的变化，但不以「称号」「头衔」「面板显示」等形式告知玩家，仅通过对话语气、是否主动搭话、是否愿意透露内部货源、盘查时是否网开一面等行为反馈体现。

#### 9.6.1 外观与缓慢传播的联动

- **模糊外观分类**：系统只对玩家外观做模糊档位划分（例如：破落/潦倒、朴素小贩、正经商家、疑似富商、违和/可疑），不向玩家明示具体数值。不同 NPC 根据各自的性格与经验，对同一档外观给出不同解读与态度；在完全没有消息来源时，NPC 主要依赖“我眼前看到的你”来判断你是何种商人。
- **基于个人认知构建世界**：每个 NPC 的世界观只由自己能看到、能听到的东西构成。玩家在某地的诈骗或大买卖，只有在该 NPC 亲身经历或从可信圈子口耳相传后，才会进入他的认知；远处地区及无连接商路的地方，可能永远不知道这件事。
- **口耳相传的缓慢传播**：诈骗/大事件不会瞬间“全城皆知”，而是在有限的社交圈内缓慢扩散——例如同一店铺伙计、邻近摊贩、常来往的商队、同势力成员。部分小事件可能只在极少数 NPC 之间流传，最终被更新的八卦覆盖，永远只是零散传闻。
- **伪装与诈骗空间**：玩家可以通过光鲜亮丽或刻意违和的穿着、随从数量、显眼货物等手段，伪装成富商或奇怪的“外来大户”，诱导尚未听过自己名号的 NPC 接受极不对等的交易规则（如先交货后付款、大额赊账等）。伪装是否成功，取决于外观档位、玩家的圆滑/笨拙程度以及 NPC 的敏锐/迟钝程度。
- **NPC 的反应时间**：NPC 对异常交易的觉察不是即时完成，而是分阶段发生——当场根据外观与对话做初步判断，事后在盘点货物、回想细节或与他人闲聊时，才可能“恍然大悟”。玩家越笨拙、NPC 越聪明谨慎、场景越安静有利于思考，NPC 越可能快速意识到被骗，甚至当场拒绝或直接翻脸；反之则可能延迟很久才反应，甚至永远搞不清楚发生了什么。
- **后果的局部与迟到**：一旦某 NPC 反应过来自己被诈骗，他会在本地对玩家的隐性商誉做出极端负向更新，并记住玩家的部分外观与行为特征；此后，即便玩家更换外观，该 NPC 也会带着强烈的不信任来解读新的伪装。同时，他有概率在后续与少数熟人交谈中提起此事，缓慢而不完全地把“你可能是骗子”的印象扩散到更广的本地人群中。

#### 9.6.2 长期车队供给与合理定价

- **以小型车队为单位的供给**：本作默认以小型牛车队作为常规跑商的基本单位，单次可运送的货物总量上限约为数吨量级（例如约 4 吨），具体数值在数值设计阶段再行确定。对于人口不大的聚落，一车关键物资即可显著缓解一段时间的紧缺；对于大城或行省中心，则需要多车多轮的持续投入才会产生结构性影响。
- **商誉按车次与时间缓慢积累**：系统按地区与关键物资类别，记录玩家通过车队向该地累计运入的“车次数量”及对应价格水平。每一次围绕当地紧缺物资、且价格接近当地可接受范围的车队交易，都会带来小幅的隐性商誉提升；单次车队的商誉增幅存在上限，即便该批货物在当下极为重要，也不足以让商誉瞬间跃升到极高。
- **连续供给的边际递减**：在较短时间内连续向同一地区运入多车相同物资，整体有利于本地，但单车新增的商誉会逐渐减小：前几车更侧重于“被认识、被记住”，后续车次则主要用于巩固“稳定供给来源”的印象，而非继续快速抬高评价。
- **合理价格优于高溢价**：系统将玩家的要价划分为若干模糊区间：接近当地常规价格的交易，更容易被视为“愿意与本地建立长期合作关系”，按正常幅度增加商誉；略高但不极端的价格，商誉变化有限，仅被视为普通商人；明显高溢价的交易，即便短期解决了燃眉之急，也只会带来很小甚至可以忽略的商誉增长，更接近“利用紧缺局势获取利润”的形象。
- **商誉需要维护而非一次性封顶**：若玩家在某地长期停止供给，或由合理价格转为反复以高溢价出售关键物资，本地对玩家的隐性商誉会在一段时间内逐步回落，从“稳定支撑者”回到普通甚至负面印象。想保持较高商誉，需要在可以接受的价格范围内，持续、间断但长期地向当地提供有意义的支持。

#### 9.6.3 交通难度、背负运输与修路

- **交通形态分层**：根据地区地形与治安状况，运输方式分为三类：有路段可使用小型牛车队进行运输，效率最高、单次运量最大；无路但地形尚可的区域依赖畜背运输，总运量与效率明显低于车队；极端环境下只能采用人背运输，单次可携带的货物极少，但往往用于送达少量关键性的救命物资。
- **难以送达的货物更受重视**：在仅能通过畜背或人背抵达的边远地区，同等重量与种类的货物，相较于经由平整道路与车队送达的情况，将被当地人视为“来之不易”的资源。在以物易物与商誉评估时，此类货物具有更高的实际交换力与心理分量，尤其是粮食、药品、净水工具等基础生存类物资。
- **高风险背负行为对商誉的影响**：在极难进入或充满危险的地区，玩家多次选择以畜背或人背方式，冒着高风险、低效率的代价送达关键物资，即便每次交付的绝对数量不大，也会在本地留下远超普通车队补给的印象。这类行为更容易被口耳相传，形成“知其艰难仍愿前来相助”的叙事，对长期商誉的正向影响显著。
- **修路作为改变地区命运的长期事件**：当玩家与某地区之间已有足够长时间的合作积累，并投入了相当规模的物资与资金后，可触发协助修建道路的长期项目。道路建成后，原先仅能人背或畜背通行的险途被改造为可行车的路线，该地区自此更容易与外界开展贸易与交流。修路行为在当地人的记忆中会被视为改变生活条件的重要节点，对玩家商誉产生深远而持久的提升效果，并显著降低后续商誉随时间回落的速度。
- **自有商队与雇员**：中后期可组建商队（护卫、搬运、代理商人），选择亲自随队或下达自动跑商指令（耗时间、有失败概率）；商誉越高，越容易招募到可靠雇员并获得更好的合作条件。
- **与地牢联动**：部分高价值货仅地牢可得或需地牢材料加工；地牢内可有「商会委托」房间，带出指定战利品换取更高一级的商誉与**游戏内货币**奖励。

### 9.7 以物易物与文明发展程度

- **基本规则**  
  - 在所有行省中，玩家均可选择以物易物的方式与 NPC 进行交易，可与货币交易并行存在。  
  - 系统内部使用抽象的“价值”概念评估双方出价；交易界面内**不向玩家展示明确的价值数值**，仅在**超出持有货币的部分**给予**模糊暗示**（如对方态度、是否犹豫等），具体形式留实现或文案定。NPC 对价值的主观判断仍影响是否同意、是否讨价还价等，并通过对话与交易结果体现玩家是否占优或吃亏。

#### 9.7.1 内部统一价值计算（含货币与物品）

- **单件物品内部价值 `value_i`**  
  - 对任意可参与交易的物品 i，统一以下式计算其**当前场景下的参考价值**：  
    \[
    value_i
    = base\_value_i \times region\_coef \times merchant\_bias \times item\_tag\_coef
    \]
  - 各因子含义：  
    - `base_value_i`：物品表中配置的**基线价值**（见 `items_template_and_style.md`），不对玩家展示；  
    - `region_coef`：地区物价系数，由地区/事件与供需系统给出；**当前最小可实现版本可统一取 1.0** 或按行省粗略区分（如贵 20% 的行省统一 1.2）；  
    - `merchant_bias`：商人整体偏贵/偏便宜系数，取自商人配置字段 `price_bias`，推荐初始范围 **0.9～1.1**；  
    - `item_tag_coef`：按物品 `tags` 与商人 `preferred_tags[]` / `disliked_tags[]` 计算的偏好系数，例如：  
      - 若物品标签集与 `preferred_tags[]` 有交集，则在 1.0 基础上上浮（如 1.1～1.3）；  
      - 若与 `disliked_tags[]` 有交集，则在 1.0 基础上下调（如 0.7～0.9）；  
      - 同时命中多条规则时，按配置表顺序**依次相乘**，最终得到 `item_tag_coef`。  
  - **实现约定**：上述系数全部从配置表读取，不在代码中写死；在未接入供需/事件系统前，`region_coef = 1.0`、`item_tag_coef` 可按简单「喜欢/不喜欢/中立」三档配置基准值。

- **货币类物品的处理**  
  - 货币同样作为普通物品，拥有 `base_value` 与 `tags`；  
  - 最小可实现版本中，直接约定货币的 `item_tag_coef = 1.0`，并通过 `base_value` 体现不同币种的换算关系；  
  - 因为货币本身就是一般等价物，**默认不再额外叠加偏好/厌恶系数**（除非后续在特定 NPC 上做特殊规则）。  

- **一次交易会话内的价值汇总**  
  - 在单次交易界面会话中（见 9.2.1），根据玩家与商人双方的「待给出」与「待拿取」清单，按下式对所有物品求和：  
  - 玩家视角：  
    - 记玩家本次**给出**的所有物品集合为 `Give_player`，**获得**的为 `Get_player`，则：  
      \[
      V\_{\text{player\_give}}
      = \sum\_{i \in Give\_{\text{player}}} value_i \times count_i
      \]
      \[
      V\_{\text{player\_get}}
      = \sum\_{i \in Get\_{\text{player}}} value_i \times count_i
      \]
    - 上式中的 `value_i` 统一使用**商人当前所在场景**下计算出的 `value_i`（即包含 `region_coef` 与商人自身偏好的结果），不再区分「买入价/卖出价」；差价由后续折扣/溢价步骤或技能系数处理。  
  - 商人视角：  
    - 记商人本次**收到**的所有物品集合为 `Give_merchant`（等同于玩家给出），**付出**的为 `Get_merchant`（等同于玩家获得），则：  
      \[
      V\_{\text{merchant\_get}}
      = \sum\_{i \in Give\_{\text{merchant}}} value_i \times count_i
      \]
      \[
      V\_{\text{merchant\_give}}
      = \sum\_{i \in Get\_{\text{merchant}}} value_i \times count_i
      \]
    - 数值上有：`V_merchant_get = V_player_give`、`V_merchant_give = V_player_get`，只是视角不同，后续感受映射使用不同比值。  
  - **数值精度与取整**：  
    - 价值计算过程中允许使用小数，链内全程保留至少两位小数；  
    - 用于资金池结算、商誉折算等时再按对应模块规则取整或四舍五入；  
    - 用于感受词映射时，直接使用浮点比值，不做额外取整。

#### 9.7.2 五档感受词映射（玩家与商人）

- **统一的五档感受词**（与 `trade_ui_layout.md` 保持一致）：  
  - `不行`、`略亏`、`可以`、`略赚`、`很赚`。  
  - 感受词始终以**当前说话者视角**解读（谁给出/拿到更值钱）。  

- **玩家视角比值 `r_player`**  
  - 定义：  
    \[
    r\_{\text{player}}
    = \frac{V\_{\text{player\_get}}}{\max(V\_{\text{player\_give}}, \varepsilon)}
    \]
    其中 \(\varepsilon\) 为极小正数（实现时可取 0.01），用于处理「几乎不给出任何东西就拿到货」的极端情况，避免除以 0。  
  - 区间与感受词映射（**可配置表化，以下为默认推荐值**）：  
    - `r_player < 0.5` → **不行**（明显吃大亏）；  
    - `0.5 ≤ r_player < 0.9` → **略亏**（感觉有点亏，但若有刚需可能仍可接受）；  
    - `0.9 ≤ r_player ≤ 1.1` → **可以**（大致公平，略有误差视为正常讨价还价波动）；  
    - `1.1 < r_player ≤ 1.5` → **略赚**（感觉自己占了一点便宜）；  
    - `r_player > 1.5` → **很赚**（玩家明显占优）。  
  - **特殊情况**：若 `V_player_give` 与 `V_player_get` 同时接近 0（双方几乎没有实质交换），则可直接固定为「可以」，视作无关痛痒的小交换；实现时可通过检查两者是否均 < 某阈值决定是否走此分支。

- **商人视角比值 `r_merchant`**  
  - 对称地从商人角度定义：  
    \[
    r\_{\text{merchant}}
    = \frac{V\_{\text{merchant\_get}}}{\max(V\_{\text{merchant\_give}}, \varepsilon)}
    = \frac{V\_{\text{player\_give}}}{\max(V\_{\text{player\_get}}, \varepsilon)}
    \]
  - 感受词映射沿用与玩家相同的区间，只是解读为「[交易对象] 觉得 XX」：  
    - `r_merchant < 0.5` → **不行**（商人明显吃大亏，极大概率拒绝或强烈犹豫）；  
    - `0.5 ≤ r_merchant < 0.9` → **略亏**；  
    - `0.9 ≤ r_merchant ≤ 1.1` → **可以**；  
    - `1.1 < r_merchant ≤ 1.5` → **略赚**；  
    - `r_merchant > 1.5` → **很赚**。  
  - **实现约定**：在 UI 文案中，玩家侧与商人侧感受词可独立显示，例如「你觉得略赚，[行脚商人] 觉得略亏。」以加强信息不对称感。

- **文明程度噪声与 NPC 认知偏差**  
  - 为体现不同行省文明程度与记账习惯的差异，可在商人视角比值上叠加**噪声项**：  
    \[
    r\_{\text{merchant, noisy}}
    = r\_{\text{merchant}} \times (1 + \delta)
    \]
    其中 \(\delta\) 为按行省与 NPC 个体抽样的随机扰动，例如在「文明程度较低」的行省中取 \(\delta \sim U(-0.1, 0.1)\)，在文明较高地区缩小到 \(\delta \sim U(-0.03, 0.03)\)。  
  - **实现约定**：  
    - 噪声仅影响**商人感受词**与是否接受/犹豫等行为，不回写到任何真实价值或资金池计算；  
    - 噪声的随机种子与刷新时机（每会话一次或每次出价一次）可由实现决定，但同一会话内至少在同一次出价评估阶段应保持一致，避免同一份报价来回切换表现；  
    - 文明程度与噪声幅度的对应关系可在地区配置表中以简单枚举或系数字段实现。  

- **与折扣、税费和技能系数的关系**  
  - 本节定义的 `value_i` 与五档感受词仅反映**物物交换的相对价值**；  
  - 实际的「应付金额」「应收金额」在此基础上再叠加：  
    - `贸易` / `谈判` 等技能带来的折扣或溢价；  
    - 税费、关卡抽成或场景附加费用；  
    - 资金池不足导致的「只付一部分钱，超出的记为商誉」等特殊规则（见 9.2.3）。  
  - **实现约定**：  
    - 五档感受词所用的 `V_player_give` / `V_player_get` **仍以内部统一价值为准，不考虑税费与技能加成**，以保持感受的直观性；  
    - 玩家最终看到的金钱收支与物品流向，按完整结算规则执行，允许出现「感觉自己略赚，但扣了不少税」这类现实感体验。
- **价值波动与隐性损失**  
  - 以物易物的隐性价值损失始终存在：同一批货物在不同 NPC 手中可能换得截然不同的回报。  
  - NPC 对物品价值的判断受到个人经验、喜好、当时处境与对玩家印象影响，导致估值本身具有主观性与不稳定性。  
  - 文明发展程度越低、度量与记账习惯越粗糙的行省，以物易物的隐性损失越难以预估，玩家更难通过直觉判断这笔交易是否“划算”。
- **文明发展程度的影响**  
  - 文明程度较低、货币体系不稳的行省：  
    - 居民对货币价值的共识较弱，更倾向直接以物易物；  
    - 对同类货物的估价差异较大，少数 NPC 可能极度高估或低估某些物品；  
    - 玩家可以通过反复试探与观察，寻找对特定货物异常看重的个体，从而获得远高于常规的回报。
  - 文明程度较高、度量与税制较完善的行省：  
    - 居民对常见货物的价值有相对稳定的共识，物物交换的结果更接近“平均水平”；  
    - 极端占便宜或极端吃亏的情况较少见，但也因此更难通过信息差获得暴利。
- **与当地需求和技术水平的关联**  
  - 若某类物品在当地技术水平下极难自产、且需求量巨大（如粮食、清洁水源、基础医疗药品、保暖材料等），其在以物易物中的“实际交换力”会显著提高。  
  - 在遭遇战乱、灾荒或疫病的行省，这类刚性需求物品更容易被当作“远超名义价值的救命资源”，居民愿意以稀有材料、装备或其他高价值物品来交换。  
  - 玩家可以通过观察当地环境、与 NPC 交谈和多次小额试探，逐渐识别各行省在当前时间段内最紧缺的物品种类，从而为后续大规模跑商与代理跑商提供决策依据。
- **与商人地位/声誉及外观的联动**  
  - 在偏重以物易物的行省中，若玩家外观与言行更接近“可信任的行脚商人”或“带来救济的外来者”，NPC 更愿意给予主动折中或略向玩家倾斜的估价。  
  - 若玩家在本地的隐性商誉较低，或曾被少数 NPC 识别为可能存在欺诈行为，即便携带大量刚需品，也可能遇到：  
    - NPC 明显压价，只愿意进行小额试探性交换；  
    - 需要中间人担保或额外条件（例如预先提供部分货物、履行某种承诺）后才愿意进行大额以物易物。

### 9.8 货币文案约定

  **货币物品**
  - 货币以物品形式存在于背包与仓库中，占用格位与负重；本作没有各地统一的抽象货币（如“银两”“金币”），流通性由商人个体决定（每个商人配置接受的货币短码，见 trader_template.md 与 items/currency_base.csv）。

  **在文档与配置中的写法**
  - 文档中凡涉及金额、支付、奖励、示例数值等场景，应尽量直接使用游戏内已有货币名称与单位（如“创世结晶 ×10”“星琼 ×5”），而不要写死「金币」「银两」等现实感强、且世界观中并不存在的通用货币称呼。
  - 若只需要模糊表达，不希望点名具体货币时，统一使用**「钱」这一不指向任何具体币种的称呼，例如：“这活不太赚钱”“他手头没什么钱”，而不要**写成“没什么银两”“没什么金币”。
  - 旧文案中出现的「银两」「金币」等称呼，在后续整理时应逐步替换为具体货币名（创世结晶、星琼等），或模糊称呼「钱」，视具体语境选择其一。

## 商人 NPC 配置模板（实例用）

> 用途：为每一个具体商人 NPC 建一个独立小段，按下列模板填写。  
> 注意：下面所有字段名均对应设计中的变量与逻辑。**交易场景**（Market / Shop / Exchange / Travel）、**旅行商人在场景内的状态与移动**、**对话打开交易界面**等规则以 `basic.md` 第 9 章（尤 9.2、9.2.1、9.4.1）为准。

---

### 商人：<在此填写商人标识>

#### 1. 基本信息

- **merchant_name**（称呼，UI 显示）  
  - 初始称呼生成规则：  
    - 初始状态：UI 显示 `？？？`；  
    - 当玩家语言不足以完整听懂真实姓名，但能通过货物与简单语句判断其售卖内容时，`merchant_name` 设为当前 `merchant_shilling`（如「卖药材的」「卖工具的」）；  
    - 当玩家语言能力达标、在对话中听懂商人自报姓名或常用称呼后，`merchant_name` 更新为该真实姓名/常用称呼，并不再回退为 `merchant_shilling`。
- **merchant_shilling**（吆喝词根，口语化外号）  
  - 含义：代表该商人最典型的「卖什么」的口语化短语，用于在玩家尚未完全掌握语言时，生成简易称呼。不为空值。
  - 与货品倾向挂钩的示例：  
    - 经营药材为主 → `卖药材的`  
    - 经营工具为主 → `卖工具的`  
    - 经营布匹为主 → `卖布的`  
    - 经营杂货为主 → `卖杂物的`
- **merchant_homeland**（地区归属，隐藏属性）  
  - 示例：某行省 ID / 某文化圈 ID  
  - 说明：暂时只影响穿着与货物风格，具体取值在世界观补完时统一规划。
- **merchant_type**（商人类型）  
  - 取值：摊贩`Vendor` / 店主`ShopOwner` / 旅行商队`Caravan`  
  - **可出现的交易场景**：摊贩仅出现在 **Market**；店主出现在 **Shop**（个别活动时可临时参与 Exchange，由事件控制）；旅行商队可出现在 **Market**（进城时）、**Exchange**（少数主城）、**Travel**（野外/道路场景内相遇时）。交易时的 scene_type 由当前地点及本商人类型共同决定，见 `basic.md` 9.2.1。  
  - 刷新与移动规则（简述即可）：  
    - 若为摊贩：出生在市场区域、出现频率大致如何。  
    - 若为店主：店铺位置、是否每天固定在此。  
    - 若为旅行商队：本商人**不绑定固定地点**。在野外/道路场景中，通过该场景的旅途事件或地图内状态（旅行/扎营，见 basic 9.4.2）出现时，与玩家相遇即进入 Travel 场景交易；进城时可在主城/聚落的 Market 或少数主城的 Exchange 出现，此时 scene_type 由当前地点决定。
- **对话与交易入口**：该商人 NPC 的对话中需提供**「交易」**选项；玩家选择后，根据当前地点与本商人类型确定 scene_type 并打开交易界面，详见 `basic.md` 9.2.1。

---

#### 2. 显性属性（玩家可感知）

- **merchant_attitude**（态度，1～4）  
  - 初始数值：1 / 2 / 3 / 4  
  - 显示格式统一为：  
    - `你觉得对方有点 <态度词>。`
  - 数值与文本对应关系：

    | merchant_attitude 值 | 态度词 | 完整示例文本      |
    | ------------------- | --- | ----------- |
    | 1                   | 冷淡  | 你觉得对方有点 冷淡。 |
    | 2                   | 谨慎  | 你觉得对方有点 谨慎。 |
    | 3                   | 友好  | 你觉得对方有点 友好。 |
    | 4                   | 热情  | 你觉得对方有点 热情。 |

- **merchant_product**（销售商品）  
  - 值为数组，储存物品tag，
  - 主要经营品类：药品 / 原材料 / 工具 / 奢侈品 / 杂货 / 食品  
  - 说明：影响销售的商品。

#### 3. 隐性属性（系统用）

- **hidden_risk_preference**（风险偏好）
  - 取值：保守（`prudent`）/ 中性（`neutral`）/ 激进（`adventurous`）  
  - 简述：代表该商人在交易意愿、价格调整等决策上的性格。在玩家语言能力未达标时，只有风险偏好为激进的商人必然会与玩家进行交易，中性的商人每日可与其对话两次，根据对话动作触发的隐藏事件返回值，可能会与玩家进行交易。保守的商人不会与语言未达标的玩家进行交易。

- **hidden_trading_switch**
  - 取值：`false` / `true`
  - 简述：用于控制当前是否可以与该商人交易，可受事件影响改动该值。

- **hidden_reliability**（可靠度）  
  - 取值：低（`low`） / 中（`medium`） / 高（`high`）  
  - 简述：用以决定商人在挂单、代理时的可靠性，其他用途待补充。

- **hidden_memory**（记性）  
  - 取值：差（`bad`） / 普通（`fine`） / 好（`good`）
  - 简述：表示该商人的记忆力。对于记性越好的角色，玩家触发事件影响消退的所需时间越长。

- **hidden_gossip**（八卦倾向）
  - 取值：消极（`negative`） / 普通（`normal`） / 积极（`positive`）
  - 简述：在生成角色时从三种态度中取其一，用来粗略控制商人对八卦的倾向，决定该商人是否更愿意听取关于玩家的八卦消息和可给出的情报量。越积极者，越愿意告诉玩家更多内容。八卦倾向越消极，玩家在非自身环境中触发的坏事越不容易对该商人造成影响。

#### 5. 资金池相关配置

- **accept_currencies**（接受的货币）  
  - 取值：该商人接受的货币**短码**列表，与 `items/currency_base.csv` 中的 **accept_code** 一致；多个短码用英文逗号分隔。  
  - 示例：`IOS,GC` 表示收原石与创世结晶；`SJW,OD,FLP` 表示收星琼、古老梦华、菲林。  
  - 说明：本作无各地通用货币，流通性由商人个体决定；每个商人可配置只收一种或多种货币。

- **cash_pool_max**（资金池上限）  
  - 初始 `cash_pool`：  
  - 上限说明：这个人一开始大致能拿出多少钱收玩家的货。
- **cash_pool_rd**（资金池恢复间隔）  
  - 单位：天（或其它你决定的时间单位）。  
  - 示例：1 天 / 3 天恢复一次。
- **cash_pool_refill**（单次恢复金额）  
  - 每经过一次 `cash_pool_rd`，为该商人恢复多少资金：  
  - 如：每 1 天恢复 200 单位（**以游戏内已有货币名称为准**，下同；文档中不写死「金币」等称呼）。
- **特殊 NPC 的按物品限额覆盖（可选）**：若本商人被标记为**特殊 NPC**，可为本商人或某物品配置自有的 `session_cap` / `period_cap` / `period_duration`，覆盖场景规则表默认值；详见 `basic.md` 9.2.2。

#### 6. 表类 / 将来扩展（可选）

  #### hidden_risk_preference · 货品倾向偏置表（bias_goods）

> 注意：这里的货品类别是「粗分类」，具体物品在实现时可映射到下列某一类。

- 默认基准：`base_risk = 40`  
- 默认权重：`w_type_default = 0.8`，`w_goods_default = 0.2`  

| 货品倾向大类         | 示例                                   | bias_goods 建议值 |
|----------------------|----------------------------------------|-------------------|
| 粮食 / 基础生活物资 | 粮食、饮水相关物资、基础保暖材料等    | -5                |
| 药品 / 医疗物资     | 药草、成药、急救包、外科耗材等        | -5                |
| 布匹 / 普通纺织品   | 各类布料、普通衣物、床品等            | 0                 |
| 工具 / 生产资料     | 农具、矿镐、木工工具、简易机关部件等  | +5                |
| 奢侈品 / 奇物       | 珠宝、名贵饰品、稀有工艺品、异域奇物  | +10               |

综合计算公式（文字版）：

\[
hidden\_risk\_preference = clamp\big( 40 + 0.8 \times bias\_{type} + 0.2 \times bias\_{goods},\ 0,\ 100 \big)
\]

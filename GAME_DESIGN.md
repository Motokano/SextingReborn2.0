# H5 丝绸朋克冒险游戏 — 完整设计文档

本文档汇总截至目前已确定的全部规则与流程，供后续直接实现（build）时参考。

---

## 一、整体设计哲学

- **世界观**：丝绸朋克（Silkpunk），锦织大陆，机关术 + 丝绸 + 竹骨 + 灵气。
- **0 教学、0 引导**：无弹窗、无任务提示、也不做“藏在世界里的教学”；规则只通过行为与结果体现。
- **高风险、高惩罚**：从一开始就使用完整死亡惩罚，不刻意对早期地牢放宽。
- **战斗非强制**：所有战斗（含最终 Boss）都可绕；进入地牢的首要目标是**生存并撤离**，而非清怪或通关。
- **无“必备物品”**：生产、料理、药品、附魔等均为可选增强，玩家可完全不依赖生产硬打地牢。

---

## 二、区域结构

### 2.1 基地（Home）

- **功能**：休整、存档；永久成长（战斗技能等级、各生产线熟练度、建筑等级）；准备本次冒险（装备、消耗品）；生产活动；与固定 NPC 对话/接任务；投保 NPC（装备快照与 AES-256 代码）。
- **生产活动**（材料 + 配方/图纸 → 产物）：
  - **制造**：一切基础；建筑所需材料与绝大部分工具来自制造。
  - **烹饪**：野外材料 + 贸易配方 → 料理，提供显著 buff。
  - **炼药/炼金**：材料 + 配方 → 战斗中即时使用的药品。
  - **锻造**：材料 + 配方/图纸 → 武器。
  - **纺织**：材料 + 配方 → 丝绸相关产物。
  - **建造/升级生产建筑**：材料主要来自制造。
  - **附魔**：地牢特殊材料 + 附魔建筑 → 装备特殊强化。
- **生产规则**：
  - 有制作时间；可多建筑并行，单建筑同一时刻只能制作 1 件；失败全额扣材料。
  - 熟练度：各线独立、不共用；角色等级 + 建筑等级；仅成功制作涨熟练度；成功/失败与熟练度挂钩；建筑升级可缩短时间、增加产出、降失败率、提品质。
  - 品质六档：粗糙 → 普通 → 精良 → 稀有 → 史诗 → 传说；颜色：灰 → 白 → 绿 → 蓝 → 紫 → 橙。

### 2.2 野外（Field）

- **定位**：低烈度区；移动、采集、与较弱敌人战斗，积累资源，为地牢做准备；经验与掉落明显低于地牢。
- **地图**：主路相对安全（几乎不遇敌）；支路/偏远区敌人与采集点更密集；资源点与城镇/地牢入口为固定格子，每局位置不变。
- **采集点**（采完后按“行动回合数”刷新）：矿脉、药草、木材/竹材、狩猎点、丝绸相关、钓鱼（食材 + 稀有材料/装备）；不强制工具，有工具效率更高。
- **敌人**：不主动远距离追逐；玩家靠近到一定范围后朝玩家移动并可能触发战斗；普通天气下弱于地牢，特定天气/时间下强度可接近低层地牢。
- **天气与时间**：影响资源品质（原有点提升 + 专属高品质点）；影响怪物种类与强度。
- **回程**：角色无便捷回城；需走回城镇/安全出口；城镇 ↔ 基地有需条件解锁的快捷传送。采集不额外增加危险。

### 2.3 城镇（Town）

- **核心**：贸易（装备、材料、配方等）。
- **附加**：有限休整、特殊服务（如鉴定）、与解锁部分传送/设施相关。

### 2.4 地牢（Dungeon）

- **结构**：多个随机生成的小房间组成大迷宫；可配置层数；房间间通过门格切换；进入后无法退回野外，只能找出口撤离或下潜。
- **无通关概念**：收益 = 实际拿到并带出的东西；系统不按层数/时间/击杀给额外结算奖励。
- **战斗**：敌人朝玩家靠近，接近触发战斗；玩家可提前离开房间完全绕过战斗（含 Boss）。

## 三、死亡惩罚与装备快照（投保）

- **死亡时**：身上全部物品消失；战斗技能扣 50 级（不扣到 0 以下）；传送到“地狱”对话后回基地，状态回满；基地仓库与生产存货不受影响。
- **投保**：下地牢前可付费对当前穿戴装备做一次快照；同一时间仅 1 个生效快照。
- **代码**：死亡后 NPC 给出一段 AES-256 加密代码（含玩家信息、地牢标识、快照装备等），玩家不可反推内容。
- **使用代码**：必须在同一地牢同一入口输入；进入后地牢仍完全随机，系统随机指定一房间生成可互动元素；需到达该房间并成功撤离才可从 NPC 取回装备（仅装备，消耗品不恢复）。
- **消耗**：一旦使用代码，不论成败，该玩家此代码即消耗；未到指定房间或死亡则失去机会。
- **共享**：同一代码可被多玩家使用；每人仅一次机会；成功则各自从 NPC 获得那套装备。

## 四、实战经验值

- 战斗胜利获得，线性累计，上限 1 亿；满值 +30% 全局伤害。
- 永不回退；实战经验远高于某敌人时，再打该弱敌不再获得实战经验。

## 五、角色基础属性

本作中角色拥有五项基础属性，这些属性既影响战斗表现，也作用于负重、恢复等系统：

- **筋骨**：影响近战攻击伤害与负重能力，是“硬力量”的直接体现。
- **柔韧**：决定肢体基础防御（动态减伤）；命中部位与伤害类型通过伤害类型微调系数参与最终伤害（详见 5.6）。
- **呼吸**：影响内力上限与体力/内力恢复速度，与生存系技能「呼吸」的效果叠加。
- **身手**：影响闪避、命中率及行动速度，对命中率计算中的“身手值”有直接贡献。
- **专注**：影响从书本中学习技能的效率，以及与精力相关的恢复速度。

以上五项基础属性**均分为先天值与后天值**：

- **先天属性**：在创建角色时决定，每项上限为 40 点，仅有极少数特殊手段可以提升。
- **后天属性**：来自技能等级、装备、buff 等途径的加成全部计入后天部分。

在各类数值计算中（如攻击伤害、负重上限、命中率等），默认使用「先天 + 后天」之和作为该项属性的实际值，不再区分来源；但在系统展示与成长设计上，始终区分先天与后天，便于强调“天赋”与“修行”的差异。

**实现约定（数据与缓存）**：五项基础属性在角色数据中均以**先天、后天两个独立字段**存储；实际值 = 先天 + 后天。当任一来源（装备、技能等级、buff 等）导致先天或后天属性变化时，**实时重算并写入缓存**，保证后续公式与 UI 读取到的为最新值。**统一入口**：装备变化、buff 增删、技能升级等均调用同一套重算入口（如 `recalcCharacterStats()`），在该入口内重算后天属性及**负重上限 \(W_{\max}\)** 等衍生值并写缓存；检查负重时只读缓存。**触发重算的事件清单**（以下事件发生后应调用统一重算入口，避免漏算）：
  - 穿戴 / 脱下 / 替换任意装备；
  - 增加或移除任意 buff（含过期）；
  - 任意技能等级变化（升级或降级）；
  - 更换内功或内功状态变化（若影响属性或衍生值）；
  - 其他会改变先天/后天来源的操作（如使用永久加属性的道具等）。**属性下限**：筋骨、柔韧、呼吸、身手、专注及技能等级等参与公式时，**最小为 0**，负数**视为 0** 再代入计算。

### 5.1 衍生属性（由基础属性计算得出的具体数值）

以下衍生属性均由五项基础属性的**实际值**（先天 + 后天）参与计算，用于战斗、生产与生存的各类公式。公式中的“筋骨”“柔韧”等均指该属性的实际值。

| 衍生属性 | 主要参与属性 | 计算公式约定 | 用途简述 |
|----------|--------------|----------------|----------|
| **负重上限** | 筋骨、力量技能等级 | 基础负重 + 筋骨系数 × 筋骨值 + 力量技能加成；装备、背包、材料等合计重量不可超过此值。 | 携带物品、生产时可持材料量。 |
| **近战伤害系数** | 筋骨 | 近战原始伤害公式中的“筋骨系数”为筋骨值的单调递增函数（如线性或根号缩放），用于武器基础伤害的倍率。 | 普攻与近战招式伤害。 |
| **肢体基础防御与伤害类型微调** | 柔韧 | 柔韧决定基础防御（动态%）；命中部位与伤害类型使用伤害类型微调系数 \(M_{\text{部位,类型}}\)；详见 5.6。 | 减伤链：基础防御后接伤害类型微调。 |
| **肢体损毁值增长** | — | 损毁值增长 = 该部位该次承受的最终伤害（减伤链结算后），向下取整；柔韧通过基础防御间接减少最终伤害从而减少损毁。 | 肢体损毁。 |
| **内力上限** | 呼吸 | 内力上限 = 基础内力 + 呼吸系数 × 呼吸值；呼吸越高，上限越高。 | 技能释放、内力护体等。 |
| **体力/内力恢复速度** | 呼吸 | 每回合或每单位时间恢复量 = 基础恢复 + 呼吸系数 × 呼吸值；与生存系「呼吸」技能效果叠加（乘法或加法在实现时统一）。 | 战斗续航、地牢内恢复。 |
| **身手值（战斗用）** | 身手 | 直接取身手实际值，用于先手判定、连击倍数、命中率公式中的“攻方/防方身手值”。 | 出手顺序、连击、命中率。 |
| **闪避相关** | 身手 | 防方身手参与命中率公式（攻方身手与防方身手的差距决定命中率）；闪避率或闪避难度可在此基础上再叠加装备与技能。 | 被攻击时的命中判定。 |
| **行动速度（战斗）** | 身手 | 战斗中的速度 = 身手值 + 装备词条 + buff；仅影响出手顺序与连击，见「六、战斗核心」。 | 先手、连击判定。 |
| **学习效率（技能书）** | 专注 | 从书本学习技能时，所需时间或消耗与专注值成反比；专注越高，学习越快或消耗越少。 | 技能书使用。 |
| **精力恢复速度** | 专注 | 与精力相关的恢复（如开箱搜索、部分生产）受专注影响；专注越高，精力恢复越快。 | 搜索、部分生产与鉴定。 |

**公式缩放约定**（实现时可采用）：

**实现约定（常数来源）**：筋骨、负重、超重体力、k_筋骨 等公式中的数值常数（如 \(W_{\text{基础}}\)、筋骨/力量系数、\(C_{\text{move}}\)、\(k\)、\(k_{\text{筋骨}}\) 的 10 与 100 等）**全部从配置表读取**，不写死在代码中，便于策划调参。

- **筋骨 → 近战伤害系数**：\(k_{\text{筋骨}} = 1 + (\text{筋骨} - 10) / 100\)（筋骨取实际值），保证低筋骨时仍有基础伤害，高筋骨时明显提升。**实现约定**：徒手与兵器伤害公式中，\(k_{\text{筋骨}}\) 在取整前**最后乘以**，即先算完其余乘区再乘 \(k_{\text{筋骨}}\)，最后向下取整得到最终扣血与损毁增加值。**仅徒手与兵器**参与该乘区；非徒手、非兵器攻击（如纯气劲、机关、法术等）**不乘** \(k_{\text{筋骨}}\)；若后续扩展需让某类攻击参与，再单独说明。
- **柔韧 → 基础防御**：见 5.6.2 动态公式（\(DR = S_{\text{柔}}/(S_{\text{柔}}+3D)\)）。
- **呼吸 → 内力上限**：\( \text{内力上限} = \text{基础值} + \alpha \times \text{呼吸} \)，\(\alpha\) 为配置常数。
- **身手 → 命中率**：已在「七、命中、招架与伤害流程」中约定“身手差距 → 命中率”；攻方身手与防方身手的差值或比值参与曲线，边际递减。

### 5.4 后天属性成长来源简述

后天属性的增加**不改变**先天上限（每项先天仍以 40 为上限），但实际值 = 先天 + 后天可超过 40。主要来源：

- **战斗技能等级**：如「基本拳脚」每 200 级 +1 后天筋骨、+1 后天柔韧；其他战斗技能可在各自条目中定义类似永久加成。
- **装备与词条**：装备提供的“+筋骨”“+身手”等一律计入后天。
- **Buff 与状态**：临时增益（料理、药品、机关术效果等）计入后天，过期即移除。
- **特殊剧情/道具**：极少数永久增加后天属性的道具或剧情，在剧情与数值设计中单独列出。

实现时，所有来源的数值先汇总为“后天该项”，再与先天相加得到**实际值**，用于 5.1、5.2、5.3 的公式与表。

### 5.5 筋骨属性细则（徒手、兵器与负重）

#### 5.5.1 初始分配与成长区间

- **创建角色时的先天分配**
  - 五项基础属性的**基础先天值**均为 10 点。
  - 角色创建时额外获得 **50 点自由点数**，可在「筋骨 / 柔韧 / 呼吸 / 身手 / 专注」之间任意分配。
  - 单项先天属性在创建阶段的上限为 **29 点**（含基础 10 点），即任一项在捏人时不能被加到 30 点及以上。
  - 全局规则“先天属性上限 40 点”仍然有效，意味着剩余空间（29～40）仅能通过**极少数特殊手段**提升。

- **筋骨典型数值区间（实际值 = 先天 + 后天）**
  - **新手期**：完成捏人并获得少量后天加成后，筋骨实际值通常落在 **20～35** 区间。
  - **中期**：围绕近战相关战斗技能进行长期修行，并配合常规装备与 buff 后，筋骨实际值可稳定提升至 **数百等级（约 300～400）**。
  - **后期与长线**：随着版本更新与高等级战斗技能、稀有装备的加入，筋骨实际值预期可达到 **2000～3000** 甚至更高；所有与筋骨相关的公式均按“高值可用，但收益边际递减”的思路设计。

#### 5.5.2 筋骨与徒手战斗

- **臂力总值 = 筋骨实际值**
  - 在徒手战斗中，角色的“臂力总值”直接等于**筋骨实际值**（先天筋骨 + 后天筋骨）。
  - 使用徒手战斗技能（包括徒手普攻）时，**臂力总值作为本次攻击的基础威力值**参与计算。

- **徒手伤害公式示意**
  - 记 \(S_{\text{筋骨}}\) 为筋骨实际值，则单次徒手攻击的伤害可表示为：
    \[
    D_{\text{徒手}}
    = \underbrace{S_{\text{筋骨}}}_{\text{基础威力值}}
    \times T_{\text{手套词条}}
    \times F_{\text{技能威力}}
    \times F_{\text{内功威力}}
    \times F_{\text{出力大小}}
    \times F_{\text{环境修正}}
    \times F_{\text{实战经验}}
    \]
  - 上式中，各乘区的含义为：
    - \(T_{\text{手套词条}}\)：徒手装备（手套等）提供的伤害修正。**实现约定**：**每手一个手套槽**（左手、右手各自独立）；徒手攻击时**只看当前出招手**所装备手套的词条系数，该手无手套则为 1.0。手套类装备在设定时于配置中**单独标注**类型或标签（如槽位 ID、装备类型）以便代码识别。
    - \(F_{\text{技能威力}}\)：招式或技能本身的威力系数；
    - \(F_{\text{内功威力}}\)：当前内功对招式威力的修正；
    - \(F_{\text{出力大小}}\)：玩家选择的出力程度（如 3 成、7 成、全力）；
    - \(F_{\text{环境修正}}\)：环境因素（地形、天气等）带来的加成或减益；
    - \(F_{\text{实战经验}}\)：由「四、实战经验值」给出的全局增伤。

#### 5.5.3 筋骨与兵器战斗

筋骨对兵器战斗的影响分为两层：**使用门槛/惩罚（只看先天筋骨）**与**额外增伤（看筋骨实际值）**。

- **（一）兵器使用门槛与惩罚——只看先天筋骨**
  - 每把兵器定义一个**先天筋骨要求**，记为 \(Req_{\text{筋骨}}\)。
  - **实现约定**：兵器/武器配置表中，该数值的字段名为 `req_innate_jingu`（**先天筋骨要求**）；未配置或为空时取默认值 20；配置为 0 或负数时**统一视为 0**（无门槛）。
  - 仅使用角色的**先天筋骨**与 \(Req_{\text{筋骨}}\) 比较，用于判定“能不能用”“是否吃惩罚”；后天筋骨不参与这一比较。
  - 记 \(S_{\text{先天}}\) 为角色的先天筋骨：
    - **无法使用**：若
      \[
      S_{\text{先天}} < \tfrac{1}{2} Req_{\text{筋骨}}
      \]
      则角色**不能装备/挥动**该兵器。
    - **惩罚区间（可用但吃削弱）**：若
      \[
      \tfrac{1}{2} Req_{\text{筋骨}} \le S_{\text{先天}} < Req_{\text{筋骨}}
      \]
      兵器可以使用，但进入惩罚区；此时伤害乘以一个线性插值的门槛修正 \(M_{\text{门槛}}\)：
      \[
      t = \frac{S_{\text{先天}} - Req_{\text{筋骨}}/2}{Req_{\text{筋骨}}/2} \in [0,1]
      \]
      \[
      M_{\text{门槛}} = 0.5 + 0.5 \times t
      \]
      - 当 \(S_{\text{先天}} = \tfrac{1}{2} Req_{\text{筋骨}}\) 时，\(M_{\text{门槛}} = 0.5\)，即仅造成 50% 伤害；
      - 当 \(S_{\text{先天}} = Req_{\text{筋骨}}\) 时，\(M_{\text{门槛}} = 1.0\)，恰好不加不减。
    - **达标与以上（无惩罚）**：若
      \[
      S_{\text{先天}} \ge Req_{\text{筋骨}}
      \]
      则 \(M_{\text{门槛}} = 1.0\)，完全不吃先天筋骨带来的惩罚。

- **（二）额外增伤——仅看先天筋骨**
  - 在门槛层面判定完之后，**仅**角色的**先天筋骨**参与额外增伤，记为 \(M_{\text{筋骨}}\)：
    \[
    M_{\text{筋骨}}
    = 1 + 0.05 \times \max(S_{\text{先天}} - Req_{\text{筋骨}},\ 0)
    \]
    即每比兵器要求多 1 点先天筋骨，增伤 +5%；**无上限**。0.05 从配置表读取。
  - 最终兵器攻击的筋骨相关乘区为：
    \[
    M_{\text{兵器-筋骨}} = M_{\text{门槛}}(S_{\text{先天}}, Req_{\text{筋骨}}) \times M_{\text{筋骨}}
    \]

- **（三）兵器伤害公式示意**
  - 记 \(B_{\text{兵器基础威力}}\) 为兵器本身的基础威力，则兵器战斗技能的伤害示意为：
    \[
    D_{\text{兵器}}
    = B_{\text{兵器基础威力}}
    \times T_{\text{兵器词条}}
    \times F_{\text{技能威力}}
    \times F_{\text{内功威力}}
    \times M_{\text{兵器-筋骨}}
    \times F_{\text{出力大小}}
    \times F_{\text{环境修正}}
    \times F_{\text{实战经验}}
    \]

#### 5.5.4 筋骨与负重

- **当前负重 W（实现约定）**：装备与背包**共用同一重量池**；\(W\) = 身上装备重量 + 背包内物品重量，与 \(W_{\max}\) 比较。物品重量单位与 \(W_{\max}\) 一致（kg）。**数值精度**：\(W\) 与 \(W_{\max}\) 全程使用浮点数计算与比较，超重判定（\(W > W_{\max}\)、\(W > 1.5 W_{\max}\)）及移动体力消耗公式中的 \(W/W_{\max}\) 均用浮点运算。**\(W_{\max}\) 缓存**：\(W_{\max}\) 在统一重算入口（见上文「数据与缓存」）内与属性一并计算并写入缓存，检查负重时只读缓存。**展示取整**：向玩家展示当前负重 \(W\) 与负重上限 \(W_{\max}\) 时，**保留一位小数**（计算与判定仍用浮点）。**力量满级武器不计负重**：当生存系技能「力量」（`survival_strength`）达到 **100 级**时，计算当前负重 \(W\) 时**不计入左手、右手武器槽中已装备的武器**重量；扣掉后若仍超重，超重判定与移动体力消耗仍按该扣掉后的 \(W\) 计算。
- **负重上限公式**
  - 记 \(W_{\max}\) 为角色当前负重上限，\(S_{\text{筋骨}}\) 为筋骨实际值，\(L_{\text{力量}}\) 为生存系技能「力量」的等级（技能 ID：`survival_strength`；未习得时视为 0），则：
    \[
    W_{\max}
    = W_{\text{基础}}
    \times \bigl(1 + 0.01 \times L_{\text{力量}} + 0.0005 \times S_{\text{筋骨}}\bigr)
    \]
    其中 **\(W_{\text{基础}} = 25\)**（kg），力量每级 +1% 总负重，筋骨每点 +0.05% 总负重。\(W_{\text{基础}}\)、0.01、0.0005 均从**配置表**读取。

- **超重惩罚规则**
  - 记当前实际负重为 \(W\)：
    - **未超重**：若 \(W \le W_{\max}\)，移动与战斗不受负重惩罚，移动体力消耗为基础值 \(C_{\text{move}}\)。
    - **轻度～中度超重（可移动，但越走越累）**：若
      \[
      W_{\max} < W \le 1.5 \times W_{\max}
      \]
      角色仍可移动，但每移动一格的体力消耗随超重比例线性增加，例如：
      \[
      C_{\text{move, 实际}}
      = C_{\text{move}} + k \cdot \bigl(\tfrac{W}{W_{\max}} - 1\bigr)
      \]
      其中 \(C_{\text{move}} = 1\)（未超重时每格 1 体力），\(k\) 从配置表读取，**当前约定为 20**（刚好超重时每格 1 体力，每超 10% 加 2 体力）。目前设计中，超重仅通过**体力消耗加剧**体现，不强制削减命中/闪避等战斗属性。
    - **严重超重（无法移动）**：若
      \[
      W > 1.5 \times W_{\max}
      \]
      角色**无法移动**，只能通过丢弃物品或使用特殊手段降低负重，直至负重不超过 \(1.5 \times W_{\max}\)。
  - **实现约定**：在**每次移动前**与**打开背包前**检查当前负重 \(W\) 与 \(W_{\max}\)，据此判定是否超重、是否严重超重。严重超重时**仅禁止移动**，不禁止使用物品、换装、战斗等其他操作。

### 5.6 柔韧属性细则（基础防御、伤害类型微调与肢体损毁）

#### 5.6.1 定位与分工

- **数值尺度**：柔韧的实际值（先天 + 后天）与筋骨保持同一量级：
  - 新手期：约 20～35；
  - 中期：约 300～400；
  - 长线后期：可达到 2000～3000 乃至更高。
- **主职**：柔韧决定角色的**肢体基础防御**，即对「内功、防具减伤后」的剩余伤害再做一次**动态百分比减伤**（见 5.6.2）；不按部位区分，全身共用同一柔韧值。
- **伤害类型微调**：各身体部位（头、胸、腹、左手、右手、左脚、右脚）对**劈砍、戳刺、钝击**三种伤害类型有**伤害类型微调系数** \(M_{\text{部位,类型}}\)，在基础防御之后作为最终伤害的乘数（见 5.6.2 表格）。
- **损毁**：肢体损毁值增长 = 该部位该次承受的**最终伤害**（减伤链结算后），**向下取整**；柔韧仅通过提高基础防御间接降低最终伤害，从而减少损毁增长，不再单独设柔韧损毁减缓系数。

#### 5.6.2 基础防御与伤害类型微调系数

- **基础防御（动态减伤）**：柔韧实际值直接作为防御值，参考 PoE 护甲公式，对经过内功→防具减伤后的**剩余伤害** \(D_{\text{进}}\) 做动态百分比减伤：
    \[
  DR_{\text{基础防御}}
    = \frac{S_{\text{柔韧}}}{S_{\text{柔韧}} + 3 \times D_{\text{进}}}
    \]
    \[
    D_{\text{出}} = D_{\text{进}} \times \left(1 - DR_{\text{基础防御}}\right)
    \]
    其中 \(S_{\text{柔韧}}\) 为柔韧实际值（先天 + 后天），\(D_{\text{进}}\) 为当前减伤步骤的输入伤害，常数 **3** 为设计锚点（可在数值阶段微调）。
  - **边界**：计算时若 \(S_{\text{柔韧}} \le 0\)，视为 0，即该步无减伤（\(DR_{\text{基础防御}} = 0\)，\(D_{\text{出}} = D_{\text{进}}\)）。
  - **数值参考**：

    | 柔韧 | 进入伤害 | DR | 输出伤害 |
    |------|---------|-----|---------|
    | 25（新手） | 20 | 29% | ≈ 14 |
    | 100 | 50 | 40% | ≈ 30 |
    | 400（中期） | 100 | 57% | ≈ 43 |
    | 2000（后期） | 200 | 77% | ≈ 46 |

  - 该公式具有**动态缩放**特性：面对小伤害时减伤比例较高，面对大伤害时减伤比例相应下降，不存在固定硬减伤上限。
  - **完整减伤链顺序**：内功（%）→ 防具（%）→ 基础防御（柔韧动态%，\(\frac{S_{\text{柔}}}{S_{\text{柔}}+3D}\)）→ **伤害类型微调系数**（\(M_{\text{部位,类型}}\)）。
  - **取整规则**：减伤链**各步之间**保留小数传递（具体小数位数在实现时确定，如一位或两位）；仅在得到**最终扣血**与**损毁增加值**时，对该值**向下取整到非负整数**再写入角色/肢体。
  - **适用范围**：当前减伤链与伤害类型微调**仅针对劈砍、戳刺、钝击**三种物理伤害类型；毒、火、真实伤害等其他类型是否参与基础防御或微调系数，在扩展时另行约定。
- **伤害类型微调系数 \(M_{\text{部位, 类型}}\)**：在基础防御之后应用。单次攻击仅带一种伤害类型（劈砍 / 戳刺 / 钝击之一）；命中某部位后，该次**最终伤害**（用于扣血与损毁） = 基础防御步骤输出 × \(M_{\text{部位, 类型}}\)，链内保留小数，仅在作为最终扣血与损毁增加值时向下取整到整数。\(M > 1\) 表示该部位对该类型更脆弱（承伤更高），\(M < 1\) 表示相对更耐。

- **推荐的 \(M_{\text{部位, 类型}}\) 表**（可在数值阶段细调）：

  | 部位 \\ 伤害类型 | 劈砍 | 戳刺 | 钝击 | 说明 |
  |----------------|------|------|------|------|
  | **头** | 1.0 | 1.1 | 1.3 | 头部对钝击最脆，戳刺略高，劈砍视为基准。 |
  | **胸** | 1.0 | 1.2 | 1.1 | 肋骨 + 内脏，对戳刺尤其危险，对钝击也略脆。 |
  | **腹** | 1.1 | 1.3 | 1.0 | 腹部软组织与空腔器官多，对劈砍/戳刺更脆，钝击中等。 |
  | **左手/右手** | 1.0 | 1.0 | 1.1 | 上肢骨骼表浅，钝击更易出现骨折，其他按基准。 |
  | **左脚/右脚** | 0.9 | 1.0 | 1.1 | 下肢肌肉较厚，对劈砍略有“肉垫”，钝击同样易骨折。 |

  - \(M_{\text{部位,类型}}\) **不做数值范围限制**，配置/策划表中填写的值直接参与计算。

#### 5.6.3 肢体损毁值增长

- **损毁值增长** = 该部位该次承受的**最终伤害**（即减伤链「内功 → 防具 → 基础防御 → 伤害类型微调系数」结算后的数值，链内保留小数），对该值**向下取整到非负整数**作为本次损毁增加值。柔韧不单独参与损毁公式，仅通过基础防御间接降低最终伤害，从而减少损毁增长。
- 骨折判定、损毁值上限、手术恢复等仍主要由肢体系统、生存系「健康」「手术」等定义。

#### 5.6.4 与生存系「健康」等的协同

- 生存系「健康」继续负责降低骨折概率、减少日常饱食/饮水消耗等；柔韧不取代健康在骨折概率上的作用。

## 六、战斗核心

- **节奏**：风来的西林式回合刻制——玩家每行动一次，所有敌人各行动一次。
- **方向与范围**：8 向移动与攻击；普攻为正前方 1 格单体。
- **耗回合**：移动一格 / 普攻 / 使用技能 / 使用道具 / 等待；不耗回合动作后续定义。
- **气劲**：技能消耗气劲；每回合自然恢复，可药品与特定行为恢复。
- **出力大小（1～12 成）**：伤害与消耗均与成数挂钩（\(F_{\text{出力}} = \text{成数}/10\)，实际消耗 = 基础消耗 × 成数/10）。**实现约定**：各攻击技能的出力相关配置（如默认成数、气劲/体力基础消耗）在**技能配置**中按技能/招式配置；玩家**打开技能栏并点击某攻击技能时**，可在 UI 中调整该技能的出力大小（1～12 成），该设定在发动该技能时生效。
- **速度与先手、连击**
  - **速度值**：战斗中的速度 = 身手值 + 装备词条调整 + buff。速度**仅影响**战斗中的出手顺序与连击，不影响每回合可移动的格数。
  - **先手判定**：每次主动攻击时，先比较双方速度，速度高者先出手。双方速度相同时，视为同时触发，同时开始计算减伤并结算伤害。
  - **连击**：若一方速度**大于等于**另一方两倍，则速度高者产生连击，本回合内连打两次；大于等于三倍则连打三次；大于等于四倍则连打四次（更高倍数可后续细化）。连击均视为**同一回合内**产生的动作；速度低的一方在对方连击全部打完后再还手一次，本回合仅此一次。
  - **连击判定与消耗**：连击的每一击均**单独**做命中、招架与伤害判定；每一击**单独扣除**一次气劲（及体力，若有），默认按正常消耗。可通过某些手段减免因连击产生的额外内力消耗（该接口为后续扩展预留）。

## 七、命中、招架与伤害流程（普攻）

1. **命中率**：先算双方“身手值”差距。身手接近 → 命中率约 80–85%；攻方身手远高 → 命中率上升但边际递减；攻方身手低 → 差距越大命中率滑坡下降。
2. **招架**：命中后按防方招架几率（招架技能 + 装备）判定。成功则本次不造成伤害；失败则按招架技能卸力比例减伤。
3. **减伤链**（卸力之后依次）：内功减伤 → 防具减伤 → 基础防御（柔韧动态%）→ 伤害类型微调系数（按命中部位与伤害类型）；链内保留小数传递，仅最终扣血与损毁增加值向下取整到整数。详见 5.6。
   - **内功**：仅在有“内力护体”时生效；百分比；被减部分记到护体盾上。
   - **防具**：百分比。
   - **基础防御**：由柔韧决定，动态公式 \(DR = S_{\text{柔}}/(S_{\text{柔}}+3D)\)。
   - **伤害类型微调**：命中部位与伤害类型查表 \(M_{\text{部位,类型}}\)，最终伤害 = 上步输出 × \(M\)。
4. **原始伤害**（减伤前）：武器基础伤害 ×（招式+内力威力系数）× 实战经验增伤 × 环境增伤 × 暴击系数 × 内力催发系数（1～12 成，仅多耗气劲）。暴击按暴击率属性判定，成功则乘固定系数。

## 八、身体部位与状态效果

- **部位**：头、胸、腹、左手、右手、左脚、右脚；命中部位按权重随机（躯干高、头低、四肢中等）。
- **效果**：
  - **头**：本回合眩晕无法行动；会自己恢复。
  - **胸**：只算伤害。
  - **腹**：一次性降低饱食度与饮水度。
  - **手**：命中/招架下降会自己恢复；有几率骨折，骨折与脚同，须特殊手段解除。
  - **脚**：暂时无法移动会自己恢复；有几率骨折 → 脚部技能失效、移动一格需两回合，须特殊手段解除。
- **肢体损毁**：每个肢体有**损毁值**；损毁值满后该肢体标记为损毁、失去功能，消耗品无法降低；可通过**手术用品**暂时扣除部分损毁值上限以恢复功能，**手术**技能等级影响可恢复的损毁值上限比例。
- **各部位损毁值上限**：

  | 部位 | 损毁值上限 | 说明 |
  |------|-----------|------|
  | **头** | 50 | 头部耐受度最低，少量损毁积累即可导致功能丧失。 |
  | **胸** | 100 | 标准上限，有肌肉与肋骨支撑，较能承受累积伤害。 |
  | **腹** | 80 | 软组织与空腔器官集中，耐受度低于胸部。 |
  | **左手/右手** | 100 | 标准上限；骨折等会更早触发，但损毁积累速度较快。 |
  | **左脚/右脚** | 100 | 标准上限；骨折与行动失能早于损毁满值触发。 |

## 九、技能系统

技能分为四大类型：**生存 / 生活 / 战斗 / 特殊**。

**实现约定（技能 ID）**：本作技能在代码与配置中通过**技能 ID** 引用；以下为已确定 ID，后续技能在细化时陆续补充。负重公式等引用「力量」等级时，使用 ID `survival_strength`；角色未习得该技能时，等级视为 0。

| 技能（显示名） | 技能 ID |
|----------------|---------|
| 力量 | `survival_strength` |

### 8.1 生存系（已定义）

| 技能 | 说明 |
|------|------|
| **语言** | 初始听不懂 NPC、看不懂物品描述；等级提升后逐渐能看懂、听懂更多内容。 |
| **耐力** | 生产活动消耗体力；等级越高，单次生产消耗体力越少。 |
| **健康** | 降低骨折概率；减少日常饱食度、饮水度消耗。 |
| **免疫** | 降低所有药物副作用；提高治愈速度；延长止痛药持续时间。 |
| **代谢** | 增强料理类物品效果。 |
| **力量** | 提升负重能力；满级 100 级时，左手、右手武器槽中已装备的武器不计入角色负重（见 5.5.4）。 |
| **活力** | 降低身体某部位受伤时触发出血的概率。 |
| **抗压** | 手部受伤时，减少命中率、招架率下降的幅度。 |
| **专注** | 地牢开箱为逐步搜索（物品信息一件件出现），完全辨认即视为搜索完毕；通过搜索未搜索过的箱子升级；等级越高搜索所需时间越短。 |
| **手术** | 肢体损毁值满后可用手术用品暂时降低损毁值上限并恢复功能；技能等级影响可恢复的损毁值上限比例。 |
| **睡眠** | 靠睡觉升级；等级越高，恢复体力所需时间越少。 |
| **呼吸** | 等级越高，每回合恢复的体力、内力越多。 |
| **耐候** | 减少因极端气候产生的体温变化幅度，减轻低温（及未来可能加入的高温）带来的额外消耗。 |

#### 8.2.1 生活技能熟练度规则

- 所有生活技能采用**“用什么升什么”**的熟练度系统，使用 50000 次视为满熟练度（100%）。
- 熟练度带来的效果一律按**乘法系数**作用在各自的基础几率上：
  - 实际几率 = 基础几率 × (1 + 熟练度对应的额外比例)
  - 例如基础高品质几率为 10%，额外 +30% 时，最终几率 = 10% × (1 + 0.3) = 13%。
- **熟练度上升速度**：某些手段可提升或降低熟练度上涨速度；该接口需为后续扩展预留（实现时预留可插拔的倍率/修正入口，不在此文档枚举具体手段）。

- **钓鱼熟练度**：每 +1% 熟练度，
  - 钓鱼成功率：基础成功率 × (1 + 0.3%)；
  - 提升品质几率：基础提升品质几率 × (1 + 1%)；
  - 稀有鱼出现几率：基础稀有几率 × (1 + 0.15%)。

- **挖矿熟练度**：每 +1% 熟练度，
  - 挖矿成功率：基础成功率 × (1 + 0.3%)；
  - 挖到高品质矿石几率：基础高品质几率 × (1 + 1%)；
  - 稀有矿/副产品几率：基础稀有几率 × (1 + 0.15%)。

- **伐木熟练度**：每 +1% 熟练度，
  - 砍伐成功率：基础成功率 × (1 + 0.3%)；
  - 高品质木材几率：基础高品质几率 × (1 + 1%)；
  - 稀有木材/副产品几率：基础稀有几率 × (1 + 0.15%)。

- **狩猎熟练度**：每 +1% 熟练度，
  - 狩猎成功率：基础成功率 × (1 + 0.3%)；
  - 高品质猎物/素材几率：基础高品质几率 × (1 + 1%)；
  - 稀有猎物/稀有掉落几率：基础稀有几率 × (1 + 0.15%)。

- **采集熟练度**：每 +1% 熟练度，
  - 采集成功率：基础成功率 × (1 + 0.3%)；
  - 高品质草药/果实几率：基础高品质几率 × (1 + 1%)；
  - 稀有植物/稀有杂物几率：基础稀有几率 × (1 + 0.15%)。

- **畜牧熟练度**：每 +1% 熟练度，
  - 畜产品产出成功率：基础成功率 × (1 + 0.3%)；
  - 畜产品高品质几率：基础高品质几率 × (1 + 1%)；
  - 不额外提高稀有动物/稀有产物几率（畜牧偏稳定量产）。

- **种植熟练度**：
  - 仅在**收获作物时**提升熟练度（种下不涨，仅收获涨）。
  - 每 +1% 熟练度：
    - 高品质作物几率：基础高品质几率 × (1 + 1%)；
    - 双倍产物几率：基础双倍几率 × (1 + 1%)。

### 8.3 战斗系

战斗技能分为五类，每条具体技能后续设计时再细化：

| 类别 | 说明（待细化） |
|------|----------------|
| **徒手** | 徒手战斗技能（拳脚、指法等）。 |
| **兵器** | 兵器战斗技能（刀、剑、棍等）。 |
| **轻功** | 身法、移动与闪避相关。 |
| **内功** | 内力运用、护体、催发等。 |
| **招架** | 格挡、卸力、招架判定相关。 |

#### 8.3.1 战斗技能通用规则

- **技能等级与潜能消耗**
  - 所有战斗技能的等级上限为 **1000 级**，使用**潜能**进行升级。
  - 单一技能从 1 级升至 1000 级，累计约需 **1500 万潜能**。
  - 从 **400 级开始**，每提升 1 级所需的潜能成本显著上升，体现后期修行的瓶颈感。

- **招式与熟练度**
  - 每个战斗技能包含若干个**招式**，每个招式有独立的**熟练度计数**。
  - 通用约定：同一招式**命中目标**时，其“使用次数” +1；打空、被闪避或被招架不计入。
  - 当某招式为多段攻击时，只要本次释放中**至少有一段命中**，该招式的使用次数仍只 +1。
  - 招式熟练度上限统一为 **50000 次使用 = 100% 熟练度**。
  - 熟练度只作为“在当前技能等级基础上的额外加成”，不会改变技能等级本身。
  - **熟练度上升速度**：与生活技能一致，某些手段可提升或降低招式熟练度上涨速度；该接口需为后续扩展预留（实现时预留可插拔的倍率/修正入口，不在此文档枚举具体手段）。

- **威力组成（等级 + 熟练度）**
  - 每个战斗技能都有一条独立的“等级 → 基础威力系数”曲线，用于定义该技能在不同等级下的**基础威力**。
  - 每个招式在计算伤害时，先取所属技能的基础威力系数，再乘以一个**线性的熟练度加成**：
    - 记 \(L\) 为技能等级，\(P\) 为该招式的使用次数（0～50000），\(R = \min(P / 50000, 1)\) 为熟练度比例。
    - 记 \(Base(L)\) 为等级 \(L\) 对应的基础威力系数，则单个招式的最终威力系数为：
      \[
      Final(L, P) = Base(L) \times (1 + R)
      \]
    - 当熟练度达到 100% 时，招式威力在等级基础上获得 **+100%（×2）** 的线性加成。
  - 各战斗技能的具体 \(Base(L)\) 曲线可不同，在对应技能条目中单独给出；本节仅约定通用结构。

- **装备到肢体与出招循环**
  - 战斗技能必须**装备在肢体上**才能在战斗中使用，当前可装备的肢体包括：左手、右手、左脚、右脚。
  - 每条肢体同时可装备 **1 个主动战斗技能 + 1 个招架技能**。
  - 每个战斗技能拥有若干个**招式槽**；玩家从已解锁的招式中选择装入槽位并设定顺序。
  - 每条装备了主动技能的肢体，都会维护一条**独立的招式循环序列**（只在该肢已装备的招式中循环）。
  - 玩家可以为四个肢体设定**出招优先顺序**（如：左手 → 右手 → 左脚 → 右脚）；在一次攻击中：
    - 系统在“未损毁且当前有可用招式”的肢体中，按优先顺序选择一条肢体作为本次出招肢体。
    - 若该肢体已损毁，直接跳到下一条肢体；若所有相关肢体均不能出招，则本次攻击视为无效。
  - 当某肢体被选中出招时：
    - 按该肢体的招式循环，取出**当前轮到的招式**尝试释放。
    - 如果因为动作标签不符、肢体状态异常等原因导致招式**无法释放**，本回合动作白费，但该肢体的招式指针仍视为“已轮到过一次”，在下次轮到该肢体时会进入下一招。

- **动作标签与肢体状态**
  - 每个招式都要求装备肢体具备某些**动作标签**（例如“拳击”“勾击”“踢击”等）。
  - 若肢体不具备招式所需的动作标签，则该招式既不能装入该肢的槽位，也无法在战斗中释放。
  - 肢体的动作标签可以被一些系统实时增减（例如通过特殊装备、buff、或永久改造等方式）；标签变化**即时生效**。
  - 若玩家在某肢上预先装备了需要特定标签的招式，而战斗中该肢失去了对应标签，则那些招式视为暂时不可用。
  - 肢体一旦被判定为“损毁”，则视为该肢无法再使用任何主动或招架招式，直到通过特殊手段恢复。

#### 8.3.2 「基本拳脚」示例

作为徒手类战斗技能的代表，「基本拳脚」使用上述通用规则，并在此基础上补充以下具体设定：

- **装备与作用肢体**
  - 「基本拳脚」可以装备在左手、右手、左脚、右脚任意肢体上，只要该肢体未损毁且具备至少一个所属招式所需的动作标签。
  - 一个角色可以在多条肢体上同时装备「基本拳脚」，各自维护独立的招式循环。

- **招式与解锁等级**
  - 「基本拳脚」下包含四个基础招式：
    - **直拳**：技能等级 **1 级** 解锁。
    - **前踢**：技能等级 **1 级** 解锁。
    - **勾拳**：技能等级 **50 级** 解锁。
    - **鞭腿**：技能等级 **50 级** 解锁。
  - 各招式的具体数值定位（轻攻/重攻/控制等）、气劲消耗与命中部位权重将在后续细化。

- **招式槽与初始配置**
  - 角色在学会「基本拳脚」时，默认已掌握直拳与前踢，对应**立即解锁 2 个招式槽**。
  - 当技能等级达到 50 级并学会勾拳与鞭腿时，每学会一个新招式**额外解锁 1 个招式槽**，最终该技能共计 **4 个槽位**。
  - UI 中仅显示当前已解锁的槽位，未解锁的槽位不显示，避免在早期暴露未来结构。
  - 玩家可以在每个槽内自由选择已解锁的招式，并为装备了「基本拳脚」的每条肢体分别设定槽内招式顺序；不同肢体可以有完全不同的出招顺序。

- **等级与基础威力曲线**
  - 「基本拳脚」的基础威力系数以 1 级为基准，记为 \(Base(1)=10\)，并随技能等级 \(L\) 提升；1000 级时约为 1 级的 10 倍，即 \(Base(1000)\approx 100\)。
  - 为体现“前期提升明显、后期逐渐变缓”，定义一个相对 1 级的倍数函数 \(M(L)\)（1 级时 \(M(1)=1\)），分三段线性插值：
    - **1 ≤ L ≤ 200**：从 1 倍平滑提升到约 4 倍：
      \[
      M(L) = 1 + \frac{3}{199}\,(L - 1)
      \]
    - **200 < L ≤ 400**：从约 4 倍提升到约 7 倍：
      \[
      M(L) = 4 + \frac{3}{200}\,(L - 200)
      \]
    - **400 < L ≤ 1000**：从约 7 倍提升到约 10 倍：
      \[
      M(L) = 7 + \frac{3}{600}\,(L - 400)
      \]
  - 对应的基础威力系数为：
    \[
    Base(L) = 10 \times M(L)
    \]
  - 在原始伤害公式中，「基本拳脚」各招式的“招式威力系数”部分即取自该 \(Base(L)\) 再结合熟练度加成。

- **熟练度与最终威力**
  - 「基本拳脚」下的每个招式都拥有独立的熟练度计数：
    - 每当该招式成功命中目标时，其“使用次数” +1。
    - 打空、被闪避或被招架不增加使用次数。
    - 若为多段攻击，只要本次释放中至少 1 段命中，即视为本招式使用 1 次。
  - 记 \(P\) 为某招式的使用次数（0～50000），则熟练度比例为：
    \[
    R = \min\left(\frac{P}{50000},\,1\right)
    \]
  - 该招式在等级 \(L\) 与使用次数 \(P\) 下的最终威力系数为：
    \[
    Final(L, P) = Base(L) \times (1 + R)
    \]
  - 当该招式熟练度达到 100%（使用次数 ≥ 50000）时，其威力在等级基础上获得 **+100%（×2）** 的线性加成。
  - 「基本拳脚」整体的“招式熟练度 100%”仅在该技能下**全部招式**熟练度均达 100% 时才视为达成，用于后续可能的额外奖励或解锁判定。

- **出招偏好与循环细节**
  - 玩家为四条肢体设定出招优先顺序后，每次普攻/使用「基本拳脚」时：
    - 系统在“未损毁且当前装备有可释放招式”的肢体中，按优先顺序选择一条作为本次出招肢体。
    - 被选中的肢体按其自身的招式循环，取出当前轮到的招式尝试释放。
  - 若本次轮到的招式因缺少动作标签、肢体状态异常等原因无法释放：
    - 本回合视为已尝试出手但**没有造成任何效果**。
    - 该肢体的招式指针仍视为“已经轮到过这一招”，在下次该肢体出手时会从下一招开始。

- **永久属性加成（后天筋骨与后天柔韧）**
  - 「基本拳脚」作为长期修行的基础功法，会在高等级时为角色提供永久的身体素质加成：
    - 每当「基本拳脚」等级累计提升 **200 级**，角色获得：
      - **后天筋骨 +1**
      - **后天柔韧 +1**
  - 该加成为全局永久加成：
    - 与当前是否装备「基本拳脚」无关，一旦获得即对所有战斗与判定生效。
    - 加成计入后天部分，不会突破先天属性的 40 点上限设定，但会与先天值相加用于实际数值计算。

### 8.4 特殊系

特殊系技能**不在常规升级/点数树中**，仅通过**剧情推进或特殊任务**解锁，每个都有独特作用，通常用于：

- 改变部分底层规则（例如某些惩罚或限制的例外情况）；
- 开启全新玩法或交互（例如特殊视角、特殊结算方式）；
- 提供无法通过数值堆叠获得的能力。

具体特殊技能列表在剧情与任务设计阶段一并细化。

  | 基础移动体力（未超重时每格消耗） | `C_move` | 1 |
  | 超重体力系数 k（每超 10% 加 k×0.1 体力） | `overweight_k` | 20 |
  | k_筋骨 公式中的偏移 | `k_jingu_offset` | 10 |
  | k_筋骨 公式中的除数 | `k_jingu_divisor` | 100 |
  | 兵器先天筋骨超额增伤系数（每超 1 点 +x%） | `weapon_jingu_bonus_coef` | 0.05 |

- **武器表**：武器应有属性（含 `req_innate_jingu` 等）在后续设计「武器属性」时一并汇总，再出表结构。
- **技能/招式配置**：出力相关（默认成数、气劲/体力基础消耗等）以 **`skill_id`** 关联；具体为每招式一行或嵌套在技能 JSON 内，以实际策划表结构为准。
- **技能 ID 与显示名**：ID → 显示名（如 `survival_strength` → 「力量」）的映射放在 **`/data/` 下技能配置表**中，每条技能包含 `id` 与显示名字段（如 `name` 或 `display_name`），便于通过读表/写表维护与扩展。
- **展示用文案**：所有需展示的文案（技能名、物品名、UI 按钮与提示、系统说明等）均从 **配置表/文案表**（如 `/data/` 下 JSON）读取，通过 **key** 引用，不写死在代码中，便于通过读表/写表维护与 agent 修改；多语言与表结构（如按模块拆表、i18n 键名约定）在实现时确定。
- **存档**：仅存于本地。分两种方式：
  - **日常/自动存档**：存放在玩家**浏览器 cookie**（或本地存储，实现时定）；只要玩家未清除缓存即可自动读取，无需输入账号密码。
  - **手动存档**：仅当玩家**手动执行存档**时，才将当前存档经 **AES-256 加密** 生成**密文代码**并交给玩家（如复制、导出）；载入时玩家输入**创建/设定该存档时使用的账号与密码**，系统进行比对，比对成功则解密并载入。
- **存档代码格式**：仅针对**手动存档**导出的密文。加密前 payload 内**包含版本号**（与存档顶层 `version` 一致）；加密后**需校验**（解密后对内容做完整性校验，校验方式实现时定，如摘要）。给玩家的代码为**纯密文**（如 Base64 编码的密文，不含明文前缀）。**载入密文**时需输入当时设定的账号与密码，比对通过后解密并载入。
- **存档顶层结构（推荐）**：解密后的存档 JSON 建议采用以下顶层 key，便于序列化与扩展；具体嵌套由实现定，可增删字段。
  - `version`：存档格式版本号，便于后续兼容与迁移。
  - `account` / `accountHash`：账号或账号校验信息，用于载入时与玩家输入的账号密码比对。
  - `character`：角色数据（先天/后天属性、技能等级、当前体力/内力、肢体状态等）。
  - `inventory`：背包（物品列表或按格位）。
  - `equipment`：当前装备，按槽位（如左手、右手、手套等）存储装备引用或 ID。
  - 其他：如地图进度、任务状态、生产建筑等，按需扩展。

# H5 丝绸朋克冒险游戏 — 完整设计文档

本文档汇总截至目前已确定的全部规则与流程，供后续直接实现（build）时参考。

---

## 一、整体设计哲学

- **世界观**：丝绸朋克（Silkpunk），锦织大陆，机关术 + 丝绸 + 竹骨 + 灵气。
- **0 教学、0 引导**：无弹窗、无任务提示、也不做“藏在世界里的教学”；规则只通过行为与结果体现。
- **高风险、高惩罚**：从一开始就使用完整死亡惩罚，不刻意对早期地牢放宽。
- **战斗非强制**：所有战斗（含最终 Boss）都可绕；进入地牢的首要目标是**生存并撤离**，而非清怪或通关。
- **无“必备物品”**：生产、料理、药品、附魔等均为可选增强，玩家可完全不依赖生产硬打地牢。

---

## 二、区域结构

### 2.1 基地（Home）

- **功能**：休整、存档；永久成长（战斗技能等级、各生产线熟练度、建筑等级）；准备本次冒险（装备、消耗品）；生产活动；与固定 NPC 对话/接任务；投保 NPC（装备快照与 AES-256 代码）。
- **生产活动**（材料 + 配方/图纸 → 产物）：
  - **制造**：一切基础；建筑所需材料与绝大部分工具来自制造。
  - **烹饪**：野外材料 + 贸易配方 → 料理，提供显著 buff。
  - **炼药/炼金**：材料 + 配方 → 战斗中即时使用的药品。
  - **锻造**：材料 + 配方/图纸 → 武器。
  - **纺织**：材料 + 配方 → 丝绸相关产物。
  - **建造/升级生产建筑**：材料主要来自制造。
  - **附魔**：地牢特殊材料 + 附魔建筑 → 装备特殊强化。
- **生产规则**：
  - 有制作时间；可多建筑并行，单建筑同一时刻只能制作 1 件；失败全额扣材料。
  - 熟练度：各线独立、不共用；角色等级 + 建筑等级；仅成功制作涨熟练度；成功/失败与熟练度挂钩；建筑升级可缩短时间、增加产出、降失败率、提品质。
  - 品质六档：粗糙 → 普通 → 精良 → 稀有 → 史诗 → 传说；颜色：灰 → 白 → 绿 → 蓝 → 紫 → 橙。

### 2.2 野外（Field）

- **定位**：低烈度区；移动、采集、与较弱敌人战斗，积累资源，为地牢做准备；经验与掉落明显低于地牢。
- **地图**：主路相对安全（几乎不遇敌）；支路/偏远区敌人与采集点更密集；资源点与城镇/地牢入口为固定格子，每局位置不变。
- **采集点**（采完后按“行动回合数”刷新）：矿脉、药草、木材/竹材、狩猎点、丝绸相关、钓鱼（食材 + 稀有材料/装备）；不强制工具，有工具效率更高。
- **敌人**：不主动远距离追逐；玩家靠近到一定范围后朝玩家移动并可能触发战斗；普通天气下弱于地牢，特定天气/时间下强度可接近低层地牢。
- **天气与时间**：影响资源品质（原有点提升 + 专属高品质点）；影响怪物种类与强度。
- **回程**：角色无便捷回城；需走回城镇/安全出口；城镇 ↔ 基地有需条件解锁的快捷传送。采集不额外增加危险。

### 2.3 城镇（Town）

- **核心**：贸易（装备、材料、配方等）。
- **附加**：有限休整、特殊服务（如鉴定）、与解锁部分传送/设施相关。

### 2.4 地牢（Dungeon）

- **结构**：多个随机生成的小房间组成大迷宫；可配置层数；房间间通过门格切换；进入后无法退回野外，只能找出口撤离或下潜。
- **无通关概念**：收益 = 实际拿到并带出的东西；系统不按层数/时间/击杀给额外结算奖励。
- **战斗**：敌人朝玩家靠近，接近触发战斗；玩家可提前离开房间完全绕过战斗（含 Boss）。

## 三、死亡惩罚与装备快照（投保）

- **死亡时**：身上全部物品消失；战斗技能扣 50 级（不扣到 0 以下）；传送到“地狱”对话后回基地，状态回满；基地仓库与生产存货不受影响。
- **投保**：下地牢前可付费对当前穿戴装备做一次快照；同一时间仅 1 个生效快照。
- **代码**：死亡后 NPC 给出一段 AES-256 加密代码（含玩家信息、地牢标识、快照装备等），玩家不可反推内容。
- **使用代码**：必须在同一地牢同一入口输入；进入后地牢仍完全随机，系统随机指定一房间生成可互动元素；需到达该房间并成功撤离才可从 NPC 取回装备（仅装备，消耗品不恢复）。
- **消耗**：一旦使用代码，不论成败，该玩家此代码即消耗；未到指定房间或死亡则失去机会。
- **共享**：同一代码可被多玩家使用；每人仅一次机会；成功则各自从 NPC 获得那套装备。

## 四、实战经验值

- 战斗胜利获得，线性累计，上限 1 亿；满值 +30% 全局伤害。
- 永不回退；实战经验远高于某敌人时，再打该弱敌不再获得实战经验。

## 五、角色基础属性

本作中角色拥有五项基础属性，这些属性既影响战斗表现，也作用于负重、恢复等系统：

- **筋骨**：影响近战攻击伤害与负重能力，是“硬力量”的直接体现。
- **柔韧**：决定肢体基础防御（动态减伤）；命中部位与伤害类型通过伤害类型微调系数参与最终伤害（详见 5.6）。
- **呼吸**：影响内力上限与体力/内力恢复速度，与生存系技能「凝气」的效果叠加。
- **身手**：影响战斗速度（进而影响先手、连击与命中率）；装备的轻功提供基础速度，身手每 +1 点使速度 +0.5%，与装备等加成相加。
- **专注**：影响技能学习的效率（含技能书学习与自修学习），并影响“精力 → 潜能消耗”的转化效率。

以上五项基础属性**均分为先天值与后天值**：

- **先天属性**：在创建角色时决定，每项上限为 40 点，仅有极少数特殊手段可以提升。
- **后天属性**：来自技能等级、装备、buff 等途径的加成全部计入后天部分。

在各类数值计算中（如攻击伤害、负重上限、命中率等），默认使用「先天 + 后天」之和作为该项属性的实际值，不再区分来源；但在系统展示与成长设计上，始终区分先天与后天，便于强调“天赋”与“修行”的差异。

**实现约定（数据与缓存）**：五项基础属性在角色数据中均以**先天、后天两个独立字段**存储；实际值 = 先天 + 后天。当任一来源（装备、技能等级、buff 等）导致先天或后天属性变化时，**实时重算并写入缓存**，保证后续公式与 UI 读取到的为最新值。**统一入口**：装备变化、buff 增删、技能升级等均调用同一套重算入口（如 `recalcCharacterStats()`），在该入口内重算后天属性及**负重上限 \(W_{\max}\)** 等衍生值并写缓存；检查负重时只读缓存。**触发重算的事件清单**（以下事件发生后应调用统一重算入口，避免漏算）：
  - 穿戴 / 脱下 / 替换任意装备；
  - 增加或移除任意 buff（含过期）；
  - 任意技能等级变化（升级或降级）；
  - 更换内功或内功状态变化（若影响属性或衍生值）；
  - 其他会改变先天/后天来源的操作（如使用永久加属性的道具等）。**属性下限**：筋骨、柔韧、呼吸、身手、专注及技能等级等参与公式时，**最小为 0**，负数**视为 0** 再代入计算。

### 5.1 衍生属性（由基础属性计算得出的具体数值）

以下衍生属性均由五项基础属性的**实际值**（先天 + 后天）参与计算，用于战斗、生产与生存的各类公式。公式中的“筋骨”“柔韧”等均指该属性的实际值。

| 衍生属性 | 主要参与属性 | 计算公式约定 | 用途简述 |
|----------|--------------|----------------|----------|
| **负重上限** | 筋骨、力量技能等级 | 基础负重 + 筋骨系数 × 筋骨值 + 力量技能加成；装备、背包、材料等合计重量不可超过此值。 | 携带物品、生产时可持材料量。 |
| **近战伤害系数** | — | （已移除）原“筋骨系数 \(k_{\text{筋骨}}\)”乘区不再使用；徒手伤害直接以筋骨实际值作为基础威力值参与计算，兵器按 5.5.3 的先天门槛/超额增伤规则计算。 | — |
| **肢体基础防御与伤害类型微调** | 柔韧 | 柔韧决定基础防御（动态%）；命中部位与伤害类型使用伤害类型微调系数 \(M_{\text{部位,类型}}\)；详见 5.6。 | 减伤链：基础防御后接伤害类型微调。 |
| **肢体损毁值增长** | — | 损毁值增长 = 该部位该次承受的最终伤害（减伤链结算后），向下取整；柔韧通过基础防御间接减少最终伤害从而减少损毁。 | 肢体损毁。 |
| **内力上限** | 呼吸、基本内功 | 基本内功技能提供内功容量基准值 \(N_{\text{base}}\)；最大内力上限 = \(N_{\text{base}} \times (1 + 0.01 \times S_{\text{呼吸}} + \sum P_{\text{其他来源}})\)，向下取整。基本内功升级不直接加当前内力上限；其他百分比来源（内功、装备、buff）与呼吸加成相加。 | 技能释放、内力护体等。 |
| **内力恢复速度** | 呼吸 | 每回合或每单位时间内力恢复量 =（内力基础恢复值 + 呼吸系数 × 呼吸值）×（1 + 凝气加成），具体常数见内力条目与配置表；体力恢复见 6.1.3「体力（Stamina）」。 | 战斗续航、地牢内恢复。 |
| **战斗速度** | 身手、轻功、装备 | 速度 = \(V_{\text{base}} \times (1 + 0.005 \times S_{\text{身手}} + \sum P_{\text{装备等}})\)；\(V_{\text{base}}\) 由装备的轻功提供，未装备轻功时为 1。先算浮点再向下取整，用于先手、连击与命中率。 | 先手、连击、命中率。 |
| **学习效率（技能书）** | 专注 | 从书本学习技能时，所需时间或消耗与专注值成反比；专注越高，学习越快或消耗越少。 | 技能书使用。 |
| **学习效率（自修）** | 专注 | 自修学习技能时：每 tick 固定消耗 1 点精力；同 tick 可消耗潜能上限 \(Pot_{\text{perEnergy}} = 200 \times (1 + 0.05 \times (\max(S_{\text{专注}},0) - 10))\)，向下取整且最小为 0；当前潜能不足本 tick 所需时自修立即停止。自修每 tick 获得的经验 = 基础经验 × \(F_{\text{专注}}\)（由配置/曲线定义）。 | 自修学习技能。 |
| **精力恢复速度** | — | 精力恢复不再直接受基础属性影响，而由后续具体系统（如睡眠、道具、被动技能等）在各自条目中单独定义。 | — |

**公式缩放约定**（实现时可采用）：

**实现约定（常数来源）**：筋骨、负重、超重体力等公式中的数值常数（如 \(W_{\text{基础}}\)、筋骨/力量系数、\(C_{\text{move}}\)、\(k\) 等）**全部从配置表读取**，不写死在代码中，便于策划调参。

- **筋骨 → 近战伤害**：不再引入额外的“筋骨系数 \(k_{\text{筋骨}}\)”乘区。徒手攻击的基础威力值直接取筋骨实际值 \(S_{\text{筋骨}}\)（见 5.5.2），不再额外乘以基于筋骨的倍率；兵器攻击的筋骨相关规则统一见 5.5.3（先天门槛/惩罚 + 先天超额增伤）。
- **柔韧 → 基础防御**：见 5.6.2 动态公式（\(DR = S_{\text{柔}}/(S_{\text{柔}}+3D)\)）。
- **呼吸 → 内力上限**：\(\text{最大内力上限} = N_{\text{base}} \times (1 + c_{\text{呼吸}} \times S_{\text{呼吸}} + \sum P_{\text{其他来源}})\)，向下取整；\(N_{\text{base}}\) 由基本内功技能提供；其中 \(c_{\text{呼吸}}\) 从配置表字段 `breath_neili_cap_pct_per_point` 读取（默认 0.01），\(P_{\text{其他来源}}\) 为其它“最大内力 +x%”来源的加总。
- **身手 → 战斗速度**：见 5.7 身手属性细则；命中率由**速度**参与，见「七、命中、招架与伤害流程」。

### 5.4 后天属性成长来源简述

后天属性的增加**不改变**先天上限（每项先天仍以 40 为上限），但实际值 = 先天 + 后天可超过 40。主要来源：

- **战斗技能等级**：如「基本拳脚」每 200 级 +1 后天筋骨、+1 后天柔韧；其他战斗技能可在各自条目中定义类似永久加成。
- **装备与词条**：装备提供的“+筋骨”“+身手”等一律计入后天。
- **Buff 与状态**：临时增益（料理、药品、机关术效果等）计入后天，过期即移除。
- **特殊剧情/道具**：极少数可**提升先天属性**（或提高先天上限）的道具或剧情，在剧情与数值设计中单独列出；若仅增加属性但不触及先天，则计入后天部分。

实现时，所有来源的数值先汇总为“后天该项”，再与先天相加得到**实际值**，用于 5.1、5.2、5.3 的公式与表。

**战斗技能学习/使用门槛**：所有战斗技能（含内功、徒手、兵器、轻功、招架等）若存在学习或使用门槛（如“筋骨≥X”“呼吸≥Y”等），一律只与角色该项属性的**先天值**比较，后天加成不参与门槛判定；与「兵器先天筋骨要求」规则一致并推广至全战斗技能。

### 5.5 筋骨属性细则（徒手、兵器与负重）

#### 5.5.1 初始分配与成长区间

- **创建角色时的先天分配**
  - 五项基础属性的**基础先天值**均为 10 点。
  - 角色创建时额外获得 **50 点自由点数**，可在「筋骨 / 柔韧 / 呼吸 / 身手 / 专注」之间任意分配。
  - **允许手动降低基础先天值换取自由点**：玩家可将任一项基础先天值从 10 点向下调低以获得等额自由点（-1 点基础先天 → +1 点自由点），但任一项在创建阶段**不能低于 0 点**。
  - **必须分配完全部自由点**：玩家必须将当前可用的自由点数全部分配完毕，才允许完成角色创建。
  - 单项先天属性在创建阶段的上限为 **29 点**（含基础 10 点），即任一项在捏人时不能被加到 30 点及以上。
  - 全局规则“先天属性上限 40 点”仍然有效，意味着剩余空间（29～40）仅能通过**极少数特殊手段**提升。

- **筋骨典型数值区间（实际值 = 先天 + 后天）**
  - **新手期**：完成捏人并获得少量后天加成后，筋骨实际值通常落在 **20～35** 区间。
  - **中期**：围绕近战相关战斗技能进行长期修行，并配合常规装备与 buff 后，筋骨实际值可稳定提升至 **数百等级（约 300～400）**。
  - **后期与长线**：随着版本更新与高等级战斗技能、稀有装备的加入，筋骨实际值预期可达到 **2000～3000** 甚至更高；所有与筋骨相关的公式均按“高值可用，但收益边际递减”的思路设计。

#### 5.5.2 筋骨与徒手战斗

- **徒手基础威力值（由筋骨曲线给出）**
  - 记 \(S_{\text{筋骨}}\) 为筋骨实际值（先天 + 后天，且小于 0 视为 0）。
  - 徒手攻击的“基础威力值”不再直接等于 \(S_{\text{筋骨}}\)，而由一条“前期增长明显、后期乏力”的饱和曲线给出：
    \[
    B_{\text{fist}}(S_{\text{筋骨}})
    = \left\lfloor 650 \times \left(1 - e^{-S_{\text{筋骨}}/450}\right) \right\rfloor
    \]
  - **实现约定**：曲线常数（650、450）最终从配置表读取；此处为当前约定值，供数值与物品设计阶段对齐使用。

- **徒手伤害公式示意**
  - 记 \(S_{\text{筋骨}}\) 为筋骨实际值，则单次徒手攻击的伤害可表示为：
    \[
    D_{\text{徒手}}
    = \underbrace{B_{\text{fist}}(S_{\text{筋骨}})}_{\text{基础威力值（筋骨曲线）}}
    \times T_{\text{手套词条}}
    \times F_{\text{技能威力}}
    \times F_{\text{内功威力}}
    \times F_{\text{出力大小}}
    \times F_{\text{环境修正}}
    \times F_{\text{实战经验}}
    \]
  - 上式中，各乘区的含义为：
    - \(T_{\text{手套词条}}\)：徒手装备（手套等）提供的伤害修正。**实现约定**：**每手一个手套槽**（左手、右手各自独立）；徒手攻击时**只看当前出招手**所装备手套的词条系数，该手无手套则为 1.0。手套类装备在设定时于配置中**单独标注**类型或标签（如槽位 ID、装备类型）以便代码识别。
    - \(F_{\text{技能威力}}\)：招式或技能本身的威力系数；
    - \(F_{\text{内功威力}}\)：当前内功对招式威力的修正；
    - \(F_{\text{出力大小}}\)：玩家选择的出力程度（如 3 成、7 成、全力）；
    - \(F_{\text{环境修正}}\)：环境因素（地形、天气等）带来的加成或减益；
    - \(F_{\text{实战经验}}\)：由「四、实战经验值」给出的全局增伤。

#### 5.5.2.1 模型表：徒手基础威力曲线与兵器基准攻击力锚点（供物品数值参考）

- **用途**：为了让“同阶段的基准兵器”与徒手在输出量级上可对齐，先固定徒手基础威力曲线，再用兵器筋骨乘区反推“基准武器的基础攻击力”应落在哪些点。
- **默认对齐条件（用于反推）**：
  - 徒手与兵器的其他乘区（手套/词条/技能威力/内功威力/出力/环境/实战等）在对齐时视为相同；
  - 以“角色先天筋骨 = 30”为参考（典型捏人上限附近）；
  - 兵器使用门槛达标（\(M_{\text{门槛}}=1\)），仅比较“超额增伤”带来的倍率差异；
  - 兵器基础攻击力指武器表中的 \(B_{\text{weapon}}\)（见 5.5.3 兵器伤害公式）。
- **兵器筋骨乘区回顾（达标时）**：
  - 若 `req_innate_jingu` = 20，则先天 30 的超额增伤 \(M_{\text{筋骨}} = 1 + 0.05 \times (30-20) = 1.5\)；
  - 若 `req_innate_jingu` = 30，则先天 30 的超额增伤 \(M_{\text{筋骨}} = 1.0\)。
- **锚点表（近似值，便于策划配表）**：

  | 筋骨实际值 S | 徒手基础威力 \(B_{\text{fist}}(S)\) | 基准武器基础攻击力（Req=20，先天30） | 基准武器基础攻击力（Req=30，先天30） |
  |---:|---:|---:|---:|
  | 30 | ≈ 41 | ≈ 28 | ≈ 41 |
  | 200 | ≈ 235 | ≈ 157 | ≈ 235 |
  | 500 | ≈ 438 | ≈ 292 | ≈ 438 |
  | 800 | ≈ 541 | ≈ 361 | ≈ 541 |
  | 1100 | ≈ 596 | ≈ 397 | ≈ 596 |
  | 1800 | ≈ 639 | ≈ 426 | ≈ 639 |
  - **解读**：在“Req=20 且先天30”的情况下，由于自带 1.5× 超额增伤，同阶段的“基准武器基础攻击力”大约是徒手基础威力的 \( \frac{1}{1.5} \)；Req=30 时则约等于徒手基础威力本身。

#### 5.5.3 筋骨与兵器战斗

筋骨对兵器战斗的影响分为两层：**使用门槛/惩罚（只看先天筋骨）**与**额外增伤（只看先天筋骨）**。**实现约定**：兵器伤害不使用 5.1 中的 \(k_{\text{筋骨}}\)（筋骨实际值系数），避免与先天门槛/超额增伤重复叠乘。

- **（一）兵器使用门槛与惩罚——只看先天筋骨**
  - 每把兵器定义一个**先天筋骨要求**，记为 \(Req_{\text{筋骨}}\)。
  - **实现约定**：兵器/武器配置表中，该数值的字段名为 `req_innate_jingu`（**先天筋骨要求**）；未配置或为空时取默认值 20；配置为 0 或负数时**统一视为 0**（无门槛）。
  - 仅使用角色的**先天筋骨**与 \(Req_{\text{筋骨}}\) 比较，用于判定“能不能用”“是否吃惩罚”；后天筋骨不参与这一比较。
  - 记 \(S_{\text{先天}}\) 为角色的先天筋骨：
    - **无法使用**：若
      \[
      S_{\text{先天}} < \tfrac{1}{2} Req_{\text{筋骨}}
      \]
      则角色**不能装备/挥动**该兵器。
    - **惩罚区间（可用但吃削弱）**：若
      \[
      \tfrac{1}{2} Req_{\text{筋骨}} \le S_{\text{先天}} < Req_{\text{筋骨}}
      \]
      兵器可以使用，但进入惩罚区；此时伤害乘以一个线性插值的门槛修正 \(M_{\text{门槛}}\)：
      \[
      t = \frac{S_{\text{先天}} - Req_{\text{筋骨}}/2}{Req_{\text{筋骨}}/2} \in [0,1]
      \]
      \[
      M_{\text{门槛}} = 0.5 + 0.5 \times t
      \]
      - 当 \(S_{\text{先天}} = \tfrac{1}{2} Req_{\text{筋骨}}\) 时，\(M_{\text{门槛}} = 0.5\)，即仅造成 50% 伤害；
      - 当 \(S_{\text{先天}} = Req_{\text{筋骨}}\) 时，\(M_{\text{门槛}} = 1.0\)，恰好不加不减。
    - **达标与以上（无惩罚）**：若
      \[
      S_{\text{先天}} \ge Req_{\text{筋骨}}
      \]
      则 \(M_{\text{门槛}} = 1.0\)，完全不吃先天筋骨带来的惩罚。

- **（二）额外增伤——仅看先天筋骨**
  - 在门槛层面判定完之后，**仅**角色的**先天筋骨**参与额外增伤，记为 \(M_{\text{筋骨}}\)：
    \[
    M_{\text{筋骨}}
    = 1 + 0.05 \times \max(S_{\text{先天}} - Req_{\text{筋骨}},\ 0)
    \]
    即每比兵器要求多 1 点先天筋骨，增伤 +5%；**无上限**。0.05 从配置表读取。
  - 最终兵器攻击的筋骨相关乘区为：
    \[
    M_{\text{兵器-筋骨}} = M_{\text{门槛}}(S_{\text{先天}}, Req_{\text{筋骨}}) \times M_{\text{筋骨}}
    \]

- **（三）兵器伤害公式示意**
  - 记 \(B_{\text{兵器基础威力}}\) 为兵器本身的基础威力，则兵器战斗技能的伤害示意为：
    \[
    D_{\text{兵器}}
    = B_{\text{兵器基础威力}}
    \times T_{\text{兵器词条}}
    \times F_{\text{技能威力}}
    \times F_{\text{内功威力}}
    \times M_{\text{兵器-筋骨}}
    \times F_{\text{出力大小}}
    \times F_{\text{环境修正}}
    \times F_{\text{实战经验}}
    \]

#### 5.5.4 筋骨与负重

- **当前负重 W（实现约定）**：装备与背包**共用同一重量池**；\(W\) = 身上装备重量 + 背包内物品重量，与 \(W_{\max}\) 比较。物品重量单位与 \(W_{\max}\) 一致（kg）。**数值精度**：\(W\) 与 \(W_{\max}\) 全程使用浮点数计算与比较，超重判定（\(W > W_{\max}\)、\(W > 1.5 W_{\max}\)）及移动体力消耗公式中的 \(W/W_{\max}\) 均用浮点运算。**\(W_{\max}\) 缓存**：\(W_{\max}\) 在统一重算入口（见上文「数据与缓存」）内与属性一并计算并写入缓存，检查负重时只读缓存。**展示取整**：向玩家展示当前负重 \(W\) 与负重上限 \(W_{\max}\) 时，**保留一位小数**（计算与判定仍用浮点）。**力量满级武器不计负重**：当生存系技能「力量」（`survival_strength`）达到 **100 级**时，计算当前负重 \(W\) 时**不计入左手、右手武器槽中已装备的武器**重量；扣掉后若仍超重，超重判定与移动体力消耗仍按该扣掉后的 \(W\) 计算。
- **负重上限公式**
  - 记 \(W_{\max}\) 为角色当前负重上限，\(S_{\text{筋骨}}\) 为筋骨实际值，\(L_{\text{力量}}\) 为生存系技能「力量」的等级（技能 ID：`survival_strength`；未习得时视为 0），则：
    \[
    W_{\max}
    = W_{\text{基础}}
    \times \bigl(1 + 0.01 \times L_{\text{力量}} + 0.0005 \times S_{\text{筋骨}}\bigr)
    \]
    其中 **\(W_{\text{基础}} = 25\)**（kg），力量每级 +1% 总负重，筋骨每点 +0.05% 总负重。\(W_{\text{基础}}\)、0.01、0.0005 均从**配置表**读取。

- **超重惩罚规则**
  - 记当前实际负重为 \(W\)：
    - **未超重**：若 \(W \le W_{\max}\)，移动与战斗不受负重惩罚，移动体力消耗为基础值 \(C_{\text{move}}\)。
    - **轻度～中度超重（可移动，但越走越累）**：若
      \[
      W_{\max} < W \le 1.5 \times W_{\max}
      \]
      角色仍可移动，但每移动一格的体力消耗随超重比例线性增加，例如：
      \[
      C_{\text{move, 实际}}
      = C_{\text{move}} + k \cdot \bigl(\tfrac{W}{W_{\max}} - 1\bigr)
      \]
      其中 \(C_{\text{move}} = 1\)（未超重时每格 1 体力），\(k\) 从配置表读取，**当前约定为 20**（刚好超重时每格 1 体力，每超 10% 加 2 体力）。目前设计中，超重仅通过**体力消耗加剧**体现，不强制削减命中/闪避等战斗属性。
    - **严重超重（无法移动）**：若
      \[
      W > 1.5 \times W_{\max}
      \]
      角色**无法移动**，只能通过丢弃物品或使用特殊手段降低负重，直至负重不超过 \(1.5 \times W_{\max}\)。
  - **实现约定（负重检查触发）**：所有**可能改变当前负重 \(W\)** 的动作都必须检查当前负重 \(W\) 与 \(W_{\max}\)，据此判定是否超重、是否严重超重。包括但不限于：移动前、打开背包前、拾取/捡起、丢弃、穿戴/脱下/替换装备、将物品在地面/背包/装备栏之间移动、制作产出并入背包、拆解/合成导致重量变化、交易买入/卖出、仓库取出/存入等。严重超重时**仅禁止移动**，不禁止使用物品、换装、战斗等其他操作。

### 5.6 柔韧属性细则（基础防御、伤害类型微调与肢体损毁）

#### 5.6.1 定位与分工

- **数值尺度**：柔韧的实际值（先天 + 后天）与筋骨保持同一量级：
  - 新手期：约 20～35；
  - 中期：约 300～400；
  - 长线后期：可达到 2000～3000 乃至更高。
- **主职**：柔韧决定角色的**肢体基础防御**，即对「内功、防具减伤后」的剩余伤害再做一次**动态百分比减伤**（见 5.6.2）；不按部位区分，全身共用同一柔韧值。
- **伤害类型微调**：各身体部位（头、胸、腹、左手、右手、左脚、右脚）对**劈砍、戳刺、钝击**三种伤害类型有**伤害类型微调系数** \(M_{\text{部位,类型}}\)，在基础防御之后作为最终伤害的乘数（见 5.6.2 表格）。
- **损毁**：肢体损毁值增长 = 该部位该次承受的**最终伤害**（减伤链结算后），**向下取整**；柔韧仅通过提高基础防御间接降低最终伤害，从而减少损毁增长，不再单独设柔韧损毁减缓系数。

#### 5.6.2 基础防御与伤害类型微调系数

- **基础防御（动态减伤）**：柔韧实际值直接作为防御值，参考 PoE 护甲公式，对经过内功→防具减伤后的**剩余伤害** \(D_{\text{进}}\) 做动态百分比减伤：
    \[
  DR_{\text{基础防御}}
    = \frac{S_{\text{柔韧}}}{S_{\text{柔韧}} + 3 \times D_{\text{进}}}
    \]
    \[
    D_{\text{出}} = D_{\text{进}} \times \left(1 - DR_{\text{基础防御}}\right)
    \]
    其中 \(S_{\text{柔韧}}\) 为柔韧实际值（先天 + 后天），\(D_{\text{进}}\) 为当前减伤步骤的输入伤害，常数 **3** 为设计锚点（可在数值阶段微调）。
  - **边界**：计算时若 \(S_{\text{柔韧}} \le 0\)，视为 0，即该步无减伤（\(DR_{\text{基础防御}} = 0\)，\(D_{\text{出}} = D_{\text{进}}\)）。
  - **数值参考**：

    | 柔韧 | 进入伤害 | DR | 输出伤害 |
    |------|---------|-----|---------|
    | 25（新手） | 20 | 29% | ≈ 14 |
    | 100 | 50 | 40% | ≈ 30 |
    | 400（中期） | 100 | 57% | ≈ 43 |
    | 2000（后期） | 200 | 77% | ≈ 46 |

  - 该公式具有**动态缩放**特性：面对小伤害时减伤比例较高，面对大伤害时减伤比例相应下降，不存在固定硬减伤上限。
  - **完整减伤链顺序**：内功（%）→ 防具（%）→ 基础防御（柔韧动态%，\(\frac{S_{\text{柔}}}{S_{\text{柔}}+3D}\)）→ **伤害类型微调系数**（\(M_{\text{部位,类型}}\)）。
- **取整规则**：减伤链**各步之间**统一**保留 1 位小数**并继续传递；仅在得到**最终扣血**与**损毁增加值**时，对该值**向下取整到非负整数**再写入角色/肢体。
  - **适用范围**：当前减伤链与伤害类型微调**仅针对劈砍、戳刺、钝击**三种物理伤害类型；毒、火、真实伤害等其他类型是否参与基础防御或微调系数，在扩展时另行约定。
- **伤害类型微调系数 \(M_{\text{部位, 类型}}\)**：在基础防御之后应用。单次攻击仅带一种伤害类型（劈砍 / 戳刺 / 钝击之一）；命中某部位后，该次**最终伤害**（用于扣血与损毁） = 基础防御步骤输出 × \(M_{\text{部位, 类型}}\)，链内保留小数，仅在作为最终扣血与损毁增加值时向下取整到整数。\(M > 1\) 表示该部位对该类型更脆弱（承伤更高），\(M < 1\) 表示相对更耐。

- **推荐的 \(M_{\text{部位, 类型}}\) 表**（可在数值阶段细调）：

  | 部位 \\ 伤害类型 | 劈砍 | 戳刺 | 钝击 | 说明 |
  |----------------|------|------|------|------|
  | **头** | 1.0 | 1.1 | 1.3 | 头部对钝击最脆，戳刺略高，劈砍视为基准。 |
  | **胸** | 1.0 | 1.2 | 1.1 | 肋骨 + 内脏，对戳刺尤其危险，对钝击也略脆。 |
  | **腹** | 1.1 | 1.3 | 1.0 | 腹部软组织与空腔器官多，对劈砍/戳刺更脆，钝击中等。 |
  | **左手/右手** | 1.0 | 1.0 | 1.1 | 上肢骨骼表浅，钝击更易出现骨折，其他按基准。 |
  | **左脚/右脚** | 0.9 | 1.0 | 1.1 | 下肢肌肉较厚，对劈砍略有“肉垫”，钝击同样易骨折。 |

  - \(M_{\text{部位,类型}}\) **不做数值范围限制**，配置/策划表中填写的值直接参与计算。

#### 5.6.3 肢体损毁值增长

- **损毁值增长** = 该部位该次承受的**最终伤害**（即减伤链「内功 → 防具 → 基础防御 → 伤害类型微调系数」结算后的数值，链内保留小数），对该值**向下取整到非负整数**作为本次损毁增加值。柔韧不单独参与损毁公式，仅通过基础防御间接降低最终伤害，从而减少损毁增长。
- 骨折判定、损毁值上限、手术恢复等仍主要由肢体系统、生存系「健康」「手术」等定义。

#### 5.6.4 与生存系「健康」等的协同

- 生存系「健康」继续负责降低骨折概率、减少日常饱食/饮水消耗等；柔韧不取代健康在骨折概率上的作用。

### 5.7 身手属性细则（战斗速度与命中率）

- **身手无上下限**：身手参与速度与命中公式时不设上下限，按实际值计算。
- **战斗速度公式**：
  - **基础速度** \(V_{\text{base}}\)：由当前装备的**轻功**提供；**未装备轻功时为 1**。
  - **身手加成**：每 +1 点身手，速度在基础值上 +0.5%，即乘区为 \(1 + 0.005 \times S_{\text{身手}}\)。
  - **装备等加成**：装备（及 buff 等）提供的速度百分比与身手百分比**相加**，放在同一乘区：
    \[
    \text{速度（浮点）} = V_{\text{base}} \times \left(1 + 0.005 \times S_{\text{身手}} + \sum P_{\text{装备等}}\right)
    \]
  - **取整与下限**：先按上式算完速度（浮点），再**向下取整到整数**；若结果小于 1，则**视为 1**。取整后的速度用于先手判定、连击倍数判定与命中率公式。其中 0.005 从配置表字段 `dexterity_speed_pct_per_point` 读取；“未装备轻功时为 1”从字段 `base_speed_no_qinggong` 读取。
- **命中率**：由攻防双方**速度值**参与，见「七、命中、招架与伤害流程」；身手通过速度间接影响命中率。

### 5.8 专注属性细则（自修学习：tick 经验与精力→潜能效率）

- **适用范围**：本节仅定义“自修学习技能”的专注相关规则；技能书学习仍按 5.1 表中「学习效率（技能书）」的描述执行。
- **专注取值**：参与自修公式时，\(S_{\text{专注}} = \max(\text{专注实际值}, 0)\)；低于 0 视为 0。
- **自修每 tick 的固定消耗**：每 tick 固定消耗 **1 点精力**。
- **每 1 点精力可消耗的潜能上限**：
  - **基准点**：当 \(S_{\text{专注}}=10\) 时，\(Pot_{\text{perEnergy}}=200\)。
  - **线性修正**：专注每比 10 高 1 点，+5%；每比 10 低 1 点，-5%：
    \[
    Pot_{\text{perEnergy}} = 200 \times \left(1 + 0.05 \times (S_{\text{专注}} - 10)\right)
    \]
  - **取整与下限**：先算浮点，再**向下取整**；若结果小于 0，则视为 0。
- **停止条件（潜能不足）**：当角色当前潜能不足以支付本 tick 的潜能消耗（即不足 \(Pot_{\text{perEnergy}}\)）时，自修**立即停止**（本 tick 不再进行）。
- **自修每 tick 的经验增长**：每 tick 获得的经验 = 基础经验 × \(F_{\text{专注}}\)。\(F_{\text{专注}}\) 的具体曲线与常数从配置表读取（本节只约定“专注正相关”与接口，不写死曲线形状）。

- **实现约定（字段一致性）**：自修相关常数统一从以下配置字段读取：`study_tick_energy_cost`（默认 1）、`focus_baseline`（默认 10）、`study_potential_per_energy_base`（默认 200）、`focus_potential_per_energy_pct_per_point`（默认 0.05）。文中出现的 1/10/200/0.05 为当前约定值，最终以配置表为准。

## 六、人物生存属性与战斗核心

### 6.1 饱食（Satiety）

- **名称与字段**：
  - 展示名：**饱食**
  - 英文字段建议：`satiety`
  - 数值范围：`0.0 ～ 120.0`，保留一位小数（其中 100.0 为“正常上限”，120.0 为“过量上限”，见下）。

- **含义**：
  - 表示角色近期摄入能量的充盈程度，用于驱动体力/内力恢复与行动限制。
  - 不等同于“长期营养状态”，长期营养如需设计，另立隐藏值或独立系统。

- **基础变化规则**：
  - **tick 与消耗**：本作中一切“会推动时间前进”的行为（在地图上移动一格、执行一次战斗/生产/操作等），统称为 **1 个 tick**。
  - 不执行动作不会推进 tick（例如停在原地打开 UI 仅查看信息，不算 tick）。
  - **每 tick 固定消耗**：每经过 **1 个 tick**，饱食固定 **-1.0**（可在配置中按全局或区域系数调整）。
  - 饱食度不会自然回升，只能通过进食或特殊手段恢复。
  - **典型时间尺度参考**：一次标准地牢探索预期约 **100～200 tick**（仅供平衡参考，不作为硬规则）。

- **饱食区间与效果**（区间包含端点，按当前饱食值判定）：

  1. **正常**：`60.0 ～ 100.0`
     - 无任何负面效果。

  2. **稍微饥饿**：`40.0 ～ 59.9`
     - 无任何数值惩罚，仅作为 UI 提示（如进度条颜色变化、文案提示）。

  3. **中等饥饿**：`15.0 ～ 39.9`
     - **体力自然恢复值 ×0.5**。
     - **内力自然恢复值 ×0.5**。
     - 仍可正常进行所有行动，包括战斗与生产。

  4. **重度饥饿**：`0.1 ～ 14.9`
     - 禁止执行一切会**消耗体力或精力**的动作，包括但不限于：
       - 消耗体力/精力的战斗技能与特殊动作；
       - 需要体力/精力的生产行为（如挖矿、伐木、锻造等）。
     - 仍允许的行为包括：不消耗体力/精力的基础移动（可视为“拖着走”）、查看界面、开关背包、进食、使用不耗体力/精力的物品等。
     - **允许攻击**：若某次攻击/动作本身不消耗体力或精力，则在重度饥饿下仍允许执行；仅对“消耗体力或精力”的动作做禁用。

  5. **极限饥饿（死亡倒计时）**：`= 0.0`
     - 当饱食度第一次降至 **0.0** 时，开始计数一个“极限饥饿倒计时”，以 tick 为单位。
     - 在饱食度持续为 0.0 的前提下：
       - 若连续经过 **100 个 tick** 仍未进食（饱食度一直保持 0.0），则角色视为因极度饥饿**昏迷并死亡**，触发正常死亡惩罚流程（见「三、死亡惩罚与装备快照」）。
       - 若在 100 tick 内通过进食使饱食度重新大于 0.0，则立即终止本次倒计时；下次再次跌到 0.0 时重新计数。
     - 在 0.0 且未死亡的这段时间内，行动限制与“重度饥饿”一致（禁止消耗体力/精力的动作）。

- **实现约定（配置字段建议）**：
  - `satiety_max`：饱食度上限（当前约定 100.0）。
  - `satiety_overcap_max`：过量饱食上限（当前约定 120.0）。
  - `satiety_tick_decay`：每 tick 固定消耗值（当前约定 1.0）。
  - `satiety_normal_min`：正常区间下界（当前约定 60.0）。
  - `satiety_mild_min`：稍微饥饿区间下界（当前约定 40.0）。
  - `satiety_moderate_min`：中等饥饿区间下界（当前约定 15.0）。
  - `satiety_starvation_ticks_to_death`：饱食度为 0.0 时连续存活 tick 数上限（当前约定 100）。

#### 6.1.1 过量进食（>100）与体重（Weight）

- **规则目标**：只有在“已经吃到 100.0”时继续进食，才会进入 100.0 以上的过量区间；过量/饥饿会在长时间维持后改变体重。

- **过量进食与上限**：
  - **正常上限**：\(satiety_{\max}=100.0\)。
  - **过量上限**：\(satiety_{\text{overcap}}=120.0\)（最高只能到 **120.0/100.0** 的效果上限）。
  - **关键触发条件**：仅当当前饱食度 **恰好为 100.0** 时，再使用任何“可增加饱食度”的物品，才允许把饱食压入 \(>100.0\) 区间。
  - **饱食未满时的进食限制**：当饱食度 **小于 100.0** 时，单次使用食物最多只能恢复到 **100.0**，不会超过。

- **体重（暂定仅记录）**：
  - 字段建议：`weight_kg`，单位 kg（体重具体作用后续在“体重系统”中细化）。
  - 体重变化由两个计数器驱动（均以“连续 tick”计数）：

  1. **过量增重（+1kg）**
     - 条件：饱食度 **持续 > 100.0** 连续 **500 tick**。
     - 结果：体重 **+1 kg**，随后该计数器清零重新开始（若仍 >100.0 则继续累积下一次）。
     - 一旦饱食度 \(\le 100.0\)，连续计数立即清零。

  2. **重度饥饿减重（-1kg）**
     - 条件：饱食度处于 **重度饥饿区间（0.1～14.9）** 连续 **500 tick**。
     - 结果：体重 **-1 kg**，随后该计数器清零重新开始（若仍在重度饥饿区间则继续累积下一次）。
     - 一旦饱食度 \(\ge 15.0\) 或进入 \(=0.0\)，连续计数立即清零。
     - 注：由于 \(satiety=0.0\) 连续 100 tick 会死亡，因此该减重机制只会发生在 0.1～14.9 区间。

- **实现约定（配置字段建议）**：
  - `satiety_overfed_ticks_to_weight_gain`：过量增重所需连续 tick（当前约定 500）。
  - `satiety_severe_hunger_ticks_to_weight_loss`：重度饥饿减重所需连续 tick（当前约定 500）。

#### 6.1.2 饮水（Thirst）

- **名称与字段**：
  - 展示名：**饮水**
  - 英文字段建议：`thirst`
  - 数值范围：`0.0 ～ 100.0`，保留一位小数（不设“过量饮水”上限）。

- **基础变化规则**：
  - 与饱食类似，饮水度随 tick 消耗，但节奏更慢。
  - **每 2 个 tick 固定消耗**：每经过 **2 个 tick**，饮水度固定 **-1.0**（可在配置中按全局或区域系数调整）。
  - 不执行会推动时间前进的动作时，不结算 tick，自然也不结算饮水度消耗。
  - 饮水度不会自然回升，只能通过饮用水类物品或特殊手段恢复。

- **饮水区间与效果**：

  1. **正常**：`60.0 ～ 100.0`
     - 无任何负面效果。

  2. **渴了**：`0.1 ～ 59.9`
     - **内力自然恢复值 ×0.5**。
     - 体力自然恢复不受饮水度直接影响（仍只受呼吸、凝气、饱食等影响）。

  3. **缺水**：`= 0.0`
     - 进入缺水状态时，开始计数一个“缺水倒计时”，以 tick 为单位。
     - 若在缺水状态下连续 **500 个 tick** 未补充饮水（饮水度始终为 0.0），则角色因严重脱水**死亡**，触发正常死亡惩罚流程。
     - 缺水状态下，角色**无法与任何可交互 NPC 进行交互**（包括对话、交易、任务交付等），直到饮水度恢复到 `>0.0`。

- **实现约定（配置字段建议）**：
  - `thirst_max`：饮水度上限（当前约定 100.0）。
  - `thirst_tick_decay_interval`：饮水度结算间隔所需 tick 数（当前约定 2）。
  - `thirst_tick_decay_amount`：每次结算饮水度固定消耗值（当前约定 1.0）。
  - `thirst_normal_min`：正常区间下界（当前约定 60.0）。
  - `thirst_death_ticks`：缺水状态下连续存活 tick 数上限（当前约定 500）。

#### 6.1.3 体力（Stamina）

- **名称与字段**：
  - 展示名：**体力**
  - 英文字段建议：`stamina`
  - 数值范围：`0.0 ～ 100.0`，保留一位小数。

- **含义**：
  - 表示角色当前可用于**体力型行动**（移动、生产、采集等）的即时体能储备。
  - 与基础属性「呼吸」、生存系技能「凝气」「耐力」等决定的“长期体质”区分；体力只关心“此刻还能再干多少活”。

- **自然恢复**：
  - 一切会推动时间前进的行为（移动一格、执行一次战斗/生产/操作等）统称为 **1 个 tick**。
  - 每经过 **1 个 tick**，体力进行一次自然恢复结算。
  - 平时状态下体力的**基础自然恢复值**为：
    \[
    R_{\text{stamina,base,normal}} = 0.5
    \]
  - 实际每 tick 体力恢复量按照「5.1 衍生属性」中的“体力恢复速度”公式计算，即：
    \[
    R_{\text{stamina}}
    = \bigl(R_{\text{stamina,base,normal}} + c_{\text{呼吸}} \cdot S_{\text{呼吸}}\bigr)
    \times (1 + P_{\text{凝气}})
    \]
    其中各常数从配置表与技能表读取。本节只约定“体力恢复使用与 5.1 同一条体力恢复公式”，不写死具体常数。
  - 恢复后体力不会超过上限 `stamina_max`（当前约定 `100.0`）。

- **休息 / 睡眠恢复**：
  - 角色进入**休息或睡眠状态**时，改用更高的基础恢复值：
    \[
    R_{\text{stamina,base,rest}} = 5.0
    \]
  - 实际恢复量仍使用 5.1 的体力恢复公式，只是将 `R_base` 切换为 `R_{\text{stamina,base,rest}}`。
  - 典型效果：在体力耗尽（`0.0`）情况下，连续约 **20 tick** 的高质量睡眠即可将体力恢复至接近满值（不考虑其他修正）。

- **消耗来源**：
  - **生产 / 采集类行为**：如采集、挖矿、伐木、锻造、炼药、纺织等，每次操作按动作配置表中的 `stamina_cost` 扣除体力。
  - **移动（超重时）**：当角色负重超过负重上限时，移动开始消耗体力（见下“超重移动与体力消耗”）。
  - **其他特殊动作**：部分非生产行为（如强行破坏机关、搬运重物等）可在动作定义中配置 `stamina_cost`，执行时统一从体力中扣除。
  - 普通战斗技能默认不消耗体力，仅消耗气劲/内力；若某些技能需要兼耗体力，则在技能配置中显式声明。

- **生产 / 采集行为的体力消耗（建议基准）**：
  - 为便于落表，可为各类生产/采集行为设定建议基准（可在数值阶段微调）：
    - 采集（灌木、草丛等）/钓鱼：约 `3.0` 体力/次。
    - 挖矿 / 伐木 / 狩猎：约 `4.0 ～ 5.0` 体力/次。
    - 锻造 / 炼药 / 纺织 / 制造：约 `2.0 ～ 3.0` 体力/次。
  - 实现时，各行为在对应配置表中以 `stamina_cost` 字段记录实际消耗；本节给出的仅为推荐区间。

- **超重移动与体力消耗**：
  - 记当前实际负重为 \(W\)、负重上限为 \(W_{\max}\)。
  - **未超重**（\(W \le W_{\max}\)）：
    - 移动不消耗体力，仅消耗 1 tick 时间。
  - **轻度～中度超重（可移动）**：
    - 当
      \[
      W_{\max} < W \le 1.5 \times W_{\max}
      \]
      角色仍可移动，但每移动一格额外消耗体力。
    - 每格移动的体力消耗：
      \[
      C_{\text{stamina,move}}
      = k \cdot \bigl(\tfrac{W}{W_{\max}} - 1\bigr)
      \]
      其中 \(k\) 从配置表字段 `move_energy_overweight_k` 读取（当前约定 `20`）。计算结果在扣除前保留 1 位小数，体力不能被扣到 0 以下。
  - **严重超重（无法移动）**：
    - 当
      \[
      W > 1.5 \times W_{\max}
      \]
      角色无法移动，因此不会触发移动体力消耗，只能通过丢弃物品或特殊手段降低负重，直至 \(W \le 1.5 \times W_{\max}\)。

- **体力为 0 与极限疲劳倒计时**：
  - 当体力第一次降至 `0.0` 时，进入“**极限疲劳**”状态，并开始以 tick 为单位计数。
  - 在体力为 `0.0` 且未昏迷的这段时间内：
    - 角色仍可执行部分基础行为（如有限移动、查看界面、进食/饮水等），
    - 但禁止执行显式标记为“需要体力（`stamina_cost > 0`）”的高消耗动作（如生产/采集等），与重度饥饿对“消耗体力/精力动作”的限制保持一致。
  - 若在体力维持 `0.0` 的前提下，连续经过 **50 个 tick** 仍未恢复到 `> 0.0`：
    - 角色因极度疲劳**强制昏迷**，进入短暂失去行动能力的状态（昏迷持续时间与恢复方式在睡眠/疲劳系统中细化）。
  - 若在 50 tick 内通过自然恢复或其他手段使体力恢复到 `> 0.0`：
    - 立即终止本次极限疲劳倒计时，本次不进入昏迷；下次体力再次降至 `0.0` 时重新开始计数。

- **实现约定（配置字段建议）**：
  - `stamina_max`：体力上限（当前约定 `100.0`）。
  - `stamina_tick_regen_base`：平时每 tick 体力基础自然恢复值（当前约定 `0.5`）。
  - `stamina_rest_tick_regen_base`：休息/睡眠状态下，每 tick 体力基础自然恢复值（当前约定 `5.0`）。
  - `stamina_zero_ticks_to_coma`：体力为 `0.0` 时触发强制昏迷前允许的连续 tick 数（当前约定 `50`）。

#### 6.1.4 精力（Energy）

- **名称与字段**：
  - 展示名：**精力**
  - 英文字段建议：`energy`
  - 数值范围：`0.0 ～ energy_max`，默认 `energy_max = 100.0`，保留一位小数。

- **含义与定位**：
  - 精力用于刻画角色在**自修、学习、思考**等长期精神活动中的即时精神资源。
  - 精力与体力、内力完全独立：体力偏“肉体劳动”、内力偏“战斗与内功运用”，精力偏“专注与学习”；三者在规则上互不共用上限或自然恢复。

- **上限与提升**：
  - 默认精力上限为 `100.0`。
  - 精力上限可通过少量特殊手段（如被动技能、道具、剧情奖励等）提升；具体来源与数值在对应技能/道具/剧情条目中单独说明。
  - 上限提升仅提高 `energy_max`，不会直接恢复当前精力值。

- **恢复规则**：
  - 精力**不会自然恢复**：不随 tick 推进自动回升。
  - 精力恢复完全由各具体系统单独定义，例如：
    - 睡眠 / 特定休息行为；
    - 药物、料理、特殊消耗品；
    - 被动技能或特殊系能力。
  - 对应实现约定见 5.1 表中「精力恢复速度」条目：精力恢复不直接受基础属性影响，而由睡眠、道具、被动技能等在各自条目中独立给出恢复量与公式。

- **消耗来源**：
  - **自修学习技能**（已定义）：
    - 见 5.8「专注属性细则」：自修每 tick 固定消耗 `study_tick_energy_cost` 点精力（当前约定为 1），并按“精力→潜能”的效率上限结算。
  - **技能与特殊效果**（接口预留）：
    - 某些技能或状态可以在发动或结算时**直接扣除精力值**，具体由技能配置中的精力消耗字段（如 `energy_cost`、`energy_damage` 等）定义。
    - 当某技能或效果在**单次结算中**使角色当前精力从 >0 直接降到 `<0`（即一次性将精力扣至 0 以下）时，视为该角色因精神崩溃或意识被彻底击穿而**立即死亡**，触发正常死亡惩罚流程。
    - 具体哪些技能/效果具有这种“致死型精力伤害”，在各自技能条目中明确标注。
  - 其他可能消耗精力的行为（如高压谈判、特殊交互等）可在各自系统设计时，通过同一精力消耗接口接入。

- **与其他系统的协同**：
  - 饱食系统的“重度饥饿 / 极限饥饿”已约定：禁止一切会**消耗体力或精力**的动作；精力相关消耗统一归在这一规则之下。
  - 专注属性（见 5.8）仅影响“精力 → 潜能”的转换效率与自修时的潜能上限，不直接改变精力的上限或恢复速度。

### 6.2 战斗核心

- **节奏**：风来的西林式回合刻制——玩家每行动一次，所有敌人各行动一次。
- **方向与范围**：8 向移动与攻击；普攻为正前方 1 格单体。
- **耗回合**：移动一格 / 普攻 / 使用技能 / 使用道具 / 等待；不耗回合动作后续定义。
- **气劲**：技能消耗气劲；每回合自然恢复，可药品与特定行为恢复。
- **出力大小（1～12 成）**：伤害与消耗均与成数挂钩（\(F_{\text{出力}} = \text{成数}/10\)，实际消耗 = 基础消耗 × 成数/10）。**实现约定**：各攻击技能的出力相关配置（如默认成数、气劲/体力基础消耗）在**技能配置**中按技能/招式配置；玩家**打开技能栏并点击某攻击技能时**，可在 UI 中调整该技能的出力大小（1～12 成），该设定在发动该技能时生效。
- **速度与先手、连击**
  - **速度值**：战斗速度由装备的轻功提供 \(V_{\text{base}}\)（未装备轻功时为 1），身手每 +1 点使速度 +0.5%，与装备等加成相加；先算浮点再**向下取整**，用于先手、连击与命中率。速度**仅影响**战斗中的出手顺序与连击，不影响每回合可移动的格数。详见 5.7。
  - **先手判定**：每次主动攻击时，先比较双方速度（取整后的值），速度高者先出手。双方速度相同时，视为同时触发，同时开始计算减伤并结算伤害。
  - **连击**：若一方速度**大于等于**另一方两倍，则速度高者产生连击，本回合内连打两次；大于等于三倍则连打三次；大于等于四倍则连打四次（更高倍数可后续细化）。连击均视为**同一回合内**产生的动作；速度低的一方在对方连击全部打完后再还手一次，本回合仅此一次。
  - **连击判定与消耗**：连击的每一击均**单独**做命中、招架与伤害判定；每一击**单独扣除**一次气劲（及体力，若有），默认按正常消耗。可通过某些手段减免因连击产生的额外内力消耗（该接口为后续扩展预留）。

## 七、命中、招架与伤害流程（普攻）

1. **命中率**：由攻防双方**速度值**（取整后）参与，**同速时**基础命中率在 80%～85%；再加速度或命中词条也**无法达到 100%**（有上界）。公式如下（常数均从配置表读取，字段见下）：
   - **速度比**：\(r = V_{\text{攻}} / \max(V_{\text{防}}, \varepsilon)\)，\(V\) 为取整后的速度，\(\varepsilon\) 防除零。
   - **基础命中率 \(P_{\text{base}}\)**：
     - **r ≥ 1**：\(P_{\text{base}} = P_{\text{eq}} + (P_{\text{base}}^{\max} - P_{\text{eq}}) \cdot \dfrac{\ln r}{L + \ln r}\)，再 clamp 到 \([P_{\text{eq}}, P_{\text{base}}^{\max}]\)。\(P_{\text{base}}^{\max} < 1\)，即再加速度也无法到 100%。
     - **r < 1**（攻方更慢，惩罚）：\(P_{\text{base}} = P_{\text{eq}} \cdot r + P_{\min} \cdot (1 - r)\)，再 clamp 到 \([P_{\min}, P_{\text{eq}}]\)。
   - **词条命中加成** \(B_{\text{命中}}\)：装备等提供的命中加成（小数）与 \(P_{\text{base}}\) 相加。
   - **最终命中率**：\(P_{\text{命中}} = \min(P_{\text{base}} + B_{\text{命中}},\ C_{\text{命中}})\)，\(C_{\text{命中}} < 1\)（如 0.99），即再加多少命中修正也无法到 100%。
   - **配置字段映射（避免歧义）**：
     - \(P_{\text{eq}}\)（同速基础命中率）← `hit_base_at_equal_speed`
     - \(P_{\text{base}}^{\max}\)（仅速度的基础上限）← `hit_base_max`
     - \(P_{\min}\)（攻方更慢时下限）← `hit_base_min`
     - \(C_{\text{命中}}\)（最终上限）← `hit_final_cap`
     - \(L\)（曲线常数）← `hit_curve_L`
     - \(\varepsilon\)（防除零）← `hit_epsilon`
2. **招架**：命中后按防方招架几率（招架技能 + 装备）判定。成功则本次不造成伤害；失败则按招架技能卸力比例减伤。
3. **减伤链**（卸力之后依次）：内功减伤 → 防具减伤 → 基础防御（柔韧动态%）→ 伤害类型微调系数（按命中部位与伤害类型）；链内保留小数传递，仅最终扣血与损毁增加值向下取整到整数。详见 5.6。
   - **内功**：仅在有“内力护体”时生效；百分比；被减部分记到护体盾上。
   - **防具**：百分比。
   - **基础防御**：由柔韧决定，动态公式 \(DR = S_{\text{柔}}/(S_{\text{柔}}+3D)\)。
   - **伤害类型微调**：命中部位与伤害类型查表 \(M_{\text{部位,类型}}\)，最终伤害 = 上步输出 × \(M\)。
4. **原始伤害**（减伤前）：武器基础伤害 ×（招式+内力威力系数）× 实战经验增伤 × 环境增伤 × 暴击系数 × 内力催发系数（1～12 成，仅多耗气劲）。暴击按暴击率属性判定，成功则乘固定系数。

## 八、身体部位与状态效果

- **部位**：头、胸、腹、左手、右手、左脚、右脚；命中部位按权重随机（躯干高、头低、四肢中等）。
- **效果**：
  - **头**：本回合眩晕无法行动；会自己恢复。
  - **胸**：只算伤害。
  - **腹**：一次性降低饱食度与饮水度。
  - **手**：命中/招架下降会自己恢复；有几率骨折，骨折与脚同，须特殊手段解除。
  - **脚**：暂时无法移动会自己恢复；有几率骨折 → 脚部技能失效、移动一格需两回合，须特殊手段解除。
- **肢体损毁**：每个肢体有**损毁值**；损毁值满后该肢体标记为损毁、失去功能，消耗品无法降低；可通过**手术用品**暂时扣除部分损毁值上限以恢复功能，**手术**技能等级影响可恢复的损毁值上限比例。
- **各部位损毁值上限**：

  | 部位 | 损毁值上限 | 说明 |
  |------|-----------|------|
  | **头** | 50 | 头部耐受度最低，少量损毁积累即可导致功能丧失。 |
  | **胸** | 100 | 标准上限，有肌肉与肋骨支撑，较能承受累积伤害。 |
  | **腹** | 80 | 软组织与空腔器官集中，耐受度低于胸部。 |
  | **左手/右手** | 100 | 标准上限；骨折等会更早触发，但损毁积累速度较快。 |
  | **左脚/右脚** | 100 | 标准上限；骨折与行动失能早于损毁满值触发。 |

#### 8.2.1 生活技能熟练度规则

- 所有生活技能采用**“用什么升什么”**的熟练度系统，使用 50000 次视为满熟练度（100%）。
- 熟练度带来的效果一律按**乘法系数**作用在各自的基础几率上：
  - 实际几率 = 基础几率 × (1 + 熟练度对应的额外比例)
  - 例如基础高品质几率为 10%，额外 +30% 时，最终几率 = 10% × (1 + 0.3) = 13%。
- **熟练度上升速度**：某些手段可提升或降低熟练度上涨速度；该接口需为后续扩展预留（实现时预留可插拔的倍率/修正入口，不在此文档枚举具体手段）。

- **钓鱼熟练度**：每 +1% 熟练度，
  - 钓鱼成功率：基础成功率 × (1 + 0.3%)；
  - 提升品质几率：基础提升品质几率 × (1 + 1%)；
  - 稀有鱼出现几率：基础稀有几率 × (1 + 0.15%)。

- **挖矿熟练度**：每 +1% 熟练度，
  - 挖矿成功率：基础成功率 × (1 + 0.3%)；
  - 挖到高品质矿石几率：基础高品质几率 × (1 + 1%)；
  - 稀有矿/副产品几率：基础稀有几率 × (1 + 0.15%)。

- **伐木熟练度**：每 +1% 熟练度，
  - 砍伐成功率：基础成功率 × (1 + 0.3%)；
  - 高品质木材几率：基础高品质几率 × (1 + 1%)；
  - 稀有木材/副产品几率：基础稀有几率 × (1 + 0.15%)。

- **狩猎熟练度**：每 +1% 熟练度，
  - 狩猎成功率：基础成功率 × (1 + 0.3%)；
  - 高品质猎物/素材几率：基础高品质几率 × (1 + 1%)；
  - 稀有猎物/稀有掉落几率：基础稀有几率 × (1 + 0.15%)。

- **采集熟练度**：每 +1% 熟练度，
  - 采集成功率：基础成功率 × (1 + 0.3%)；
  - 高品质草药/果实几率：基础高品质几率 × (1 + 1%)；
  - 稀有植物/稀有杂物几率：基础稀有几率 × (1 + 0.15%)。

- **畜牧熟练度**：每 +1% 熟练度，
  - 畜产品产出成功率：基础成功率 × (1 + 0.3%)；
  - 畜产品高品质几率：基础高品质几率 × (1 + 1%)；
  - 不额外提高稀有动物/稀有产物几率（畜牧偏稳定量产）。

- **种植熟练度**：
  - 仅在**收获作物时**提升熟练度（种下不涨，仅收获涨）。
  - 每 +1% 熟练度：
    - 高品质作物几率：基础高品质几率 × (1 + 1%)；
    - 双倍产物几率：基础双倍几率 × (1 + 1%)。

#### 8.3.1 战斗技能通用规则

- **技能等级与潜能消耗**
  - 所有战斗技能的等级上限为 **1000 级**，使用**潜能**进行升级。
  - 单一技能从 1 级升至 1000 级，累计约需 **1500 万潜能**。
  - 从 **400 级开始**，每提升 1 级所需的潜能成本显著上升，体现后期修行的瓶颈感。

- **招式与熟练度**
  - 每个战斗技能包含若干个**招式**，每个招式有独立的**熟练度计数**。
  - 通用约定：同一招式**命中目标**时，其“使用次数” +1；打空、被闪避或被招架不计入。
  - 当某招式为多段攻击时，只要本次释放中**至少有一段命中**，该招式的使用次数仍只 +1。
  - 招式熟练度上限统一为 **50000 次使用 = 100% 熟练度**。
  - 熟练度只作为“在当前技能等级基础上的额外加成”，不会改变技能等级本身。
  - **熟练度上升速度**：与生活技能一致，某些手段可提升或降低招式熟练度上涨速度；该接口需为后续扩展预留（实现时预留可插拔的倍率/修正入口，不在此文档枚举具体手段）。

- **威力组成（等级 + 熟练度）**
  - 每个战斗技能都有一条独立的“等级 → 基础威力系数”曲线，用于定义该技能在不同等级下的**基础威力**。
  - 每个招式在计算伤害时，先取所属技能的基础威力系数，再乘以一个**线性的熟练度加成**：
    - 记 \(L\) 为技能等级，\(P\) 为该招式的使用次数（0～50000），\(R = \min(P / 50000, 1)\) 为熟练度比例。
    - 记 \(Base(L)\) 为等级 \(L\) 对应的基础威力系数，则单个招式的最终威力系数为：
      \[
      Final(L, P) = Base(L) \times (1 + R)
      \]
    - 当熟练度达到 100% 时，招式威力在等级基础上获得 **+100%（×2）** 的线性加成。
  - 各战斗技能的具体 \(Base(L)\) 曲线可不同，在对应技能条目中单独给出；本节仅约定通用结构。

- **装备到肢体与出招循环**
  - 战斗技能必须**装备在肢体上**才能在战斗中使用，当前可装备的肢体包括：左手、右手、左脚、右脚。
  - 每条肢体同时可装备 **1 个主动战斗技能 + 1 个招架技能**。
  - 每个战斗技能拥有若干个**招式槽**；玩家从已解锁的招式中选择装入槽位并设定顺序。
  - 每条装备了主动技能的肢体，都会维护一条**独立的招式循环序列**（只在该肢已装备的招式中循环）。
  - 玩家可以为四个肢体设定**出招优先顺序**（如：左手 → 右手 → 左脚 → 右脚）；在一次攻击中：
    - 系统在“未损毁且当前有可用招式”的肢体中，按优先顺序选择一条肢体作为本次出招肢体。
    - 若该肢体已损毁，直接跳到下一条肢体；若所有相关肢体均不能出招，则本次攻击视为无效。
  - 当某肢体被选中出招时：
    - 按该肢体的招式循环，取出**当前轮到的招式**尝试释放。
    - 如果因为动作标签不符、肢体状态异常等原因导致招式**无法释放**，本回合动作白费，但该肢体的招式指针仍视为“已轮到过一次”，在下次轮到该肢体时会进入下一招。

- **动作标签与肢体状态**
  - 每个招式都要求装备肢体具备某些**动作标签**（例如“拳击”“勾击”“踢击”等）。
  - 若肢体不具备招式所需的动作标签，则该招式既不能装入该肢的槽位，也无法在战斗中释放。
  - 肢体的动作标签可以被一些系统实时增减（例如通过特殊装备、buff、或永久改造等方式）；标签变化**即时生效**。
  - 若玩家在某肢上预先装备了需要特定标签的招式，而战斗中该肢失去了对应标签，则那些招式视为暂时不可用。
  - 肢体一旦被判定为“损毁”，则视为该肢无法再使用任何主动或招架招式，直到通过特殊手段恢复。

#### 8.3.2 「基本拳脚」示例

作为徒手类战斗技能的代表，「基本拳脚」使用上述通用规则，并在此基础上补充以下具体设定：

- **装备与作用肢体**
  - 「基本拳脚」可以装备在左手、右手、左脚、右脚任意肢体上，只要该肢体未损毁且具备至少一个所属招式所需的动作标签。
  - 一个角色可以在多条肢体上同时装备「基本拳脚」，各自维护独立的招式循环。

- **招式与解锁等级**
  - 「基本拳脚」下包含四个基础招式：
    - **直拳**：技能等级 **1 级** 解锁。
    - **前踢**：技能等级 **1 级** 解锁。
    - **勾拳**：技能等级 **50 级** 解锁。
    - **鞭腿**：技能等级 **50 级** 解锁。
  - 各招式的具体数值定位（轻攻/重攻/控制等）、气劲消耗与命中部位权重将在后续细化。

- **招式槽与初始配置**
  - 角色在学会「基本拳脚」时，默认已掌握直拳与前踢，对应**立即解锁 2 个招式槽**。
  - 当技能等级达到 50 级并学会勾拳与鞭腿时，每学会一个新招式**额外解锁 1 个招式槽**，最终该技能共计 **4 个槽位**。
  - UI 中仅显示当前已解锁的槽位，未解锁的槽位不显示，避免在早期暴露未来结构。
  - 玩家可以在每个槽内自由选择已解锁的招式，并为装备了「基本拳脚」的每条肢体分别设定槽内招式顺序；不同肢体可以有完全不同的出招顺序。

- **等级与基础威力曲线**
  - 「基本拳脚」的基础威力系数以 1 级为基准，记为 \(Base(1)=10\)，并随技能等级 \(L\) 提升；1000 级时约为 1 级的 10 倍，即 \(Base(1000)\approx 100\)。
  - 为体现“前期提升明显、后期逐渐变缓”，定义一个相对 1 级的倍数函数 \(M(L)\)（1 级时 \(M(1)=1\)），分三段线性插值：
    - **1 ≤ L ≤ 200**：从 1 倍平滑提升到约 4 倍：
      \[
      M(L) = 1 + \frac{3}{199}\,(L - 1)
      \]
    - **200 < L ≤ 400**：从约 4 倍提升到约 7 倍：
      \[
      M(L) = 4 + \frac{3}{200}\,(L - 200)
      \]
    - **400 < L ≤ 1000**：从约 7 倍提升到约 10 倍：
      \[
      M(L) = 7 + \frac{3}{600}\,(L - 400)
      \]
  - 对应的基础威力系数为：
    \[
    Base(L) = 10 \times M(L)
    \]
  - 在原始伤害公式中，「基本拳脚」各招式的“招式威力系数”部分即取自该 \(Base(L)\) 再结合熟练度加成。

- **熟练度与最终威力**
  - 「基本拳脚」下的每个招式都拥有独立的熟练度计数：
    - 每当该招式成功命中目标时，其“使用次数” +1。
    - 打空、被闪避或被招架不增加使用次数。
    - 若为多段攻击，只要本次释放中至少 1 段命中，即视为本招式使用 1 次。
  - 记 \(P\) 为某招式的使用次数（0～50000），则熟练度比例为：
    \[
    R = \min\left(\frac{P}{50000},\,1\right)
    \]
  - 该招式在等级 \(L\) 与使用次数 \(P\) 下的最终威力系数为：
    \[
    Final(L, P) = Base(L) \times (1 + R)
    \]
  - 当该招式熟练度达到 100%（使用次数 ≥ 50000）时，其威力在等级基础上获得 **+100%（×2）** 的线性加成。
  - 「基本拳脚」整体的“招式熟练度 100%”仅在该技能下**全部招式**熟练度均达 100% 时才视为达成，用于后续可能的额外奖励或解锁判定。

- **出招偏好与循环细节**
  - 玩家为四条肢体设定出招优先顺序后，每次普攻/使用「基本拳脚」时：
    - 系统在“未损毁且当前装备有可释放招式”的肢体中，按优先顺序选择一条作为本次出招肢体。
    - 被选中的肢体按其自身的招式循环，取出当前轮到的招式尝试释放。
  - 若本次轮到的招式因缺少动作标签、肢体状态异常等原因无法释放：
    - 本回合视为已尝试出手但**没有造成任何效果**。
    - 该肢体的招式指针仍视为“已经轮到过这一招”，在下次该肢体出手时会从下一招开始。

- **永久属性加成（后天筋骨与后天柔韧）**
  - 「基本拳脚」作为长期修行的基础功法，会在高等级时为角色提供永久的身体素质加成：
    - 每当「基本拳脚」等级累计提升 **200 级**，角色获得：
      - **后天筋骨 +1**
      - **后天柔韧 +1**
  - 该加成为全局永久加成：
    - 与当前是否装备「基本拳脚」无关，一旦获得即对所有战斗与判定生效。
    - 加成计入后天部分，不会突破先天属性的 40 点上限设定，但会与先天值相加用于实际数值计算。

## 九（附）、玩家间交易系统

- **入口**：玩家与**某个指定 NPC** 对话，进行玩家间交易。
- **剧情表述**：交易码 = **柜号**；接头暗号 = **解锁口令**；最终通过**比对特征码**确认领取者身份。
- **发起流程**：与该 NPC 对话后，**缴纳一定财物** → **选择要交易的物品** → **输入交易对象的特征码** → 系统**生成一个交易码**。
- **生成交易码时**：
  - **禁止自交易**：若交易对象的特征码与发起方（当前角色）特征码一致，则**不允许生成交易码**。
  - **自动扣除**玩家选定的交易物品（从本方背包/装备中移除）。
  - **自动重新生成一份存档码**（即执行一次“玩家选择存档”并导出）；该存档码中增加**存档世代**参数，规则见「十、实现约定」存档系统之存档世代。
- **接头暗号**：每次生成交易码时，发起方需设置一个**接头暗号**（口令），作为该次交易的加解密凭证。接头暗号**至少 4 位**。
- **交易码**：为密文；**解密后包含三个信息**：
  1. 发起方特征码；
  2. 被交易对象特征码；
  3. 交易物品信息。
- **使用交易码**：在**同一 NPC** 处可选择**使用交易码**。玩家粘贴/输入交易码后，还需输入本次交易对应的**接头暗号**；系统使用接头暗号解密交易码并**比对特征码**：仅当当前角色的特征码与密文中的**被交易对象的特征码**一致时（比对通过），该玩家**获得交易物品**（物品加入本方背包等）；比对不通过则无法领取。
- **暗号输错惩罚**：使用交易码时，接头暗号连续**输错 3 次**，该交易码**直接作废**。错误次数按**本地存档**对该交易码累计计数（同一存档换设备/换浏览器后仍继承次数）。
- **兑换规则**：同一交易码对同一特征码（被交易对象）**只能成功领取一次**；兑换成功后记录该特征码与**兑换时间**。交易码**超过 1 天不允许兑换**；时效校验统一使用 **GMT+8 中国时区**。已兑换记录在**过 1 天后自动从存档中擦除**，以精简存档。
- **时间来源（时效校验）**：兑换交易码时，系统优先请求 `CurrentTime.now` 公共接口获取当前时间（例如 `https://currenttime.now/public/api/ip`）。成功则用其返回时间作为“当前时间”；若网络请求失败/超时/无法解析，则回退使用**本机电脑时间**作为“当前时间”。无论使用哪一种时间来源，时效校验均统一按 **GMT+8 中国时区**计算自然日。
- **不可撤销**：发起方**不可撤销**交易码；有效期内对方未领取则交易码过期。
- **过期处理**：超过有效期的交易码不可兑换，其中物品视为**被 NPC 吞掉**（消失，不返还发起方）。

### 存档系统（与交易系统强关联，此处集中约定）

- **定位与存放**：存档**仅存于本地**，无服务器端；与「三、死亡惩罚与装备快照」中的**保险代码**是两套东西——保险代码只含装备快照与地牢标识，用于死亡后取回装备；存档是完整游戏进度。存档为**覆盖式**，**不保留历史存档**，仅保留当前一份（实时存档与存档码均为对当前状态的覆盖写入）。
- **实时存档**：游戏采用**实时存档**。当前进度在游玩过程中持续写入本地（如浏览器本地存储，实现时定）；同一设备上再次打开游戏时，直接读取本地进度继续游玩，**无需账号密码**。
- **玩家选择存档**：玩家执行「存档」时，**仅**将**当前所有进度**用 **AES-256 加密**为**密文**（存档码）并交给玩家（复制、导出等）。不产生与实时存档不同的另一套数据，只是把同一份当前进度导出为密文；用于换设备、备份、以及后续与「玩家间交易」等需要跨存档识别的场景。载入时玩家粘贴/导入密文并输入该存档设定时的**账号与密码**，系统比对通过后解密并载入。
- **存档世代**：存档码（玩家选择存档导出的密文）的 payload 内**包含存档世代参数**（如 `saveGeneration`，实现时定字段名）。载入时：若**本地存在实时存档**，则当密文中的存档世代**早于**当前本地实时存档的世代时，**不允许载入**（不允许载入比当前存档更早的存档）；世代相等或更新则允许。若**本地不存在实时存档**（如清缓存/首次打开/换浏览器），则允许载入任意世代的存档码作为初始化。生成交易码时会自动重新生成一份存档码，此时本地存档世代递增或更新，以保证交出物品后的进度为“最新一代”。
- **存档码自动更新**：本作为 **JS 实现的 H5 网页游戏**。**每 50 tick** 自动更新一次游戏内部维护的存档码（序列化当前进度 + 加密，并推进存档世代）。**玩家手动导出**存档码时，**再以当前进度做一次**序列化与加密，将得到的密文交给玩家，因此导出的存档码为导出时刻的最新状态。**性能**：若将序列化与 AES 加密放入 **Web Worker** 或 **requestIdleCallback** 异步执行，每 50 tick 的自动更新与手动导出时的“再更新一次”对主线程与帧率影响可控，一般不严重；实现时可按存档体积与目标设备酌情调整 50 tick 间隔或关闭自动更新。
- **存档体积**：存档体积**可以控制**。可通过只序列化必要字段、使用短键名、对加密前 payload 做**压缩**（如 gzip/deflate，解密后解压再解析）等方式控制体积；实现时可根据需要设定背包/地图等数据的上限或精简策略，以兼顾体积与 50 tick 自动更新的性能。
- **密文约定**（玩家导出的存档密文）：
  - 加密前 payload 内**包含版本号**（与存档顶层 `version` 一致）及**存档世代**。
  - 加密后**需校验**（解密后对内容做完整性校验，校验方式实现时定，如摘要）。
  - 给玩家的代码为**纯密文**（如 Base64 编码，不含明文前缀）。
  - 载入密文时**必须**输入当时设定的账号与密码，比对通过后才解密并载入；且须通过存档世代校验（见上）。
- **存档顶层结构（推荐）**：解密后的存档 JSON 建议采用以下顶层 key；具体嵌套由实现定，可增删字段。
  - `version`：存档格式版本号，便于后续兼容与迁移。
  - **存档世代**（如 `saveGeneration`）：用于载入时与当前本地世代比较，禁止载入更早的存档。
  - `account` / `accountHash`：账号或账号校验信息，用于载入时与玩家输入的账号密码比对。
  - `character`：角色数据（先天/后天属性、技能等级、当前体力/内力、肢体状态等）。
  - `inventory`：背包（物品列表或按格位）。
  - `equipment`：当前装备，按槽位（如左手、右手、手套等）存储装备引用或 ID。
  - 其他：如地图进度、任务状态、生产建筑、仓库等，按需扩展。
- **与交易系统的关联**：玩家间交易依赖**特征码**标识交易双方；交易码解密后含双方特征码与交易物品信息。生成交易码时会**自动重新生成存档码**并更新**存档世代**，避免通过载入旧存档取回已交出的物品。详见「九（附）、玩家间交易系统」。

## 八（附）、敌人属性与构成（精简自主角）

敌人不沿用主角的完整属性与技能栏，仅保留参与战斗结算所需的数据；与主角共用同一套命中、招架、减伤链与部位规则，但在数值来源上做精简与直接配置。

### 敌人保留与主角一致的部分

- **肢体方案**：与玩家相同，采用**头、胸、腹、左手、右手、左脚、右脚**七部位；命中部位权重、损毁值上限、伤害类型微调系数 \(M_{\text{部位,类型}}\)、部位效果（眩晕、骨折等）与「八、身体部位与状态效果」一致，便于同一套伤害与损毁逻辑复用。**设计时需声明**：敌人的**攻击技能也装备在肢体上**（如左手、右手、左脚、右脚）；**肢体被毁则该肢体上的技能无法使用**，与主角“损毁肢体无法出招”的规则一致。**已损毁部位再受击**时损毁增加值均分到其余未损毁部位，与 **09-body-parts.md** 规则相同。

### 敌人专用配置（与主角的差异）

- **速度**：**直接配置整数**。不按身手/轻功/装备计算，每个敌人类型在配置表中给定一个**速度值** \(V\)，直接参与先手、连击与命中率公式（与玩家速度比 \(r = V_{\text{攻}}/\max(V_{\text{防}},\varepsilon)\) 等，见「七、命中、招架与伤害流程」）。

- **内功与身体减伤**：不采用主角的「柔韧 → 基础防御」动态公式。为敌人**直接设计**两类减伤系数，在配置表中为每种敌人设定：
  - **内功减伤**（可选）：可理解为“气劲护体”的百分比减伤，在减伤链中的位置与主角的防具减伤或护体层等价（具体顺序在实现时与护体/防具统一约定）。
  - **身体减伤系数**：直接配置一个**身体减伤百分比**（或等价系数），在减伤链中替代主角的「基础防御（柔韧）」一步；即玩家命中敌人后，在卸力、护体（若有）、内功减伤之后，再乘以该身体减伤系数，再乘伤害类型微调 \(M_{\text{部位,类型}}\)。

- **招架**：规则与主角一致（命中后先判定招架，成功则本次不造成伤害；失败则先按卸力比例硬减伤，再进入护体→内功/防具→身体减伤→伤害类型微调）。**敌人招架不享受柔韧加成**：仅使用**基础招架几率**与**基础卸力比例**，不再乘以 \((1 + \text{系数} \times \text{柔韧})\)。两值在敌人配置表中直接给出，可设 0 表示不招架。

- **攻击动作**：**单独为敌人设计**。敌人不使用主角的技能树与招式池；每种敌人类型在配置表中拥有自己的**攻击动作列表**（含普攻与可选技能），每条动作**装备在指定肢体上**（见上“肢体方案”），肢体损毁则无法使用该肢上的动作。**威力**：可为每条动作**直接指定一个数字范围**（如威力最小值与最大值）；**攻击时在该范围内随机取一值作为本次威力**，再参与伤害公式，即单次伤害在范围内浮动。

- **气劲条**：敌人**仅设计一条气劲条**。气劲上限、每回合自然恢复量（或每 N 回合恢复）在敌人配置表中设定；敌人释放需消耗气劲的攻击动作时，从该气劲条扣除，不足则不可使用该动作。不沿用主角的底气/行气/呼吸法体系。

- **护体**：**是否具备护体、护体减伤比例与护体容量**均可**按敌人具体设定**。若该敌人配置了护体，则与主角的底气护体在同一减伤链位置生效（卸力之后、内功/身体减伤之前），护体被击破前按配置的减伤比例与容量结算；未配置则无护体层。

### 敌人掉落池

- **设计时提前定义**：每种敌人的**掉落池**在设计该敌人时**提前定义**，明确该敌人死亡后可掉落的**物品有哪些**。
- **每项物品的爆率**：掉落池中**每一项物品的爆率均需详细声明**（如概率、权重或等价规则）；实现时按该爆率在击杀该敌人时进行掉落判定。爆率的具体含义（独立概率、互斥多选一、权重抽样等）在配置表结构或全局规则中统一约定。
- **物品来源**：掉落池中的物品**在物品表中定义**；敌人配置中通过**物品 ID**（或等价引用）指向物品表内的条目，不重复定义物品属性。物品表见项目内物品/配置相关文档。

### 敌人不包含的内容

- 无生存属性：无饱食、饮水、体力、精力、心情、定力、性能力、性别、营养、体温、体重等。
- 无基础属性与衍生：无筋骨/柔韧/呼吸/身手/专注及由此衍生的负重、学习效率等。
- 无技能栏：无生存/生活/战斗/特殊四条技能栏；仅有上述“攻击动作列表”与气劲条。
- 无装备系统：无多槽位装备与负重；内功/身体减伤、招架、速度等均在敌人配置表中直接给出。

### 实现约定（配置字段建议）

- 敌人类型表（或等价配置）建议字段：`speed`（速度，整数）、`parry_rate`（基础招架几率）、`parry_damage_reduce`（基础卸力比例）、`inner_damage_reduce`（内功减伤，如 0～1）、`body_damage_reduce`（身体减伤系数，如 0～1）、`qi_max`（气劲上限）、`qi_regen_per_turn`（每回合气劲恢复）；`has_shield`（是否有护体）、`shield_damage_reduce`（护体减伤比例）、`shield_capacity`（护体容量）；攻击动作列表（每条动作需指定**装备肢体**、**威力范围**（如 `power_min`/`power_max`，攻击时在范围内随机取整或浮点）、伤害类型、气劲消耗、范围等）；**掉落池**（`loot_table` 或等价）：列表项为（**物品 ID**，**爆率**），物品 ID 引用物品表，爆率按全局约定填写（如 0～1 独立概率、或权重值等），设计时须为每项掉落明确声明爆率。肢体与部位数据与主角共用同一套部位定义与损毁值上限表。

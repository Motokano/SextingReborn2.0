## 九、技能系统

技能分为四大类型：**生存 / 生活 / 战斗 / 特殊**。

**实现约定（技能 ID）**：本作技能在代码与配置中通过**技能 ID** 引用；以下为已确定 ID，后续技能在细化时陆续补充。负重公式等引用「力量」等级时，使用 ID `survival_strength`；角色未习得该技能时，等级视为 0。

|----------------|---------|

| 凝气 | `survival_ningqi` |
| 搜索 | `survival_search` |
| 语言 | `survival_language` |

### 8.1 生存系（已定义）

| 技能 | 说明 |
|------|------|
| **语言** | 初始听不懂 NPC、看不懂物品描述；等级提升后逐渐能看懂、听懂更多内容。 |
| **耐力** | 生产活动消耗体力；等级越高，单次生产消耗体力越少。 |
| **健康** | 降低骨折概率；减少日常饱食度、饮水度消耗。 |
| **免疫** | 降低所有药物副作用；提高治愈速度；延长止痛药持续时间。 |
| **代谢** | 增强料理类物品效果。 |
| **力量** | 提升负重能力；满级 100 级时，左手、右手武器槽中已装备的武器不计入角色负重（见 5.5.4）。 |
| **活力** | 降低身体某部位受伤时触发出血的概率。 |
| **抗压** | 手部受伤时，减少命中率、招架率下降的幅度。 |
| **搜索** | 地牢开箱为逐步搜索（物品信息一件件出现），完全辨认即视为搜索完毕；通过搜索未搜索过的箱子升级；等级越高搜索所需时间越短。 |
| **手术** | 肢体损毁值满后可用手术用品暂时降低损毁值上限并恢复功能；技能等级影响可恢复的损毁值上限比例。 |
| **睡眠** | 靠睡觉升级；等级越高，恢复体力所需时间越少。 |
| **凝气** | 等级越高，每回合恢复的体力、底气越多。 |
| **耐候** | 减少因极端气候产生的体温变化幅度，减轻低温（及未来可能加入的高温）带来的额外消耗。 |

#### 8.2.1 生活技能熟练度规则

- 所有生活技能采用**“用什么升什么”**的熟练度系统，使用 5000000 次（五百万次）视为满熟练度（100%）。
- 熟练度带来的效果一律按**乘法系数**作用在各自的基础几率上：
  - 实际几率 = 基础几率 × (1 + 熟练度对应的额外比例)
  - 例如基础高品质几率为 10%，额外 +30% 时，最终几率 = 10% × (1 + 0.3) = 13%。
- **熟练度上升速度**：某些手段可提升或降低熟练度上涨速度；该接口需为后续扩展预留（实现时预留可插拔的倍率/修正入口，不在此文档枚举具体手段）。

- **钓鱼熟练度**：每 +1% 熟练度，
  - 钓鱼成功率：基础成功率 × (1 + 0.3%)；
  - 提升品质几率：基础提升品质几率 × (1 + 1%)；
  - 稀有鱼出现几率：基础稀有几率 × (1 + 0.15%)。

- **挖矿熟练度**：每 +1% 熟练度，
  - 挖矿成功率：基础成功率 × (1 + 0.3%)；
  - 挖到高品质矿石几率：基础高品质几率 × (1 + 1%)；
  - 稀有矿/副产品几率：基础稀有几率 × (1 + 0.15%)。

- **伐木熟练度**：每 +1% 熟练度，
  - 砍伐成功率：基础成功率 × (1 + 0.3%)；
  - 高品质木材几率：基础高品质几率 × (1 + 1%)；
  - 稀有木材/副产品几率：基础稀有几率 × (1 + 0.15%)。

- **狩猎熟练度**：每 +1% 熟练度，
  - 狩猎成功率：基础成功率 × (1 + 0.3%)；
  - 高品质猎物/素材几率：基础高品质几率 × (1 + 1%)；
  - 稀有猎物/稀有掉落几率：基础稀有几率 × (1 + 0.15%)。

- **采集熟练度**：每 +1% 熟练度，
  - 采集成功率：基础成功率 × (1 + 0.3%)；
  - 高品质草药/果实几率：基础高品质几率 × (1 + 1%)；
  - 稀有植物/稀有杂物几率：基础稀有几率 × (1 + 0.15%)。

- **畜牧熟练度**：每 +1% 熟练度，
  - 畜产品产出成功率：基础成功率 × (1 + 0.3%)；
  - 畜产品高品质几率：基础高品质几率 × (1 + 1%)；
  - 不额外提高稀有动物/稀有产物几率（畜牧偏稳定量产）。

- **种植熟练度**：
  - 仅在**收获作物时**提升熟练度（种下不涨，仅收获涨）。
  - 每 +1% 熟练度：
    - 高品质作物几率：基础高品质几率 × (1 + 1%)；
    - 双倍产物几率：基础双倍几率 × (1 + 1%)。

#### 8.2.2 采集技能与采集点机制

采集在**灌木丛**与**草丛**两类采集点上进行；产出由各采集点对应的**产出表**决定，产出表可区分药草、野果、杂物等，具体物品种类在表中配置（见下「产出表」）。地图上通过格子元素 id 引用采集点类型，逻辑根据 id 选择对应产出表与规则。

**采集技能（仅看熟练度）**：

- **采集成功率**：仅与**采集熟练度**相关。**实际成功率 = base + base × (熟练度% × 0.003)**，其中 base 为采集点基础采集成功率（如 0.6），熟练度% 为当前采集熟练度百分比（0～100）；不设上限，若结果大于 1 由实现夹紧。
- **熟练度与五百万次满**：与 8.2.1 生活技能熟练度为**同一套**；每成功采集 1 次计 1 次，满 **5000000 次（五百万次）** = 熟练度 100%。
- **每次采完**：每次采集完成（成功率判定通过并完成产出结算后）熟练度计数 +1（与五百万次满一致）。

**采集技能统一流程**（灌木丛、草丛共用）：

1. **前置条件**：当前格为采集点（元素 id 为 `gathering_bush` 或 `gathering_grass`）；角色**背包未满**且**体力 > 0**（背包已满或体力为空时不可采集，不扣体力、不推进 tick）。
2. **成功率判定**：每个采集点类型有各自的**基础采集成功率** base（在采集点表中配置）；**实际成功率 = base + base × (熟练度% × 0.003)**（熟练度% 为当前采集熟练度百分比）。先进行本次采集成功率判定；**只有判定成功后才进入步骤 3 的产出判定**。**判定失败**：本次无产出，**仍消耗 2 体力并推进 1 tick**。
3. **产出**：成功率判定通过后，**先**按产出表行**权重**随机抽取 **1 行**得到物品 id 与基础品质档位，**再**根据**采集熟练度**做一次**品质上修**：**熟练度每高 5%，有 5% 概率将本次品质上修一档**（品质 1～6 = 粗糙～传说，上修后最高到传说/6 档）。即熟练度 5% 时 5% 概率上修一档、熟练度 10% 时 10% 概率上修一档，依此类推；上修仅一次，不连续 roll。
4. **体力**：每次采集（无论成功失败）消耗 **2** 点体力（与 06 生存一致，可从配置读取）。
5. **工具**：不强制工具，徒手可采；工具为可选。有工具时同时加成**成功率**与**高品质/稀有概率**，在**工具表**中增加对应字段（如 `gathering_success_bonus`、`gathering_quality_bonus` 等），具体数值在工具表配置。
6. **熟练度**：采集成功即熟练度计数 +1（与 8.2.1 同一套，五百万次满），熟练度参与成功率、品质上修及 8.2.1 中高品质/稀有几率的计算。

**可采次数与刷新**：灌木丛与草丛均**不设采空**，可一直采；唯一限制为**背包已满**或**体力为空**时不可采集。

**挂机**：采集为可挂机动作（见六、生存）。挂机时**每 1 tick 执行一次**「成功率判定 → 成功则产出表抽取」；仅当**背包已满**、**体力为空**或**玩家手动停止**时结束挂机。

**采集点类型与地图调用**：

| 采集点 | 元素 id（地图格子引用） | 展示名 | 产出表 |
|--------|-------------------------|--------|--------|
| 灌木丛 | `gathering_bush`        | 灌木丛 | 灌木丛产出表（单独配置） |
| 草丛   | `gathering_grass`       | 草丛   | 草丛产出表（单独配置）   |

- **触发方式**：玩家**站在该格执行「采集」**时，系统根据当前格元素 id 判定为灌木丛或草丛，并走对应产出表与上述流程。
- **实现约定**：地图数据中，若某格为采集点，则该格元素类型/实体 id 为 `gathering_bush` 或 `gathering_grass`；配置表（如采集点表、地图格子表）中通过该 id 关联到对应产出表 id，便于扩展更多采集点类型。

**配置字段建议**（采集点表，供地图与逻辑读取）：

- `gathering_point_id`：与地图元素 id 一致（如 `gathering_bush`、`gathering_grass`）。
- `display_name`：展示名。
- `base_gathering_success_rate`：该采集点类型的**基础采集成功率** base（如 0.6 表示 60%）；灌木丛与草丛**建议均为 0.6**。实际成功率 = **base + base × (熟练度% × 0.003)**（熟练度% 为 0～100）。
- `loot_table_id`：产出表 id，指向该类型采集点使用的产出表。
- `stamina_cost`：每次采集体力消耗（当前约定 2）。
- `tool_required`：是否强制工具（当前均为 false）。工具对采集的加成在**工具表**中配置：`gathering_success_bonus`（成功率加成）、`gathering_quality_bonus`（高品质/稀有概率加成）等。

**产出表**：

- **表结构**：每条（每行）为 **物品 id + 权重 + 品质档位**，在同一行上直接写权重即可。**品质 1～6 分别对应粗糙、普通、精良、稀有、史诗、传说**（品质即稀有度，不单独再 roll「稀有」）。**抽取与品质上修**：先按表内权重随机到一行得到物品 id 与基础品质档位，再按**采集熟练度**做一次品质上修：熟练度每高 5%，有 5% 概率上修一档（最高到传说），见上流程步骤 3。
- **灌木丛与草丛表完全独立**：两张产出表互不共用条目，各自配置。
- **内容倾向（暂定）**：
  - **灌木丛产出表**：以**野果**类物品为主；具体物品 id、权重、品质档位在表中配置。
  - **草丛产出表**：以**药草**类物品为主；具体物品 id、权重、品质档位在表中配置。
- **物品 id**：由设计/配置自行生成或命名，在物品表与产出表中一致引用即可。
- **单件重量**：野果、药草在物品表中均约定单件 **0.1 kg**（如物品表字段 `weight_kg` 等）；负重与背包判定按此计算。
- **品质范围**：采集产出的野果、药草均可出现**品质六档**（1～6 = 粗糙～传说）；产出表每行配置品质档位与权重。

**配置字段建议**（产出表每行）：`item_id`、`weight`、`quality_tier`（1～6 对应粗糙～传说）；产出表 id 与采集点表之 `loot_table_id` 对应。物品表中野果/药草类物品的 `weight_kg` 建议为 0.1。

**工具表**（与采集相关）：增加 `gathering_success_bonus`（对采集成功率的加成）、`gathering_quality_bonus`（对高品质/稀有概率的加成）等字段，具体字段名与数值在工具表配置。

| 类别 | 说明（待细化） |
|------|----------------|
| **徒手** | 徒手战斗技能（拳脚、指法等）。 |

#### 8.3.1 战斗技能通用规则

- **技能等级与潜能消耗（四档）**
  - 所有战斗技能的等级上限为 **1000 级**，使用**潜能**进行升级。
  - 战斗技能按**难度系数**分为**四档**，**共用同一条潜能消耗曲线**，仅**基数**不同：同一等级所需潜能 = **基础曲线该等级消耗 × 难度系数**（难度 1/2/3/4 对应系数 1×/2×/3×/4×）。故一档练满约 2000 万，二档 4000 万，三档 6000 万，四档 8000 万。
  - **基础曲线（难度 1 时的每级潜能消耗）**——三段设计，200 级前平缓、400 级后明显变陡：
    - **1～200 级**：每级消耗 **5000** 潜能（常数），合计 100 万，便于快速到 200 级。
    - **201～400 级**：每级消耗**线性**从 5000 增至 **20000**，即 \(Cost(L) = 5000 + 75 \times (L - 200)\)（\(L \in [201,400]\)），合计约 250 万。
    - **401～1000 级**：每级消耗**线性**从 20000 增至 **35000**，即 \(Cost(L) = 20000 + 25 \times (L - 400)\)（\(L \in [401,1000]\)），合计约 1650 万。
    - 以上三段合计约 **2000 万**（难度 1）；难度 2/3/4 时每级消耗分别 ×2/×3/×4，总消耗约 4000 万/6000 万/8000 万。
  - **实现约定**：上述常数（5000、20000、35000、75、25）及难度系数从配置表读取，便于调参。**难度在配置技能时声明**：在战斗技能配置中用字段**难度 1 / 2 / 3 / 4** 表示所属档位；**未配置或未填写时默认难度 1**。策划在设计每条战斗技能时须明确声明该技能的难度档位。

- **技能等级上限的额外约束（可选）**
  - 设计战斗技能时，可声明该技能的**当前等级不得超过**某一指定战斗技能的当前等级；约束**一律不得超过**，即升级、师傅传授、载入存档等所有会改变等级的场合都须校验，依赖技能等级下降（如死亡扣级）时，本技能等级若高于依赖则夹紧至依赖技能当前等级。
  - **依赖关系不能成环**：若技能 A 声明等级不得超过技能 B，则 B 不得再声明等级不得超过 A（或经 A 传递形成的任何环）；设计时与配置校验时均须保证无环。
  - 示例：后续设计「散打」时，可声明散打的当前等级不能超过「基本拳脚」的当前等级；未声明则仅受全局 1000 级上限与师傅可传授上限约束。
  - **实现约定**：技能配置中可选字段（如 `level_cap_skill_id`）引用另一战斗技能 ID；所有会改变等级的逻辑（升级、传授、载入、死亡扣级等）均须校验并夹紧；配置加载或编辑时校验依赖关系无环。

- **战斗技能升级途径**
  - 战斗技能等级提升（消耗潜能）可通过以下两类途径获得进度：
    - **师傅 NPC 直接传授**：与具备传授该技能条件的师傅 NPC 进行「学习」交互，**学习一次消耗 10 点精力**，推进 **1 tick**；潜能消耗与升级进度按该技能难度与当前等级依上文曲线结算。**师傅只能传授到其自身该技能的等级为止**：师傅 NPC 对该技能有一个**可传授上限等级**，玩家通过该师傅学习时，最多只能被教到该等级；在学习界面中需展示「师傅该技能等级」或「可教至多少级」，供玩家判断。**设计 NPC 时声明**：每位师傅在各技能上的可传授等级（或上限）在配置该 NPC 时单独声明。
    - **自修**：角色可挂机进行自修；**每 1 tick 固定消耗 1 点精力**，潜能消耗与升级进度按同一曲线结算。**部分技能自修前需满足特定条件**（如已习得某技能、达到某属性、完成某剧情等）；若该技能有自修门槛，须在**设计该技能时在配置中声明**（如条件类型、判定字段或任务 ID），未满足条件时不可选择该技能进行自修。
  - **实现约定**：传授的“学习一次 = 10 精力 + 1 tick”与自修“每 tick 1 精力”的常数可从配置表读取；技能配置中为每条技能标注是否可被传授、自修条件（可选，无则默认可自修）。**NPC 配置**：设计师傅 NPC 时须为每个可传授技能声明该师傅的**可传授上限等级**（即师傅「会这个技能多少级、最多能教到多少级」）；学习界面读取并展示该上限，供玩家查看。

- **技能等级与基础威力**
  - 每个战斗技能都有一条独立的“**技能等级 L → 基础威力系数 Base(L)**”曲线，用于定义该技能在不同等级下的**基础威力**。
  - 单次招式在计算伤害时，先取所属战斗技能当前等级 \(L\) 对应的基础威力系数 `Base(L)`，再乘以该招式的熟练度乘区与其他乘区（呼吸法威力、出力大小、环境、实战经验等）。

- **招式与招式熟练度（参与伤害）**
  - 每个战斗技能包含若干个**招式**；**每个招式都是独立实体，只隶属于某一条战斗技能，不跨技能共享**。
  - 每个招式有自己的**使用次数计数** \(P\)，仅在**该招式成功命中目标时**计数 +1：
    - 打空、被闪避或被招架不增加使用次数；
    - 若为多段攻击，只要本次释放中**至少一段命中**，本次也只视为使用 1 次。
  - 统一约定：**单个招式满熟练度 = 使用 50000 次 = 100% 熟练度**。
  - 记某招式当前使用次数为 \(P\)，则其**熟练度比例**为：
    \[
    R_{\text{move}} = \min\left(\frac{P}{50000},\,1\right)
    \]
  - 该招式在技能等级为 \(L\)、使用次数为 \(P\) 时的**最终威力系数**为：
    \[
    Final_{\text{move}}(L, P)
    = Base(L) \times (1 + R_{\text{move}})
    \]
    - 即：**这一次出手是哪一招，就只看这招自己的熟练度**；
    - 熟练度满时，该招式在当前技能等级基础上获得 **+100%（×2）** 的线性加成。
  - **熟练度上升速度**：与生活技能一致，某些手段可提升或降低招式熟练度上涨速度；该接口需为后续扩展预留（实现时预留可插拔的倍率/修正入口，不在此文档枚举具体手段）。
  - **实现约定**：伤害公式中的“技能/招式熟练度乘区”，一律使用**“当前出手招式的熟练度 \(R_{\text{move}}\)”**，**不再使用“整条战斗技能的总熟练度”作为伤害乘区**。

- **技能总熟练度（不参与单次伤害，只用于奖励）**
  - 为了体现“整条功夫练得有多熟”，每条战斗技能在内部维护一个**技能总熟练度**，用于解锁该技能专属的被动奖励。
  - 设某条战斗技能下共有 \(N\) 个招式，第 \(i\) 个招式的熟练度比例为 \(R_i\)（按上文 \(R_{\text{move}}\) 规则计算）。
  - 该战斗技能的**技能总熟练度**定义为所有招式熟练度的**算术平均值**：
    \[
    SkillProficiency
    = \frac{1}{N}\sum_{i=1}^{N} R_i
    \]
  - 直观理解（以 4 招为例）：
    - 单招式满熟练度时 \(R_i = 1\)，对总熟练度贡献 \(1/4 = 25\%\)；
    - 四招全满时，总熟练度 = 100%；
    - 若只练满两招，其余两招为 0，则总熟练度为 50%。
  - **用途限定**：
    - **技能总熟练度不参与任何单次伤害计算**，只作为“该技能整体掌握程度”的指标；
    - 仅用于本技能对应的**熟练度奖励设计**（见下“后遗症/构式”）。
  - **实现约定（配置表）**：
    - 具体“在多少 % 技能总熟练度时解锁哪些奖励”，完全由**战斗技能配置表定义**；
    - 本文档不写死统一档位，设计战斗技能时必须**同时设计该技能的熟练度奖励触发点与效果**。

- **透支：战斗技能的被动槽位解锁**
  - 特殊技能「透支」通过剧情解锁后，**为战斗技能开启被动槽位**：
    - 每条战斗技能可以拥有若干个被动槽位，用于装备该技能对应的“后遗症”（构式）；
    - 槽位数量与上限由技能配置表定义（如字段 `max_post_effect_slots`），本节不写死具体数值。
  - 槽位为**挂在战斗技能上的局部被动槽**，而非全局统一被动栏；后遗症的作用范围见下「后遗症」小节。

- **后遗症（由技能熟练度解锁的构式，被动技能）**
  - **定义与性质**：
    - **后遗症**是一类由战斗技能的**技能总熟练度**解锁的**被动技能**；
    - 在实现与数据结构上，后遗症是一种**可装备的构式**：只要装备在某条战斗技能的被动槽位中，就持续生效。
  - **效果形态**：
    - 后遗症不局限于“+x% 伤害”类数值增益，可以改变规则或标签，例如：
      - 示例：「以掌为刀」——当装备者使用**钝击类伤害的招式**时，将本次攻击的**伤害类型从“钝击”改为“劈砍”**，后续伤害类型微调与部位系数均按“劈砍”结算。
  - **绑定关系与作用范围**：
    - 默认情况下，后遗症只对**挂载它的那条战斗技能**生效：
      - 例如某后遗症装备在「基本拳脚」上，则只影响「基本拳脚」下的招式；
      - 不影响其他战斗技能，除非该后遗症在配置中明确声明为“全局作用”。
  - **作用范围接口预留（实现约定）**：
    - 在后遗症/构式配置中预留作用范围字段（示意）：
      - `scope = "self_skill"`：默认，只影响挂载它的这条战斗技能；
      - `scope = "global"`：影响所有满足条件的招式/行为（全局被动）；
      - 未来可扩展 `scope = "tag:xxx"`：仅影响带特定标签的技能或招式。
    - 实现时，后遗症的逻辑统一从该字段判断作用范围，便于后续扩展“影响全局”的特殊后遗症，而不修改底层结构。
  - **解锁方式（实现约定）**：
    - 每条战斗技能的**技能总熟练度档位**与对应可解锁的后遗症列表，在该技能的配置表中定义；
    - 设计战斗技能时，须同时给出：
      - 技能本身的等级曲线 `Base(L)`；
      - 该技能下所有招式列表与招式熟练度规则（沿用通用规则即可）；
      - 技能总熟练度的奖励设计（在哪些熟练度百分比解锁哪些后遗症）；
      - （若适用）**等级上限依赖**：本技能等级不得超过哪条战斗技能的等级；若无则不必声明。

- **装备到肢体与出招循环**
  - 战斗技能必须**装备在肢体上**才能在战斗中使用，当前可装备的肢体包括：左手、右手、左脚、右脚。
  - 每条肢体同时可装备 **1 个主动战斗技能 + 1 个招架技能**。
  - 每个战斗技能拥有若干个**招式槽**；玩家从已解锁的招式中选择装入槽位并设定顺序。
  - 每条装备了主动技能的肢体，都会维护一条**独立的招式循环序列**（只在该肢已装备的招式中循环）。
  - 玩家可以为四个肢体设定**出招优先顺序**（如：左手 → 右手 → 左脚 → 右脚）；在一次攻击中：
    - 系统在“未损毁且当前有可用招式”的肢体中，按优先顺序选择一条肢体作为本次出招肢体。
    - 若该肢体已损毁，直接跳到下一条肢体；若所有相关肢体均不能出招，则本次攻击视为无效。
  - 当某肢体被选中出招时：
    - 按该肢体的招式循环，取出**当前轮到的招式**尝试释放。
    - 如果因为动作标签不符、肢体状态异常等原因导致招式**无法释放**，本回合动作白费，但该肢体的招式指针仍视为“已轮到过一次”，在下次轮到该肢体时会进入下一招。

- **动作标签与肢体状态**
  - 每个招式都要求装备肢体具备某些**动作标签**（例如“拳击”“勾击”“踢击”等）。
  - 若肢体不具备招式所需的动作标签，则该招式既不能装入该肢的槽位，也无法在战斗中释放。
  - 肢体的动作标签可以被一些系统实时增减（例如通过特殊装备、buff、或永久改造等方式）；标签变化**即时生效**。
  - 若玩家在某肢上预先装备了需要特定标签的招式，而战斗中该肢失去了对应标签，则那些招式视为暂时不可用。
  - 肢体一旦被判定为“损毁”，则视为该肢无法再使用任何主动或招架招式，直到通过特殊手段恢复。

- **招架相关声明与接口**  
  - 招架等级随**本体战斗技能**等级，不单独设招架等级。  
  - 招架相关数值（基础招架成功率、基础卸力比例等）**随该技能等级线性提升**，具体数值在设计该技能时在技能配置中定义。  
  - 若某技能或招式**可用于招架**，须在技能/招式配置中声明：基础招架成功率、基础卸力比例（可为等级函数或表）。  
  - 若招架成功时有**特殊效果**（如回复底气、反击等），须声明效果内容及该效果**是否受柔韧的招架效果加成**；实现上保留招架特殊效果的调用接口（按技能/招式 ID 或标签触发），供扩展使用。  
  - 部分技能**仅能作为招架技能**装备（不提供主动攻击招式、不可装备为肢体的主动战斗技能），须在技能配置中标注。

#### 8.3.2 「基本拳脚」示例

作为徒手类战斗技能的代表，「基本拳脚」使用上述通用规则，并在此基础上补充以下具体设定：

- **装备与作用肢体**
  - 「基本拳脚」可以装备在左手、右手、左脚、右脚任意肢体上，只要该肢体未损毁且具备至少一个所属招式所需的动作标签。
  - 一个角色可以在多条肢体上同时装备「基本拳脚」；出招与招式循环规则按 8.3.1 通用规则执行。

- **招式与解锁等级**
  - 「基本拳脚」下包含四个基础招式：
    - **直拳**：技能等级 **1 级** 解锁。
    - **前踢**：技能等级 **1 级** 解锁。
    - **勾拳**：技能等级 **50 级** 解锁。
    - **鞭腿**：技能等级 **50 级** 解锁。
  - 各招式的具体数值定位（轻攻/重攻/控制等）、气劲消耗与命中部位权重将在后续细化。

- **招式槽与初始配置**
  - 角色在学会「基本拳脚」时，默认已掌握直拳与前踢，对应**立即解锁 2 个招式槽**。
  - 当技能等级达到 50 级并学会勾拳与鞭腿时，每学会一个新招式**额外解锁 1 个招式槽**，最终该技能共计 **4 个槽位**。
  - UI 中仅显示当前已解锁的槽位，未解锁的槽位不显示，避免在早期暴露未来结构。
  - 玩家可以在每个槽内自由选择已解锁的招式，并为装备了「基本拳脚」的每条肢体分别设定槽内招式顺序；不同肢体可以有完全不同的出招顺序。

- **等级与基础威力曲线**
  - 「基本拳脚」的基础威力系数以 1 级为基准，记为 \(Base(1)=10\)，并随技能等级 \(L\) 提升；1000 级时约为 1 级的 10 倍，即 \(Base(1000)\approx 100\)。
  - 为体现“前期提升明显、后期逐渐变缓”，定义一个相对 1 级的倍数函数 \(M(L)\)（1 级时 \(M(1)=1\)），分三段线性插值：
    - **1 ≤ L ≤ 200**：从 1 倍平滑提升到约 4 倍：
      \[
      M(L) = 1 + \frac{3}{199}\,(L - 1)
      \]
    - **200 < L ≤ 400**：从约 4 倍提升到约 7 倍：
      \[
      M(L) = 4 + \frac{3}{200}\,(L - 200)
      \]
    - **400 < L ≤ 1000**：从约 7 倍提升到约 10 倍：
      \[
      M(L) = 7 + \frac{3}{600}\,(L - 400)
      \]
  - 对应的基础威力系数为：
    \[
    Base(L) = 10 \times M(L)
    \]
  - 在原始伤害公式中，「基本拳脚」各招式的“招式威力系数”部分即取自该 \(Base(L)\) 再结合熟练度加成。

- **熟练度与最终威力**
  - 「基本拳脚」下各招式的熟练度计数、熟练度比例 \(R_{\text{move}}\)、最终威力系数 \(Final_{\text{move}}(L, P) = Base(L) \times (1 + R_{\text{move}})\) 等，均按 8.3.1「招式与招式熟练度（参与伤害）」通用规则执行。
  - 该技能的**技能总熟练度**按 8.3.1 的算术平均折算；四招全满时技能总熟练度为 100%，用于解锁该技能对应的后遗症（构式）。

- **永久属性加成（后天筋骨与后天柔韧）**
  - 「基本拳脚」作为长期修行的基础功法，会在高等级时为角色提供永久的身体素质加成：
    - 每当「基本拳脚」等级累计提升 **200 级**，角色获得：
      - **后天筋骨 +1**
      - **后天柔韧 +1**
  - 该加成为全局永久加成：
    - 与当前是否装备「基本拳脚」无关，一旦获得即对所有战斗与判定生效。
    - 加成计入后天部分，不会突破先天属性的 40 点上限设定，但会与先天值相加用于实际数值计算。

#### 8.3.3 「基本招架」

- **定位**  
  - **仅能作为招架技能**装备；不提供主动攻击招式，不可装备为肢体的主动战斗技能。可与任意主动战斗技能搭配装备于同一肢体的招架槽；招架等级 = 本技能等级。

- **等级上限**  
  - 1000 级；升级方式与其它战斗技能一致（潜能等）。

- **招架数值（线性随等级）**  
  - 满级 1000 时：**基础招架成功率 = 35%**，**基础卸力比例 = 35%**。  
  - 线性成长公式（\(L\) = 技能等级，\(L \in [1, 1000]\)）：  
    - **基础招架成功率**\((L) = 0.35 \times (L/1000)\)  
    - **基础卸力比例**\((L) = 0.35 \times (L/1000)\)  
  - 再经柔韧乘算：实际招架几率 = 基础招架成功率\((L)\) × (1 + 0.005×柔韧)，卸力比例同理（系数 0.005 来自配置 `parry_flexibility_mult_per_point`）。

- **无招架特殊效果**  
  - 不回复底气、不反击等；若有后续扩展再在技能配置中声明并走招架特殊效果接口。

- **实现约定**  
  - 技能 ID 建议 `combat_basic_parry`。配置表中标注「仅招架技能」、不可装备为主动技能；为该技能提供等级→基础招架成功率/基础卸力比例表或上述公式参数，便于调参。

### 8.4 特殊系

特殊系技能**不在常规升级/点数树中**，仅通过**剧情推进或特殊任务**解锁，每个都有独特作用，通常用于：

- 改变部分底层规则（例如某些惩罚或限制的例外情况）；
- 开启全新玩法或交互（例如特殊视角、特殊结算方式）；
- 提供无法通过数值堆叠获得的能力。

具体特殊技能列表在剧情与任务设计阶段一并细化。

#### 8.4.1 特殊技能栏

- **地位与结构**
  - 特殊技能栏与**战斗技能栏、生活技能栏、生存技能栏**并列，为同一层级的**独立技能栏**；角色拥有四个技能栏：生存、生活、战斗、特殊。
  - 特殊技能栏内展示与管理的，均为**特殊系技能**（通过剧情或特殊任务解锁，不进入常规升级/点数树）。

- **技能用途（判定与认知）**
  - 栏内技能**绝大多数**用于被**其他技能、事件、系统**引用做判定，例如：
    - 作为门槛（如「拥有某特殊技能才可学习/使用某战斗技能」）；
    - 作为事件分支条件（如剧情/任务根据是否拥有某特殊技能走不同分支）；
    - 作为是否开放某功能、某交互的判定条件。
  - **认知引擎**以「角色是否拥有/解锁栏内某特殊技能」为条件，判定是否**开启或展示对应的 UI**（例如某界面、某入口、某选项是否对玩家可见或可用）。即：特殊技能栏既是数据来源，也是 UI 开放与否的判定依据。

- **实现约定**
  - 在角色/存档数据结构中，特殊技能与生存/生活/战斗技能一样，占**独立栏位**（如 `skills.special` 或等价结构），便于与其它三栏统一处理。
  - 认知引擎在决定是否展示某 UI 时，应查询**特殊技能栏**内是否包含配置所要求的特殊技能（或等级等），再决定是否开启该 UI。

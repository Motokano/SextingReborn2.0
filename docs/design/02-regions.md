# 二、区域结构

### 2.1 基地（Home）

- **功能**：休整、存档；永久成长（战斗技能等级、各生产线熟练度、建筑等级）；准备本次冒险（装备、消耗品）；生产活动；与固定 NPC 对话/接任务；投保 NPC（装备快照与 AES-256 代码）。
- **生产活动**（材料 + 配方/图纸 → 产物）：
  - **制造**：一切基础；建筑所需材料与绝大部分工具来自制造。
  - **烹饪**：野外材料 + 贸易配方 → 料理，提供显著 buff。
  - **炼药/炼金**：材料 + 配方 → 战斗中即时使用的药品。
  - **锻造**：材料 + 配方/图纸 → 武器。
  - **纺织**：材料 + 配方 → 丝绸相关产物。
  - **建造/升级生产建筑**：材料主要来自制造。
  - **附魔**：地牢特殊材料 + 附魔建筑 → 装备特殊强化。
- **生产规则**：
  - 有制作时间；可多建筑并行，单建筑同一时刻只能制作 1 件；失败全额扣材料。
  - 熟练度：各线独立、不共用；角色等级 + 建筑等级；仅成功制作涨熟练度；成功/失败与熟练度挂钩；建筑升级可缩短时间、增加产出、降失败率、提品质。
  - 品质六档：粗糙 → 普通 → 精良 → 稀有 → 史诗 → 传说；颜色：灰 → 白 → 绿 → 蓝 → 紫 → 橙。

### 2.2 野外（Field）

- **定位**：低烈度区；移动、采集、与较弱敌人战斗，积累资源，为地牢做准备；经验与掉落明显低于地牢。
- **地图**：主路相对安全（几乎不遇敌）；支路/偏远区敌人与采集点更密集；资源点与城镇/地牢入口为固定格子，每局位置不变。
- **采集点**（采完后按“行动回合数”刷新）：矿脉、药草、木材/竹材、狩猎点、丝绸相关、钓鱼（食材 + 稀有材料/装备）；不强制工具，有工具效率更高。
- **敌人**：不主动远距离追逐；玩家靠近到一定范围后朝玩家移动并可能触发战斗；普通天气下弱于地牢，特定天气/时间下强度可接近低层地牢。
- **天气与时间**：影响资源品质（原有点提升 + 专属高品质点）；影响怪物种类与强度。
- **回程**：角色无便捷回城；需走回城镇/安全出口；城镇 ↔ 基地有需条件解锁的快捷传送。采集不额外增加危险。

### 2.3 城镇（Town）

- **核心**：贸易（装备、材料、配方等）。
- **附加**：有限休整、特殊服务（如鉴定）、与解锁部分传送/设施相关。

### 2.4 地牢（Dungeon）

- **结构**：多个随机生成的小房间组成大迷宫；可配置层数；房间间通过门格切换；进入后无法退回野外，只能找出口撤离或下潜。
- **无通关概念**：收益 = 实际拿到并带出的东西；系统不按层数/时间/击杀给额外结算奖励。
- **战斗**：敌人朝玩家靠近，接近触发战斗；玩家可提前离开房间完全绕过战斗（含 Boss）。

### 2.5 游戏界面与画面

- **整体原则**：与「一、设计哲学」一致，0 教学、0 引导；界面只提供状态与操作入口，不弹教程。
- **通用布局**：
  - **主视图**：当前区域的主要画面；基地/城镇/野外/地牢均为**格子地图 + 角色 + 实体**，玩家在格子上移动。
  - **HUD/状态区**：生存与战斗相关量（饱食、饮水、体力、底气、精力、负重等）；必显项与可选折叠由实现与配置决定；数值来源见「五、角色基础属性」「六、生存属性」。
  - **快捷操作**：背包、技能、装备、行气、设置/存档等入口；是否占 tick 与「十、实现约定」一致（如打开背包消耗 1 tick）。
- **右侧技能/系统入口（与 demo 布局一致）**：界面**右侧**为竖向排列的**三个入口按钮**，即**技能系统与背包的 UI 入口**，实现时须保持该布局：
  1. **背包**（🎒）：点击打开背包/物品栏与装备栏面板（见下「背包面板与装备栏」）。
  2. **战斗技能**（⚔️）：点击打开战斗技能枢纽弹窗（武学枢纽：呼吸/轻功核心、肢体负载、招式序列等，见 demo 中 `#modal-combat`）。
  3. **生活技能**（⚙️）：点击打开生活技能/生产相关面板（制造、烹饪、炼药、锻造、纺织等入口或总览）。
  - 布局约定：三钮竖直排列、等大（如 56×56 px）、间距统一（如 24 px）；位于视口右侧、垂直居中；样式与 demo 一致（深底、描边、hover 高亮、战斗技能可带强调如 scale）。该右侧栏为技能系统与背包的**唯一常驻入口**，子界面以弹窗/侧滑等形式打开。
- **左侧人物状态区（与 demo 布局一致）**：界面**左侧**为竖向滚动的**人物状态面板**，今后凡展示人物状态信息均沿用该样式与结构，实现时须与 demo 保持一致：
  - **容器**：左侧固定宽度（如 288px / w-72），上留白、下留白（避免压住底部日志），中间为可滚动区域；深色半透明底、整体 flex 纵向排列。
  - **角色卡**（顶部）：角色名大号加粗白字、字间距加宽；副行小号金色（#fbbf24）显示称号（如「称号 · 初入末世」）；卡片背景 #2d241e、描边 #d4a373/40、内边距、阴影。
  - **即时机能/状态块**：小卡片为单位，背景 #1c1612、描边 #4d3f35、内边距；正文 11px、加粗。每行「标签 + 数值」左右对齐：标签用 stone-500，数值用白/绿/琥珀等表状态（如健康/轻快/轻伤）。
  - **肢体检视块**：同上卡片样式；块顶有小标题「肢体检视」——9px、金色、半透明、全大写、字间距加宽。七部位每行「部位名 + 状态」（完好 / 轻伤(骨折) 等）；异常状态用琥珀色强调。
  - **生存循环块**：同上卡片；每项为「标签 + 文字状态」一行（如饱食/饮水），下方一条细进度条：背景深色、填充色按类型区分（如饱食橙、饮水蓝），宽度按当前数值比例。
  - **快捷腰带**：本质为**口袋 + 背心**的格数映射，即口袋与背心内所有格在快捷栏中的展示；**格序为先口袋、后背心**（索引 0～口袋格数-1 为口袋，其后为背心）。布局为 **3 列、多行**，行数由当前口袋格数 + 背心格数决定（无衣服则无口袋格，无背心则无背心格，对应格不显示或禁用）。每格方形容器，深底、细描边、hover 高亮金色；格内为图标或物品，可点击。
  - **可折叠块**（如基础五维）：`details`/`summary` 实现；标题行小号、灰色、全大写；展开内容与上述卡片同风格，行内「属性名 + 描述值」左右对齐。
  - **滚动与样式**：状态区内容可纵向滚动，滚动条细窄、金色系；块与块之间统一间距（如 space-y-3）。所有文案可从配置/表读取，数值与「五、角色基础属性」「六、生存属性」「九、身体部位」等对应。
  - **装备摘要**：左侧状态区**不展示**当前装备摘要；装备**仅在打开装备面板时**查看（见下「背包面板与装备栏」）。
- **背包面板与装备栏**：点击右侧「背包」按钮后，打开**同一大面板**；该面板内以 **Tab 或左右分栏** 同时提供「物品栏」与「装备栏」视图。**打开**该面板消耗 1 tick，**关闭**消耗 1 tick；**在「物品栏」与「装备栏」之间切换不额外消耗 tick**。物品栏视图中，四类容器（口袋、背心、背包、载具）为同一面板内**从上到下四个区域**，未装备背心/背包或未拖拽载具时，对应区域不显示（直接去掉该块）。装备栏视图中，15 个装备槽以**简单列表或网格**呈现，清晰可点即可；槽位定义见装备栏设计文档（如 05 与独立装备栏文档）。
- **分区域**：基地、城镇、野外、地牢均以**同一套格图交互**呈现——主视图为可移动的 2D 格图，相机跟随玩家；生产、贸易、NPC 等通过**走到对应建筑/格位再交互**进入子界面或弹窗。

### 2.6 地图结构

- **拓扑**：采用**多张独立格图**。野外、基地、城镇、地牢（每层或每房间）各为一张格图；区域切换通过**格上传送点**完成，不共用同一张大世界图。
- **格子**：
  - 每张地图为**矩形格网**，宽度与高度**按配置表**（每图可不同）；格子在逻辑上为整数坐标 (x, y)。
  - 视口固定大小（如 744×456 px），格子在视口内以固定步长渲染（如 demo 的 72 px/格）；实现时步长与视口尺寸可配置。
- **传送点**：
  - 部分格子为**传送点**。玩家**移动进入该格**时触发切换：当前地图 → 目标地图，玩家出现在目标地图的**指定格子**。
  - 传送点配置至少包含：本图坐标 (x, y)、目标地图 ID、目标图落点坐标 (target_x, target_y)。若目标落点为「返回进入前所在格」，则由运行时在进入某图时记录入口格，离开时读回；配置上可用缺省或特殊标记表示「使用记录中的入口」。
- **渲染**：格子层**继续用 DOM** 绘制（与现有 demo 一致）；Canvas 用于特效、角色等。地图数据与传送点表存放于 `/data/`，结构见 2.8 及实现约定。

### 2.7 移动与区域切换

- **移动**：
  - 玩家在格图上**一步一格**移动（8 邻域）；每移动 1 格 = **1 tick**，触发饱食、饮水、体力等 tick 结算（见「六、生存属性」）；超重与严重超重规则不变。
  - 输入方式：点选相邻格或方向键一格，由实现决定；与战斗的 8 向、每回合 1 格移动一致。
- **传送点触发**：
  - 当玩家**步上传送点所在格**时，先完成本格移动（消耗 1 tick），再执行切换：加载目标地图，将玩家置于目标图的指定格；若目标为「返回」，则置于进入当前图时记录的**上一张图的入口格**。
- **往返约定**：从 A 图经传送点进入 B 图时，**记录 (A, 入口格坐标)**；从 B 图使用「返回」类传送点离开时，回到 A 图该入口格，不要求配置反向传送点坐标。

### 2.8 三张示例地图（野外、基地、城镇）

以下为最先实现的三张格图：**野外（field）**、**基地（home）**、**城镇（town）**。尺寸与传送点为示例，可依平衡调整。

| 地图 ID | 名称 | 宽×高（格） | 说明 |
|---------|------|-------------|------|
| `field` | 野外 | 25×25 | 主活动区；有通往基地、城镇的传送点；地牢入口可后续扩展。 |
| `home`  | 基地 | 16×16 | 休整与生产；有通往野外的传送点（出口）。 |
| `town`  | 城镇 | 16×16 | 贸易与休整；有通往野外的传送点（返回进入前所在格）。 |

- **野外 → 基地**：野外某格为传送点，目标地图 `home`，落点固定（如基地内 (8,1)）。
- **基地 → 野外**：基地某格为传送点，目标 `field`，落点使用**进入基地时记录的野外格**。
- **野外 → 城镇**：野外某格为传送点，目标 `town`，落点固定（如城镇内 (8,1)）。
- **城镇 → 野外**：城镇某格为传送点，目标 `field`，落点使用**进入城镇时记录的野外格**。

地图配置建议字段（存于 `/data/`，如 `maps/field.json`、`maps/home.json`、`maps/town.json`）：  
`map_id`、`name`、`width`、`height`、`portals`（数组，每项含 `x`、`y`、`target_map_id`、`target_x`、`target_y`；若为返回则 `target_x`/`target_y` 省略或填特殊值，由实现约定）、`npcs`（可选，数组，每项含 `x`、`y`、`npc_id`，其余 NPC 字段在「NPC 设计」确定后补充）。地形、资源点、敌人等为后续扩展，当前可只配传送点、尺寸与 NPC 摆放。

### 2.9 地图编辑器工具

- **用途**：方便策划/开发者设计格图、放置传送点与 NPC，并导出为上述地图 JSON，放入 `/data/maps/`。
- **必备功能**：
  - 新建/编辑地图：设置 `map_id`、`name`、`width`、`height`，生成格网；格子上可悬停显示坐标。
  - **传送点**：点选格子后，在侧栏填写目标地图、落点坐标、是否「返回上一格」、label；有传送点的格子在画布上高亮；支持编辑、删除。
  - **摆放 NPC**：点选格子后可在该格放置 NPC；侧栏填写 `npc_id` 及后续 NPC 设计确定的字段（如显示名、类型）；有 NPC 的格子在画布上单独标识；支持编辑、删除。同一格可仅允许一个 NPC（或按实现约定）。
  - 导出：生成与当前地图配置格式一致的 JSON（含 `portals`、`npcs`），可复制或下载后放入 `data/maps/`。
  - 导入（可选）：粘贴或选择已有地图 JSON，加载后继续编辑。
- **实现形式**：单页 HTML（如 `tools/map-editor.html`），浏览器打开即可使用，无需服务端。地形、资源点等可后续在编辑器中扩展。

### 2.10 实现约定（UI 与地图）

以下为与策划确认后的实现约定，实现时须遵守。

- **地牢格图**：地牢采用**每层一张格图**；一层内多个房间在同一张格图上，房间间通过门格切换，不采用「每房间一张图」。
- **返回传送**：「返回」型传送点表示回到**上一张图**的入口格（不是回到整条链的起点）。配置上**省略** `target_x`、`target_y`（不写或不存在）即表示该传送点为返回型；运行时用**入口栈**记录 (来源图, 入口格)，返回时 pop 取上一张图及入口格。
- **传送点配置**：地图有若干固定传送点，每个传送点固定传送到某张地图的某一格，或配置为返回型（见上）。入口栈在进入新图时 push，返回时 pop。
- **相机**：主视图相机**玩家始终在视口中心**；地图大于视口时会出现裁切。
- **HUD/状态区**：首版**全部显示**（饱食、饮水、体力、底气、精力、负重等）；实现时必须**预留技能判定接口**，成品中由该接口判断玩家是否拥有对应技能，再决定是否全部显示（未拥有时可按设计折叠或隐藏部分项）。
- **右侧三按钮（背包 / 战斗技能 / 生活技能）**：**打开**与**关闭**面板各消耗 **1 tick**；与「十四、实现约定」中 tick 成本配置一致。
- **背包面板与装备栏**：背包与装备为**同一面板**；打开/关闭该面板各 1 tick。面板内从「物品栏」切换到「装备栏」或反向切换**不额外消耗 tick**。
- **同格多交互**：若同一格存在多种可交互（如传送点 + NPC、建筑入口等），**弹出菜单**供玩家选择一项执行；传送点不占格，仅占格实体（NPC/资源点/敌人）与传送点可同格。
- **占格规则**：**资源点、敌人、NPC 占格**；主角不能站在有占格实体的格子上。**一格只有一种占格物**（要么一个 NPC，要么一个资源点，要么一个敌人，不能同格多类）。传送点不占格，可与占格实体共存于同一格。
- **地形**：支持**地形不可行走**（如水域、墙）。逻辑上仅区分**可走 / 不可走**；渲染上可按格或按层配置**不同图块**（多种贴图），扩展视觉表现。
- **出生点**：新游戏使用**固定出生点**（如基地某格），该出生点**可通过事件修改**（事件系统调用接口设置「下次新游戏的出生点」为指定 map_id + 坐标）。**读档**时使用存档中记录的**当前 map_id 与坐标**。实现时**必须提供事件改出生点的接口**。
- **入口栈**：区域切换时用**栈**记录入口；从 A 进入 B 时 push(A, 入口格)，从 B 使用返回型传送点时 pop 得到上一张图及落点格，实现连续「城镇→基地→野外」式返回。
- **主菜单**：首次打开或进入游戏前提供**主菜单**，选项包括「新游戏」「继续（读本地实时存档）」「载入存档码」；选择后再进入游戏。
- **底部日志**：有**固定底部日志区域**，用于显示最近若干条战斗/系统消息。该区域须支持**可拖动、可缩放**，方便玩家自行调整位置与大小；实现时需持久化玩家对日志区域的位置与尺寸设置（如写入本地配置或存档）。
- **设置/存档入口**：与现有 demo 一致——界面**右侧**有**齿轮按钮**，点击后展开或打开设置/存档等相关功能（设置、存档、读档等入口集中在此）。
- **同格交互菜单**：当同一格存在多种可交互时，弹出的菜单为**靠近该格/角色位置的浮动菜单**（非屏幕中央大模态框），选项列出传送、与 NPC 对话、进入建筑等，玩家点选一项执行。
